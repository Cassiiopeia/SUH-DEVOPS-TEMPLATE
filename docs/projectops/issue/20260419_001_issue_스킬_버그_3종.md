📝현재 문제점
---

- **산출물 경로 불일치**: `issue` 스킬이 `.issue/` 폴더에 파일을 저장하고 있으나, 다른 스킬(`plan`, `analyze`, `report` 등)은 `docs/suh-template/{skill_id}/` 경로를 사용함. 일관성이 없음
- **사용자 확인 없이 즉시 GitHub 등록**: 이슈 파일 생성 후 사용자 검토 없이 바로 GitHub API를 호출하여 이슈를 등록함. 내용 확인 및 수정 기회가 없음
- **PYTHONPATH 누락**: `python3 -m suh_template.cli` 호출 시 `PYTHONPATH`가 설정되지 않아 `ModuleNotFoundError: No module named 'suh_template'` 발생. `suh_template` 패키지가 `scripts/` 하위에 있어 명시적 설정 필요

🛠️해결 방안 / 제안 기능
---

- **산출물 경로 통일**: `get-output-path issue` 커맨드 사용하여 `docs/suh-template/issue/YYYYMMDD_번호_제목.md` 경로로 저장
- **확인 단계 추가**: 로컬 파일 저장 → 사용자에게 내용 확인 요청 → 승인 후 GitHub 등록 순서로 프로세스 재구성. 사용자 승인 전 GitHub API 호출 절대 금지
- **PYTHONPATH 명시**: 모든 `python3 -m suh_template.cli` 호출 앞에 `PYTHONPATH="$PROJECT_ROOT/scripts"` 추가. `common-rules.md`에 규칙 문서화

⚙️작업 내용
---

- `skills/issue/SKILL.md`: 산출물 경로 `.issue/` → `docs/suh-template/issue/`, 확인 단계 추가, PYTHONPATH 명시
- `skills/references/common-rules.md`: suh_template CLI 실행 시 PYTHONPATH 설정 규칙 섹션 추가
- `skills/references/doc-output-path.md`: PYTHONPATH 필수 명시
- 전체 스킬 13개 파일 PYTHONPATH 누락 일괄 수정
