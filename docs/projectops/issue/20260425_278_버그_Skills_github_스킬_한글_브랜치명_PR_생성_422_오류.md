🗒️ 설명
---

- 브랜치명에 한글이 포함된 경우(`20260420_#653_FCM_푸시_페이로드에_라우팅용_데이터_포함_필요`) GitHub API PR 생성 시 `422 Validation Failed: head invalid` 오류 발생
- 한글 브랜치명이 실제로 remote에 push되어 있어도 API 요청의 `head` 필드에서 invalid로 처리됨

🔄 재현 방법
---

1. 한글이 포함된 브랜치명으로 작업 후 remote push
2. `cassiiopeia:github`로 PR 생성 요청
3. API `head` 필드에 한글 브랜치명 전달 시 `422 Validation Failed` 발생

📸 참고 자료
---

- PowerShell `ConvertTo-Json`이 한글 포함 문자열 인코딩 불일치 가능성
- GitHub API `head` 필드는 `owner:branch` 형식도 지원함

✅ 예상 동작
---

- PR 생성 전 `git ls-remote origin "브랜치명"`으로 remote 브랜치 존재 여부 사전 확인
- `head` 필드에 `owner:branch` 형식(`TEAM-ROMROM:브랜치명`) 사용으로 레포 명시
- 한글 포함 브랜치명은 URL 인코딩 처리 후 API 호출

⚙️ 환경 정보
---

- **OS**: Windows 11
- **브랜치 형식**: `YYYYMMDD_#번호_한글제목` (SUH-DEVOPS-TEMPLATE 표준 브랜치명)
