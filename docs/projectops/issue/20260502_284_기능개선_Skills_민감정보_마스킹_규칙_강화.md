# 🚀[기능개선][Skills] 민감정보 마스킹 규칙 강화 — DB유저명 누락 및 *** 익명별표 허용 문제

- **라벨**: 작업전
- **담당자**: Cassiiopeia

## 📝 현재 문제점

`skills/references/common-rules.md`의 민감정보 마스킹 규칙에 두 가지 누락/오류가 있어 실제 운용 시 민감 정보가 문서에 노출되는 사고가 발생했다.

### 문제 1: DB 유저명이 마스킹 대상에서 누락

마스킹 규칙 표에 API Key, Password, 서버 주소 등은 있으나 **DB 유저명(계정명)** 항목이 없었다.
AI가 `username: kimchi` 같은 실제 계정명을 위험 항목으로 판단하지 못해 그대로 문서에 기재됨.

**영향 파일**: `skills/references/common-rules.md`

### 문제 2: `***` 익명 별표 허용 문구

"스크린샷/로그 인용 시 민감 값은 `***` 또는 플레이스홀더로 대체" 문구로 인해
AI가 `{PASSWORD}` 같은 명명된 플레이스홀더 대신 `********`를 사용하는 경우가 발생했다.
`********`는 어떤 종류의 정보인지 알 수 없어 문서 가독성과 재현성이 떨어진다.

**영향 파일**: `skills/references/common-rules.md`

### 문제 3: plan / analyze / ppt skill 자체검토 누락

파일을 저장하는 skill 중 `issue`, `report`만 "파일 저장 직전 자체검토 프로토콜" 참조가 명시되어 있고,
`plan`, `analyze`, `ppt` skill에는 해당 프로토콜 참조가 없었다.

**영향 파일**: `skills/plan/SKILL.md`, `skills/analyze/SKILL.md`, `skills/ppt/SKILL.md`

---

## 🛠️ 해결 방안

### common-rules.md

- 마스킹 규칙 표에 `DB 유저명 / 계정명 → {DB_USERNAME}, {ADMIN_ID}` 항목 추가
- 파일 저장 직전 자체검토 표에 `username: kimchi` 같은 계정명 노출 위험 예시 추가
- `***` 허용 문구 삭제 → "플레이스홀더로 대체 (`***` 금지)" 로 교체
- 판단 기준에 `{DB_USERNAME}` 안전 예시, 계정명 노출 위험 예시, `********` 금지 추가

### plan / analyze / ppt SKILL.md

- 각 skill의 "산출물 저장" 섹션에 자체검토 프로토콜 참조 문구 추가
