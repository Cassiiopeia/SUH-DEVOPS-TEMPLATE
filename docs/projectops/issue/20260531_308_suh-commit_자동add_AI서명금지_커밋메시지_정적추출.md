# 🚀[기능개선][Skills] commit 자동 add·AI 서명 금지·커밋 메시지 정적 추출

라벨: 작업전
담당자: Cassiiopeia

📝 현재 문제점
---

- `commit` 스킬이 staged 파일이 없으면 즉시 멈추고 사용자에게 직접 `git add`를 요구해, 호출 후에도 커밋이 한 번에 끝나지 않음.
- 핵심 원칙에 "커밋하지 않는다"는 규칙이 명확하지 않아, 승인 후에도 존재하지 않는 규칙을 근거로 커밋이 거부되는 동작이 발생함.
- 생성되는 커밋에 AI 서명/푸터(`Co-Authored-By`, `Generated with Claude`, `🤖` 등)가 포함될 여지가 있어, 사용자가 직접 작성한 커밋처럼 보이지 않음.
- 커밋 메시지의 이슈 제목 부분을 agent가 매번 자유 해석으로 추출해, 가끔 이슈를 제대로 살피지 않고 엉뚱한 제목/요약을 넣는 오류가 발생함.

🛠️ 해결 방안 / 제안 기능
---

- staged 파일이 없으면 변경 파일을 자동으로 스테이징(`git add -A`)한 뒤 진행한다. 이슈별 분리 커밋이 명시된 경우에만 해당 파일만 선택적으로 add한다.
- 핵심 원칙에서 스테이징 자동화 규칙을 명확히 한다.
- 커밋 메시지에 AI 서명/푸터를 절대 포함하지 않는다. `--author` 옵션이나 추가 trailer 없이 사용자 git 설정 그대로 커밋해, 사용자가 직접 작성한 것처럼 보이게 한다.
- 커밋 메시지의 이슈 제목·URL은 agent 해석 없이 정적 추출한다. `SUH-ISSUE-HELPER`의 `extractIssueTitle` 로직(`[태그]`·이모지·제어문자 정규식 제거)을 스킬에 이식해, GitHub API 응답값을 정규식으로 정리한 값을 그대로 사용한다. agent의 자유 판단은 타입(feat/fix/…)과 변경설명으로만 제한한다.
- `git push`는 기존대로 실행하지 않는다(CLAUDE.md 규칙 유지).

⚙️ 작업 내용
---

- `skills/commit/SKILL.md` 핵심 원칙 수정: 자동 add, AI 서명 금지 규칙 추가
- 2단계(스테이징): staged 없으면 `git add -A` 자동 실행
- 3단계(이슈 조회): 이슈 제목·URL을 정규식 기반 정적 추출 스크립트로 결정적 추출 (SUH-ISSUE-HELPER 로직 이식)
- 5단계(메시지 제안): 제목·URL은 정적 추출값 그대로, 타입·변경설명만 agent 추천 후 사용자 승인
- 6단계(커밋 실행): AI 서명/푸터 및 `--author` 금지 명시
