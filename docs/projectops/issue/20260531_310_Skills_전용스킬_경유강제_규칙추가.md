📝 현재 문제점
---

- GitHub 이슈·PR·커밋·보고서 작업이 전용 스킬(`issue`/`github`/`commit`/`report`)을 거치지 않고 `curl`·GitHub API로 직접 처리되는 우회가 발생할 수 있다.
- 특히 "이슈 만들고 커밋하고 보고서까지"처럼 여러 작업이 한 번에 요청되면, "한 번에 끝내는 게 빠르다"는 판단으로 스킬을 건너뛰게 된다.
- 우회 시 스킬이 보장하는 절차 — **이슈 템플릿 규격, 제목 이모지+태그 prefix, 중복 이슈 검사, 로컬 `.md` 파일 저장** — 가 모두 누락되어 규격 미달 이슈가 생성된다.
- 기존 `common-rules.md` §"AI 행동 강제 원칙"에는 "확인 후 생성"만 있고, **"전용 스킬을 거쳐라"**는 명시가 없어 우회를 막지 못했다.

🛠️ 해결 방안 / 제안 기능
---

- `references/common-rules.md`에 §"전용 스킬 경유 강제(우회 금지)" 섹션을 신설한다.
  - 작업별 전용 스킬 매핑표: 이슈 생성→`issue`, PR/댓글/Secret→`github`, 이슈 기반 커밋→`commit`, 보고서→`report`
  - 복합 요청 처리 원칙: 여러 작업이 한 번에 요청되어도 각 단계를 전용 스킬로 순서대로 호출하며, 빠르다는 이유로 우회하지 않는다.
  - 예외 명시: 이미 생성된 이슈의 규격 보정, 스킬이 내부 절차로 위임한 curl 호출은 직접 호출 허용.
- `issue` 스킬 description을 보강한다.
  - 트리거 키워드 확장: "이슈 만들어줘/올려줘/등록/작성", "버그·기능요청·QA·디자인 요청 이슈" 등
  - 우회 금지 명시: curl 직접 이슈 생성 금지, 복합 요청에서도 이슈 생성 단계는 반드시 이 스킬 경유.

⚙️ 작업 내용
---

- `skills/references/common-rules.md`: §"전용 스킬 경유 강제" 섹션 추가 (모든 스킬 공유 규칙)
- `skills/issue/SKILL.md`: frontmatter description 트리거 키워드 + 우회 금지 문구 보강
- `github`/`commit`/`report`는 common-rules 강제표로 커버되므로 description 과수정은 하지 않는다 (scope 절제).
