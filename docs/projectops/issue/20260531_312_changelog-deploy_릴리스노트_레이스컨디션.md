# ❗[버그][Skills] changelog-deploy 릴리스 노트가 워크플로우 본문 초기화와 충돌해 사라지는 문제

라벨: 작업전
담당자: Cassiiopeia

📝 현재 문제점
---

- `changelog-deploy` 스킬로 deploy를 진행하면, 스킬이 작성한 릴리스 노트가 PR 본문에서 사라지고 CodeRabbit Summary 10분 대기로 빠지는 경우가 발생한다.
- 원인은 스킬과 `PROJECT-COMMON-AUTO-CHANGELOG-CONTROL` 워크플로우 간 실행 순서 충돌이다.
  - 스킬은 deploy PR을 **빈 본문으로 먼저 생성**한 뒤, 커밋을 분석해 릴리스 노트를 나중에 작성한다.
  - 워크플로우는 PR 생성(opened) 즉시 트리거되어 본문을 확인하는데, 이 시점에 본문이 비어 있으면 본문을 초기화한다.
  - 빈 PR이 잠깐 존재하는 순간 워크플로우가 끼어들어, 뒤늦게 작성된 릴리스 노트가 무시되거나 덮어써진다.
- 워크플로우 자체에는 이미 본문에 `Summary by CodeRabbit`이 있으면 초기화를 건너뛰는 조건부 로직이 있다. 즉, PR이 처음부터 릴리스 노트를 담고 생성되면 충돌이 발생하지 않는다.

🛠️ 해결 방안
---

- 워크플로우 로직(본문 초기화 조건부 처리)은 그대로 유지한다.
- 스킬의 실행 순서를 변경해, PR을 빈 본문으로 먼저 만들지 않고 릴리스 노트를 완성한 뒤 그 본문을 담아 PR을 생성한다.
  - 변경 전: push → PR 생성(빈 본문) → 커밋 분석 → 릴리스 노트 작성 → PR 본문 업데이트
  - 변경 후: push → 커밋 분석 → 릴리스 노트 작성 → PR 생성(릴리스 노트 본문 포함)
- 이렇게 하면 워크플로우가 PR 생성 직후 본문을 확인하는 순간 이미 `Summary by CodeRabbit`이 들어 있어, 본문 초기화를 건너뛰고 즉시 진행한다. 10분 대기 없이 automerge로 이어진다.
- main → deploy diff 및 커밋 메시지 분석은 기존과 동일하게 수행하며, 오히려 PR 생성보다 앞 단계로 이동한다.

⚙️ 작업 내용
---

- `skills/changelog-deploy/SKILL.md` deploy 모드 단계 재배치: 커밋 분석·릴리스 노트 작성을 PR 생성보다 앞으로 이동, PR 생성 시 본문에 릴리스 노트 포함
- fix 모드도 동일 원칙 점검 (fix 모드는 기존 PR 닫고 새로 여는 구조라 영향 확인)
