# 🚀[기능개선][Skills] changelog-deploy 릴리스 노트 사용자 승인 게이트 + 자동화 옵션

라벨: 작업전
담당자: Cassiiopeia

📝 현재 문제점
---

- `changelog-deploy` 스킬은 deploy 모드 5단계에서 릴리스 노트 파일을 작성한 직후 6단계 PR 생성으로 즉시 진행한다.
- 사용자가 PR이 생성되기 전 릴리스 노트 본문을 검토할 시점이 전혀 없다.
- 결과적으로 의도와 다른 본문으로 PR이 생성돼도 사용자가 개입할 수 없고, `PROJECT-COMMON-AUTO-CHANGELOG-CONTROL` 워크플로우가 automerge를 진행해 운영 배포까지 흘러간다.
- fix 모드(4단계 → 5단계)도 동일한 구조라 같은 문제가 발생한다.
- 반대로 자동화를 원하는 사용자(자리를 비우고 deploy를 돌리고 싶은 케이스)도 존재하므로 게이트를 일률적으로 강제하면 기존 사용 흐름을 해친다.

🛠️ 해결 방안 / 제안 기능
---

- deploy 모드 5단계와 6단계 사이에 **사용자 승인 게이트(5.5단계)**, fix 모드 4단계와 5단계 사이에 **승인 게이트(4.5단계)**를 추가한다.
- 게이트는 두 가지 모드를 지원한다:
  - **수동 승인 모드(기본)**: 릴리스 노트 본문을 표시하고 사용자 확인을 받은 뒤 PR 생성
  - **자동 진행 모드**: 본문 표시는 하되 확인 없이 즉시 PR 생성
- 설정 범위: **이 레포에서만 자동/수동** 또는 **모든 레포에서 자동/수동**의 두 단계로 선택 가능 (전체 기본값 + 레포별 오버라이드).
- 사용자에게 노출되는 안내는 자연어로만 구성한다. 설정 파일 경로·키 이름·내부 구조는 사용자에게 보여주지 않는다. agent가 사용자의 자연어 응답(예: "다음부턴 자동으로 해줘", "다시 수동으로")을 받아 설정을 직접 갱신한다.

⚙️ 작업 내용
---

- **SKILL.md (`skills/changelog-deploy/SKILL.md`)**
  - 핵심 원칙 섹션: "릴리스 노트 본문은 PR 생성 전 사용자에게 보여준다 (자동 모드로 명시 설정된 경우 제외)" 원칙 추가
  - 시작 전 단계: 현재 레포·전체 기준의 자동/수동 모드 값을 config에서 추출해 agent가 기억
  - deploy 5.5단계 신규
    - 자동 모드: 본문 표시 + "자동으로 설정되어 있어 바로 진행합니다. (수동으로 확인받고 싶으시면 말씀해주세요.)" 안내 후 6단계 진행
    - 수동 모드: 본문 표시 + 승인/수정 요청 분기
    - **첫 실행(설정 누락 상태)**: 수동 승인 직후 한 번만 "다음부턴 이 확인 단계를 건너뛸까요? 1) 이 레포만 자동 2) 모든 레포 자동 3) 매번 확인" 질문, 응답에 따라 agent가 config 자동 갱신
    - 수정 요청 시 노트 재작성 후 5.5단계 재진입(승인 떨어질 때까지 루프)
  - fix 4.5단계 신규: deploy 5.5와 동일 구조
- **`skills/config.json.example`**
  - `github` 섹션과 `github.repos[]` 항목에 자동/수동 옵션 예시 추가
- **`skills/references/config-rules.md` §7 github 섹션**
  - 글로벌·레포별 옵션 필드 표 추가, 해석 우선순위 명시
  - 옵션 값은 사용자가 직접 편집하기보다 changelog-deploy 스킬이 자동 갱신하도록 설계됐음을 명시
