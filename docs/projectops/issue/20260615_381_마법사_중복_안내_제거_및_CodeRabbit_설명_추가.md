📝 현재 문제점
---

- `template_integrator.ps1` / `.sh` 대화형 마법사의 여러 확인 단계에서, 안내 박스 안에 `Y/y - 예 ... / N/n - 아니오 ...` 두 줄을 출력한 뒤 바로 아래에서 화살표 메뉴(`1) 예 / 2) 아니오`)로 동일한 선택지를 다시 보여준다. 같은 선택지가 두 번 노출되어 화면이 중복되고 혼란스럽다.
- `version.yml 업데이트` 안내 박스에도 `Y 업데이트하고 계속 / N 통합 취소` 두 줄이 화살표 메뉴와 중복으로 들어가 있다.
- `.coderabbit.yaml`을 적용할 때 CodeRabbit이 무엇이며 어떤 설정이 들어가는지에 대한 안내가 전혀 없다. 그 결과 파일만 받아두고 실제로 CodeRabbit 서비스에서 저장소를 활성화하지 않아 "리뷰가 안 달린다"고 겪는 사용자가 많다.

🛠️ 해결 방안 / 제안 기능
---

- 중복 `Y/N` 안내 두 줄을 제거하고, 선택지는 화살표 메뉴(`Ask-YesNo`/`ask_yes_no`) 한 곳에서만 보여준다. 맥락이 사라지는 프롬프트("선택")는 실제 질문 문구로 교체한다.
- `version.yml 업데이트` 안내 박스에서도 동일한 중복 두 줄을 제거한다.
- `.coderabbit.yaml` 적용 직전에 CodeRabbit 소개 + 실제 적용되는 설정 요약(리뷰 언어 ko-KR, main 대상 자동 리뷰, chill 성향, PR 채팅 자동응답) + "https://coderabbit.ai 에서 저장소를 한 번 활성화해야 동작한다"는 안내를 출력한다.
- 위 변경은 `.sh`와 `.ps1` 양쪽에 대칭으로 적용하고, `bash -n` / PowerShell 파서로 문법을 검증한다.
