📝 현재 문제점
---

`template_integrator.ps1`(PowerShell 마법사)에는 **첫 화면 배너 + 현재 상태를 파싱해 보여주는 5개 층**의 UI가 있으나, Node로 포팅한 `npx projectops` CLI에는 이 중 프로젝트 분석 개요의 축소판만 있고 나머지는 없거나 빈약합니다.

- 현재 npx 첫 화면: `┌  projectops — 대화형 통합 마법사` (한 줄)
- `.ps1` 첫 화면: 박스 배너 + Version/Author/Mode/Repo + 감지 로그 + 분석 카드 + IDE 상태 + Breaking Changes 박스

즉 npx CLI가 `.ps1`보다 **첫인상·정보량에서 후퇴**해 있습니다. 또한 배너 브랜딩이 옛 이름(`S U H · DEVOPS · TEMPLATE`) 그대로라 새 이름(`projectops`)과 어긋납니다.

🛠️ 해결 방안 / 제안 기능
---

`.ps1`의 첫 화면 + 상태 표시 **5개 층 전부**를 npx CLI로 포팅하되, `.ps1`보다 **더 세련되게**(정보량은 `.ps1` 이상 유지, 후퇴 금지) 만듭니다.

**포팅 대상 5개 층**
1. **시작 배너** — 브랜딩을 `P R O J E C T O P S`(자간 벌린 큰 타이틀) + Author: Cassiiopeia + Repo: github.com/Cassiiopeia/projectops 로 갱신
2. **감지 로그** — 프로젝트 타입·버전·브랜치 감지 진행 표시
3. **프로젝트 분석 개요 카드** — 타입/버전/브랜치/모드/옵션/모노레포 경로
4. **IDE Skills 현재 상태** — IDE별 설치/버전/업데이트 가능 여부 파싱 표시
5. **신규/업데이트 판별 + Breaking Changes 박스**

**확정 사항**
- 포팅 범위: 5개 층 전부
- 목표 수준: `.ps1`보다 더 멋지게 (후퇴 금지)
- 배너 브랜딩: `P R O J E C T O P S` 자간 타이틀, Author: Cassiiopeia

**미확정 (브레인스토밍 이어서 진행)**
- 배너 실제 디자인 시안 (박스 스타일·색·자간 폭) — 시각 mockup 비교 예정
- 분석 카드/IDE 상태 카드 레이아웃, 색 팔레트
- 비대화형(`--force`) 모드에서 배너 노출 범위
