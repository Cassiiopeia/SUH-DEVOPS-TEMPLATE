📝 현재 문제점
---

레포/플러그인 이름이 `SUH-DEVOPS-TEMPLATE`/`cassiiopeia` → `projectops`로 바뀌었지만(#459), IDE 어댑터의 설치 흐름(`npx projectops`)이 옛 이름의 설치본을 인지·정리하지 못했다.

- **claude**: 옛 `cassiiopeia@cassiiopeia-marketplace`가 `failed to load` 좀비 상태로 남고, 옛 `cassiiopeia-marketplace` 캐시가 신규와 공존. 사용자가 넘어가려면 `marketplace remove` → `add` → `install` 6단계를 수동으로 밟아야 했다.
- **cursor**: `cursor-skills-meta.json`의 name이 옛값(`cassiiopeia`)이거나 버전이 옛 버전이어도 재설치되지 않음.
- **config 유실 위험**: 스킬은 `~/.projectops/config/config.json` 한 곳만 읽는데, 옛 경로(`~/.suh-template`, `~/.cassiiopeia`)에 PAT 등이 남아있어도 못 읽어 사용자가 재등록해야 함.
- **cursor remove 버그**: 기존 `remove()`가 `~/.cursor/skills` 폴더를 통째로 삭제 → 같은 폴더를 쓰는 다른 출처 스킬(somansa-tools 등)까지 함께 삭제되는 문제.

🛠️ 해결 방안 / 제안 기능
---

`npx projectops`(각 IDE 어댑터 `apply()`) 실행 시 옛 이름 설치본을 자동 감지·정리·이주한다. 조용히 자동 실행하고 로그만 남긴다.

- **공용 헬퍼 `src/core/ide/legacy.js` 추가**: `isLegacyVersion`(버전 기준점 판정), `migrateConfigRoot`(옛 config 루트 → `~/.projectops`로 무손실 이관, 신규가 비었을 때만 복사·옛 파일 보존), `hasNonEmptyJson`.
- **어댑터 5개에 `migrateLegacy` 추가** 후 `apply()` 시작 시 호출:
  - claude: 옛 `cassiiopeia` 플러그인 uninstall + `cassiiopeia-marketplace` remove
  - codex: 옛 `SUH-DEVOPS-TEMPLATE` native skills + marketplace 정리
  - gemini: 옛 extension uninstall
  - pi: 신·구 clone 공존 시 옛 `SUH-DEVOPS-TEMPLATE` clone + harness loader 정리
  - cursor: 옛 이름/버전(≤4.2.4) meta 감지 시 projectops 소유(`pro-*`/`suh-*`/meta)만 선별 삭제 후 재설치
- **cursor remove 버그 동반 수정**: 폴더 통째 삭제 대신 projectops 소유 항목만 선별 삭제 → 다른 출처 스킬 보존.
- 감지 기준은 이름 OR 버전 하이브리드. 정리 명령은 모두 실패 무해(없으면 no-op), `migrateLegacy`/`migrateConfigRoot`는 idempotent.
