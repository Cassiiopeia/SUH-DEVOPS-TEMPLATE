# 🚀[기능개선][통합마법사] 마법사 출력 가독성 및 옵션 승계 UX 개선 3건

> 라벨: 작업전
> 담당자: Cassiiopeia

🗒️ 설명
---

suh-project-utility 실물 통합 QA(2026-07-12, `docs/qa/2026-07-12-suh-project-utility-wizard-qa.md`)에서 수집한 마법사 UX 개선 3건입니다. 기능 오동작은 아니지만 사용자 체감 품질을 떨어뜨리는 항목들입니다.

**1) breaking-changes 배너 가독성 저하**

- 구버전→현재 사이의 CRITICAL/WARNING **전문이 통째로 덤프**되어 벽글이 됩니다 (v2.7.7→v4.2.8 실측 시 5개 항목 수천 자).
- 긴 본문이 터미널 폭에서 래핑되며 `║` 박스 경계가 붕괴되고, 좁은 터미널에서는 한글이 깨져 보이는 지점도 있습니다.
- 사용자가 읽기를 포기하게 되어 정작 CRITICAL 안내가 전달되지 않습니다.

**2) 완료 요약 "새로 설치됨" 목록이 실제 설치 내역과 무관**

- `새로 설치됨 (0개):`라고 표시하면서 파일 9개를 나열하는 등 카운터와 목록이 불일치합니다.
- 목록이 대상 레포 workflows 디렉토리 스캔(존재하는 PROJECT-* 전부) 기반이라, 방금 confirm 안내를 받은 **레거시 파일**(예: PROJECT-COMMON-SYNOLOGY-SECRET-FILE-UPLOAD.yaml)까지 "새로 설치됨"으로 표기됩니다.

**3) 구 `options.synology: true` 신 축 미승계**

- 구 키 nexus/npm_publish는 통합 시 신 축(deploy/publish)으로 자동 변환되지만, `synology: true`는 승계 규칙이 없어 비대화형 업데이트 시 `secret_backup: false`로 조용히 확정됩니다.
- 구 Synology 옵션 사용 레포가 업데이트 후 Secret 백업 워크플로우를 잃어버릴 수 있습니다.

🛠️ 해결 방안
---

1. breaking-changes 배너: 버전·제목·1줄 요약만 리스트로 출력하고, 전문은 대화형에서 "자세히 보기" 선택 시 표시(비대화형은 문서 링크 안내). 박스 폭은 터미널 폭 기반 래핑으로 전환.
2. 완료 요약: 복사 엔진이 반환하는 실제 복사 파일 목록을 그대로 출력하고 디렉토리 스캔 방식 제거. 카운터와 목록의 소스를 일치시킴.
3. 구 synology 키: 통합/업데이트 시 `synology: true` → `secret_backup: true` 승계 규칙 추가 (nexus/npm_publish 자동 변환과 동일 패턴).
