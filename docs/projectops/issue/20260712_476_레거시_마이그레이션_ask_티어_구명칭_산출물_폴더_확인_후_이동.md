# ⚙️[기능추가][통합마법사] 레거시 마이그레이션 ask 티어 추가 — 구명칭 산출물 폴더 확인 후 이동

> 라벨: 작업전
> 담당자: Cassiiopeia

🗒️ 설명
---

- 리브랜딩으로 스킬 산출물 폴더가 `docs/suh-template/` → `docs/projectops/`로 개명되었지만, 레거시 마이그레이션 레지스트리(#470)에는 이런 **폴더 리네임**을 다룰 항목/카테고리가 없습니다.
- 기존 통합 레포에는 구명칭 폴더에 이슈·보고서 문서가 쌓여 있어, 신 스킬이 새 경로에 저장하기 시작하면 산출물이 두 폴더로 갈라집니다.
- 이 폴더는 템플릿 소유 파일이 아니라 **사용자가 작성한 문서**이므로, 기존 safe 티어(자동 .bak)나 confirm 티어(안내만)와 성격이 다릅니다 — **사용자에게 물어보고 손실 없는 이동**을 수행하는 제3 티어가 필요합니다.

🛠️ 해결 방안
---

- 레지스트리에 **`ask` 티어 + `legacy-dir` 카테고리** 추가:
  - 대화형: 감지 시 "폴더를 새 이름으로 이동할까요? (파일 손실 없음)" 확인 후 이동
  - 비대화형(--force): 사용자 콘텐츠이므로 자동 조치 없이 안내만 (confirm과 동일 정책)
- 이동 규칙: 대상 폴더가 없으면 통째 rename, 이미 있으면 재귀 병합(충돌 파일은 남기고 건수 보고 — 사용자 판단 존중)
- 첫 항목: `docs/suh-template/` → `docs/projectops/` (since 4.2.9 리브랜딩)
- 레지스트리 스키마 주석·검증 테스트(`test/migrations.test.js`)의 tier/category 허용 목록 확장
