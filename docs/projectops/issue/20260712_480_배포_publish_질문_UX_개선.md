# 🚀[기능개선][통합마법사] 배포/publish 질문 UX 개선 — deploy·publish 두 축을 먼저 갈라 묻기

> 라벨: 작업전
> 담당자: Cassiiopeia

🗒️ 설명
---

Spring 프로젝트 통합 중 배포 질문 단계에서 사용자가 혼란을 겪었습니다 (실측: suh-project-utility, v4.2.12).

현재 마법사는 **deploy(서버 실행물, 택1)와 publish(라이브러리 레지스트리, 다중)** 를 코드상 이미 분리(#439)해 두 번 순차로 묻지만, 두 질문이 서로를 모르는 채 나와 다음 문제가 생깁니다:

1. **첫 질문("어디에 배포하나요?")이 deploy만 툭 던진다.** Spring 사용자는 "라이브러리(Nexus·Maven Central)는 여기서 고르나?"라고 기대하지만 여기선 못 고른다. 라이브러리 배포 선택지(publish)는 뒤에 따로 나오는데, 지금 화면에서는 그게 있는지 알 수 없다.
2. **"배포하지 않음 (라이브러리/CI 전용)" 선택지의 괄호가 바로 다음 질문(publish)에서 다룰 내용을 미리 언급**해 혼란을 키운다. 여기서 "라이브러리"를 고르는 게 아닌데 라이브러리라는 단어가 이 선택지에 붙어 있다.
3. **"CI 전용"이라는 표현이 불친절**하다. 일반 사용자가 못 알아들을 수 있다.
4. 두 축이 **하나의 큰 그림으로 먼저 제시되지 않아**, 사용자가 "내 프로젝트가 서버 앱인지 / 남이 갖다 쓰는 라이브러리인지 / 둘 다인지"를 판단할 맥락이 부족하다.

📸 참고 자료 (현재 화면)
---

```
🚀 이 프로젝트를 어디에 배포하나요?
   서버·호스팅에 올릴 계획이 있으면 고르고, 지금 없으면 '배포 안 함'으로 두면 됩니다.
◆  배포 방식을 선택하세요
│  ● Docker + SSH 서버 배포 (기본)
│  ○ Vercel
│  ○ 배포하지 않음 (라이브러리/CI 전용)   ← '라이브러리'가 여기 붙어 혼란
```

→ 바로 다음 질문에서 publish(Nexus/npm/GitHub Packages)를 또 묻는다.

🛠️ 해결 방안
---

- **두 축을 먼저 한 화면에서 갈라 안내한 뒤 질문**한다. 예: "이 프로젝트는 (A) 서버에 올려 돌리는 실행물인가요, (B) 남이 가져다 쓰는 라이브러리인가요, (C) 둘 다인가요?"를 맥락으로 제시.
- **deploy 선택지 문구에서 publish 개념(라이브러리)을 제거**한다. "배포하지 않음" 라벨을 "서버에 올리지 않음 (라이브러리만 배포하거나, 빌드 검증만)" 등 이 질문의 범위에 맞게 정정.
- **"CI 전용" 표현 제거** — "빌드 검증만" 같은 평이한 말로 대체.
- publish 질문의 유도 문구도 deploy와 대칭이 되게 다듬어, 두 질문이 하나의 그림(deploy 축 + publish 축)임을 사용자가 인지하게 한다.
- 동작(선택 결과 저장·버전.yml 축)은 그대로 두고 **질문의 문구·순서·맥락 제시만** 개선한다.

✅ 기대 동작
---

- Spring 사용자가 "서버 앱 배포 + 라이브러리 배포"를 각각 독립적으로 이해하고, 서버 배포만 하고 publish는 비워두는 흐름이 자연스럽게 느껴진다.
- deploy 선택지에서 라이브러리/CI 같은 다른 축·전문 용어가 사라진다.
