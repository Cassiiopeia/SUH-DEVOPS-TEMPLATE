📝 현재 문제점
---

- 마법사가 통합 도중 개발(릴리스 소스) 브랜치가 없으면 **직접 생성하고 origin push까지 완료**해줍니다 (#477 ensureDeployBranch).
- 그런데 완료 요약의 "다음 3가지 작업" 안내는 이 사실을 모른 채 여전히 `2️⃣ develop 브랜치 생성 (아직 없다면) → git checkout -b develop && git push -u origin develop` 을 지시합니다.
- 사용자가 방금 마법사 화면에서 "✅ origin/develop push 완료"를 봤는데 곧바로 "브랜치를 만들어라"는 안내를 받으면 혼란스럽고, 안내대로 따라 하면 이미 있는 브랜치에 불필요한 명령을 실행하게 됩니다.

- v4.2.15 실측 (suh-project-utility 통합 로그): 마법사가 develop 생성 + push 성공 → 완료 요약이 같은 브랜치 생성을 재지시.

🛠️ 해결 방안 / 제안 기능
---

- 마법사가 이번 실행에서 개발 브랜치를 직접 생성(또는 push)했으면, 완료 요약의 해당 항목을 "✅ 완료됨" 표시로 바꾸거나 항목 자체를 생략한다.
- 브랜치가 이미 존재해서 생성이 필요 없던 경우에도 동일하게 처리한다 (원격 존재 확인 결과를 요약에 전달).
- 생성하지 않았고 존재도 확인 못 한 경우에만 기존 안내를 유지한다.
