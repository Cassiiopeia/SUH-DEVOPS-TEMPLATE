📝 현재 문제점
---

- 기존 통합 레포에서 `npx projectops@latest` 실행 시 "업데이트 v4.2.19 → v4.2.21"을 감지해 표시하면서도, 메뉴는 신규 설치용 5개(전체 설치/버전 관리만/워크플로우만/이슈·PR 템플릿만/AI 스킬만)뿐입니다.
- 업데이트하려는 사용자는 "전체 설치 (처음이라면 추천)"를 골라야 해서 혼란스럽고, 이후 env 계획 질문과 워크플로우 충돌 3지선다를 매번 통과해야 합니다.
- 충돌 3지선다의 기본값이 "건너뛰기"라서 엔터만 치면 갱신된 워크플로우가 실제로 반영되지 않는 함정이 있습니다.
- version.yml에 이전 통합의 범위(mode)가 기록되지 않아, 업데이트가 무엇을 재실행해야 하는지 알 수 없습니다.

🛠️ 해결 방안 / 제안 기능
---

- 업데이트 감지 시 메뉴 맨 위에 **"업데이트 — 저장된 설정 그대로 v{이전} → v{현재} 반영 (추천)"** 항목을 기본 선택으로 추가합니다.
- 업데이트 모드는 저장된 통합 범위(mode, 없으면 full)를 재실행하되 질문을 최소화합니다:
  - 분석 카드는 표시만 하고 확인 루프 생략, env 계획 질문 생략 (저장값과 기본값 사용)
  - 워크플로우 충돌은 일괄 확인 1회(기본 Y) 후 전부 .bak 백업 + 새 버전 교체
  - 교체 시 기존 설치본의 env 실값(FIREBASE_APP_ID 등)을 추출해 새 파일에 이월(carryover) — 사용자가 채운 값 보존
  - 마이그레이션/고아 정리 확인은 기존대로 유지 (안전망)
  - AI 스킬은 설치된 IDE만 질문 없이 자동 업데이트
- version.yml template 블록에 `mode`를 기록해 다음 업데이트부터 같은 범위를 재실행합니다.
