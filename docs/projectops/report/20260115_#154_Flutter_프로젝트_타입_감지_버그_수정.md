## 📌 작업 개요

Flutter 프로젝트에서 템플릿 통합 스크립트 실행 시 프로젝트 타입을 Node.js로 잘못 감지하는 버그 수정. `package.json` 파일 체크 순서를 변경하여 명확한 프레임워크 마커를 우선 감지하도록 개선.

---

## 🔍 문제 분석

### 버그 재현 상황

사용자의 Flutter 프로젝트(RomRom-FE)에서 템플릿 통합 스크립트 실행 시:

```
🔅 프로젝트 타입 자동 감지 중...
  🔸 감지됨: Node.js          ❌ 잘못된 감지
🔅 버전 정보 자동 감지 중...
  🔸 pubspec.yaml에서 발견: v1.8.22   ✅ Flutter 파일 존재함에도 불구
```

- `pubspec.yaml` 존재 (Flutter 프로젝트 마커)
- `package.json` 존재 (개발 도구용: Prettier, ESLint 등)
- 프로젝트 타입이 `node`로 잘못 감지됨
- 버전은 `pubspec.yaml`에서 정상 추출

### 근본 원인

**프로젝트 타입 감지 함수의 체크 순서 문제**

기존 로직:
```
1. package.json 체크 → 존재하면 Node.js로 감지하고 즉시 반환 ❌
2. build.gradle 체크 → Spring Boot
3. pubspec.yaml 체크 → Flutter (도달하지 못함!)
4. pyproject.toml 체크 → Python
```

**문제점**:
- Flutter 프로젝트에서 `package.json`을 개발 도구용으로 사용하는 경우가 많음
  - Flutter 웹 빌드 도구
  - 코드 포맷팅 도구 (Prettier, ESLint)
  - npm scripts로 Flutter 명령어 래핑
  - 하이브리드 프로젝트 구성
- `package.json` 체크가 먼저 실행되어 Flutter 감지 코드에 도달 불가
- Spring Boot 프로젝트도 동일한 문제 발생 가능

## ✅ 구현 내용

### 1. 프로젝트 타입 감지 우선순위 재설계

**새로운 우선순위 전략**:

```
우선순위 1: 명확한 프레임워크 마커
├─ pubspec.yaml → Flutter
├─ build.gradle/pom.xml → Spring Boot
└─ pyproject.toml → Python

우선순위 2: Node.js 에코시스템 세부 분류
└─ package.json 내용 분석
    ├─ @react-native → React Native
    ├─ "next" → Next.js
    ├─ "react" → React
    └─ 기타 → Node.js
```

**근거**:
- Flutter, Spring Boot, Python은 **고유한 마커 파일**을 가짐
- Node.js 에코시스템은 모두 `package.json`을 공유하므로 세부 분류 필요
- `package.json`은 다른 프레임워크에서 **보조 도구용**으로 사용될 수 있음

### 2. template_integrator.ps1 수정 (Windows)

**파일**: `template_integrator.ps1`
**함수**: `Detect-ProjectType` (라인 381-450)

**주요 변경사항**:

```powershell
function Detect-ProjectType {
    Print-Step "프로젝트 타입 자동 감지 중..."

    # ===================================================
    # 우선순위 1: 명확한 프레임워크 마커 파일 체크
    # Flutter, Spring, Python은 고유한 마커 파일을 가지므로 우선 체크
    # ===================================================

    # Flutter (최우선)
    if (Test-Path "pubspec.yaml") {
        Print-Info "감지됨: Flutter"
        return "flutter"
    }

    # Spring Boot
    if ((Test-Path "build.gradle") -or (Test-Path "build.gradle.kts") -or (Test-Path "pom.xml")) {
        Print-Info "감지됨: Spring Boot"
        return "spring"
    }

    # Python
    if ((Test-Path "pyproject.toml") -or (Test-Path "setup.py") -or (Test-Path "requirements.txt")) {
        Print-Info "감지됨: Python"
        return "python"
    }

    # ===================================================
    # 우선순위 2: Node.js 에코시스템 세부 분류
    # package.json은 여러 프로젝트 타입에서 보조 도구로 사용될 수 있으므로 나중에 체크
    # ===================================================

    # Node.js / React / React Native / Next.js
    if (Test-Path "package.json") {
        $packageJson = Get-Content "package.json" -Raw
        # React Native, Next.js, React, Node.js 순서대로 체크
        # ...
    }
}
```

**변경 효과**:
- Flutter 체크를 최상단으로 이동 → `package.json` 체크 전에 실행
- 명확한 주석으로 우선순위 구분 (`=====` 구분선)
- 기존 Node.js 세부 분류 로직 모두 보존 (하위 호환성)

### 3. template_integrator.sh 수정 (Linux/macOS)

**파일**: `template_integrator.sh`
**함수**: `detect_project_type` (라인 453-527)

**주요 변경사항**:

```bash
detect_project_type() {
    print_step "프로젝트 타입 자동 감지 중..."

    # ===================================================
    # 우선순위 1: 명확한 프레임워크 마커 파일 체크
    # ===================================================

    # Flutter (최우선)
    if [ -f "pubspec.yaml" ]; then
        print_info "감지됨: Flutter"
        echo "flutter"
        return
    fi

    # Spring Boot
    if [ -f "build.gradle" ] || [ -f "build.gradle.kts" ] || [ -f "pom.xml" ]; then
        print_info "감지됨: Spring Boot"
        echo "spring"
        return
    fi

    # Python
    if [ -f "pyproject.toml" ] || [ -f "setup.py" ] || [ -f "requirements.txt" ]; then
        print_info "감지됨: Python"
        echo "python"
        return
    fi

    # ===================================================
    # 우선순위 2: Node.js 에코시스템 세부 분류
    # ===================================================

    if [ -f "package.json" ]; then
        # Next.js 체크 (새로 추가됨!)
        if grep -q "\"next\"" package.json; then
            print_info "감지됨: Next.js"
            echo "next"
            return
        fi
        # React Native, React, Node.js 순서대로 체크
        # ...
    fi
}
```

**추가 개선사항**:
- PowerShell 버전과 로직 동기화
- **Next.js 감지 로직 추가** (기존 Bash 버전에 누락되어 있었음)
- 동일한 우선순위 전략 적용
