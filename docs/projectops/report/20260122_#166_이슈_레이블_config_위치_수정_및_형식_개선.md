## 📌 작업 개요

기존 이슈 레이블에 띄어쓰기가 포함되어 레이블 연동 시 오류 발생 문제 해결. 레이블 설정 파일을 `.github/ISSUE_TEMPLATE/`에서 `.github/config/`로 이동하여 구조 개선. 레이블명 띄어쓰기 제거, 신규 레이블 추가, 중복 레이블 통합 완료.

---

## 🔍 문제 분석

### 기존 문제점
1. **레이블 띄어쓰기 문제**: `작업 전`, `작업 중`, `확인 대기` 등 띄어쓰기 포함 → 연동 시 오류 발생
2. **파일 위치 부적절**: `.github/ISSUE_TEMPLATE/issue-label.yml` → 템플릿 폴더에 설정 파일 위치 혼란
3. **중복 레이블**: `취소` 레이블 2개 존재
4. **라벨 부족**: 보류 상태 라벨 없음

### 해결 방안
- 레이블명 띄어쓰기 제거 (붙여쓰기)
- 파일 위치 이동: `.github/config/issue-labels.yml`
- 라벨명 변경: `확인 대기` → `담당자확인`
- 신규 라벨 추가: `보류` (회색 #9E9E9E)
- 중복 라벨 통합: `취소` 1개로 통합 (color: f28b25)

## 📌 영향 범위 및 호환성

### 영향받는 시스템
1. **GitHub Actions**: 워크플로우 6개 (라벨 트리거 조건)

- GitHub Projects Board의 Status 컬럼 이름도 동일하게 변경 필요
- `crazy-max/ghaction-github-labeler`는 기존 라벨 삭제 후 새 라벨 생성
