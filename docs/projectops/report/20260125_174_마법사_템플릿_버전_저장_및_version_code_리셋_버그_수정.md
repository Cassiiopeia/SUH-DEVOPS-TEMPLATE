# 마법사 템플릿 버전 저장 및 version_code 리셋 버그 수정

**이슈**: [#174](https://github.com/Cassiiopeia/SUH-DEVOPS-TEMPLATE/issues/174)

---

### 📌 작업 개요

template_integrator 스크립트에서 발생한 두 가지 버그 수정:
1. 템플릿 버전이 `metadata.template.version`에 저장되지 않는 문제
2. macOS에서 `grep -P` 미지원으로 인해 기존 `version_code`가 1로 리셋되는 문제

---

### 🔍 문제 분석

#### 문제 1: 템플릿 버전 저장 안됨

**원인**: `save_template_options` 함수 호출 시 잘못된 버전 전달
- Bash: `DEFAULT_VERSION` (1.3.14) 전달 → 실제 템플릿 버전(2.7.5)이 아님
- PowerShell: `ProjectVersion` (사용자 프로젝트 버전) 전달 → 템플릿 버전이 아님

**영향**: Breaking Changes 경고 기능이 작동하지 않음. 이 기능은 `metadata.template.version`을 읽어서 사용자에게 중요 변경사항을 알려주는데, 버전이 저장되지 않으면 경고가 표시되지 않음.

#### 문제 2: version_code 리셋 (macOS)

**원인**: `grep -P` (Perl regex) 옵션이 macOS BSD grep에서 미지원
```
grep: invalid option -- P
usage: grep [-abcdDEFGHhIiJLlMmnOopqRSsUVvwXxZz] ...
```

**영향**: 기존 `version_code: 46`이 `1`로 리셋됨. 앱 스토어 배포 시 빌드 번호가 낮아지면 업로드 거부됨.

### ✅ 구현 내용

#### 1. macOS grep -P 호환성 수정

- **파일**: `template_integrator.sh`
- **위치**: 라인 960-961
- **변경 내용**: `grep -oP '\d+'` → `sed 's/[^0-9]//g'` 변경

**Before**:
```bash
existing_version_code=$(grep -E '^version_code:\s*[0-9]+' version.yml | grep -oP '\d+' | head -1)
```

**After**:
```bash
existing_version_code=$(grep -E '^version_code:\s*[0-9]+' version.yml | sed 's/[^0-9]//g' | head -1)
```

**이유**: `sed 's/[^0-9]//g'`는 POSIX 호환으로 BSD grep과 GNU grep 모두에서 동작함.

#### 2. TEMPLATE_VERSION 전역 변수 추가 (Bash)

- **파일**: `template_integrator.sh`
- **위치**: 라인 123-124
- **변경 내용**: 다운로드한 템플릿의 실제 버전을 저장할 전역 변수 추가

```bash
# 다운로드한 템플릿의 실제 버전 (download_template에서 설정됨)
TEMPLATE_VERSION=""
```

---

#### 3. download_template에서 템플릿 버전 읽기 (Bash)

- **파일**: `template_integrator.sh`
- **위치**: 라인 890-899
- **변경 내용**: 템플릿 다운로드 후 version.yml에서 버전 추출

```bash
# 다운로드한 템플릿에서 버전 읽기 (TEMPLATE_VERSION 전역 변수에 저장)
if [ -f "$TEMP_DIR/version.yml" ]; then
    TEMPLATE_VERSION=$(grep -E '^version:\s*' "$TEMP_DIR/version.yml" | sed 's/version:[[:space:]]*["'\'']*\([^"'\'']*\)["'\'']*$/\1/' | head -1)
    if [ -z "$TEMPLATE_VERSION" ]; then
        TEMPLATE_VERSION="$DEFAULT_VERSION"
    fi
else
    TEMPLATE_VERSION="$DEFAULT_VERSION"
fi
```

#### 4. save_template_options에 올바른 버전 전달 (Bash)

- **파일**: `template_integrator.sh`
- **위치**: 라인 2428-2429
- **변경 내용**: `DEFAULT_VERSION` → `TEMPLATE_VERSION` 변경

**Before**:


**After**:

#### 6. Download-Template에서 템플릿 버전 읽기 (PowerShell)

- **파일**: `template_integrator.ps1`
- **위치**: 라인 761-772
- **변경 내용**: 템플릿 다운로드 후 version.yml에서 버전 추출

```powershell
# 다운로드한 템플릿에서 버전 읽기 (TemplateVersion 전역 변수에 저장)
$templateVersionFile = Join-Path $TEMP_DIR "version.yml"
if (Test-Path $templateVersionFile) {
    $versionContent = Get-Content $templateVersionFile -Raw -ErrorAction SilentlyContinue
    if ($versionContent -match 'version:\s*[''"]?([0-9.]+)') {
        $script:TemplateVersion = $matches[1]
    } else {
        $script:TemplateVersion = $DEFAULT_VERSION
    }
} else {
    $script:TemplateVersion = $DEFAULT_VERSION
}
```

### 🧪 테스트 및 검증

#### macOS에서 version_code 추출 테스트
```bash
# 레거시 version.yml로 테스트
echo "version_code: 46" | grep -E '^version_code:\s*[0-9]+' | sed 's/[^0-9]//g'
# 결과: 46 (정상)
```

#### 템플릿 버전 저장 확인
통합 실행 후 version.yml 확인:
```yaml
metadata:
  template:
    version: "2.7.5"  # ← 현재 템플릿 버전이 저장됨
```

---

### 📌 참고사항

- **버전 추출 방식**: 다양한 인용 형식 (`"2.7.5"`, `'2.7.5'`, `2.7.5`) 모두 처리
- **Fallback**: 버전 추출 실패 시 `DEFAULT_VERSION` 사용
- **Breaking Changes 경고**: 이제 `metadata.template.version`을 기반으로 정상 작동함
