### 📌 작업 개요
`/issue` custom command 신규 추가. 사용자가 대략적으로 설명하면 이슈 타입을 자동 판단하여 GitHub 이슈 템플릿에 맞는 제목과 본문을 자동 작성해주는 기능 구현

### 🎯 구현 목표
- 버그 / 기능요청 / 기능추가 / 기능개선 / 디자인 / QA 이슈를 대략적인 설명만으로 자동 작성
- 기존 이슈 템플릿(`.github/ISSUE_TEMPLATE/`) 구조를 그대로 활용
- 기존 이슈 제목 패턴(이모지 + 카테고리 태그) 준수

### ✅ 구현 내용

#### `/issue` custom command 파일 생성
- **파일**: `.claude/commands/issue.md`, `.cursor/commands/issue.md`
- **변경 내용**: 신규 파일 추가 (양쪽 동일한 내용)
- **이유**: Claude Code와 Cursor IDE 양쪽에서 동일하게 `/issue` 명령어 사용 가능하도록

### 🔧 주요 변경사항 상세

#### issue.md command 구조

**동작 프로세스 (3단계)**:
1. 사용자 입력 분석 → 이슈 타입 자동 판단 (버그/기능/디자인/QA)
2. 부족한 정보가 있으면 짧게 질문, 명확한 건 알아서 판단
3. 해당 템플릿 구조에 맞춰 제목 + 본문 출력

**지원 이슈 타입 및 제목 형식**:
| 이모지 | 타입 | 예시 |
|--------|------|------|
| ❗ | 버그 | `❗ [버그][로그인] 로그인 시 앱 강제 종료` |
| ⚙️ | 기능추가 | `⚙️ [기능추가][CustomCommand] 이슈 자동 생성` |
| 🚀 | 기능개선 | `🚀 [기능개선][Label] 이슈 레이블 형식 개선` |
| 🔧 | 기능요청 | `🔧 [기능요청][Flutter] OAuth 마법사 구현` |
| 🎨 | 디자인 | `🎨 [디자인][프로필] 프로필 페이지 UI 개선` |
| 🔍 | QA/시험 | `🔍 [시험요청][로그인] 로그인 기능 시험` |

**주요 규칙**:
- 코드적인 내용 포함 금지 (기능/현상 중심으로만 작성)
- 긴급(🔥) 태그는 사용자가 직접 "긴급"이라고 말한 경우에만 추가
- 담당자는 항상 템플릿 기본값("이름") 유지
- 각 이슈 타입별로 기존 4가지 템플릿(bug_report, feature_request, design_request, qa_request) 구조를 그대로 사용

**특이사항**:
- 기능 타입은 추가/개선/요청으로 세분류하여 이모지와 태그를 다르게 적용
- 기능추가/기능개선의 경우 필요하면 `⚙️ 작업 내용` 섹션 추가 가능 (코드 내용이 아닌 작업 항목 수준)

### 🧪 테스트 및 검증
- `/issue 지금 issue custom command를 개발해야되는 이슈를 써줘` 명령어로 실행 테스트 완료
- 기능추가 타입으로 자동 판단하여 feature_request 템플릿에 맞게 제목 + 본문 정상 출력 확인
