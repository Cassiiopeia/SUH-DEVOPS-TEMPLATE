### 📌 작업 개요
`@suh-lab app build` 커맨드 사용 시 Android는 빌드 완료 후 즉시 완료 댓글이 달리지만, iOS는 TestFlight 배포까지 완료되어야 최종 댓글이 달리는 타이밍 불일치 문제 수정.
`build-ios-test` Job(Job 2) 성공 시 댓글 제목을 즉시 "✅ 완료" 상태로 표시하도록 변경.

**이슈**: [#198](https://github.com/Cassiiopeia/SUH-DEVOPS-TEMPLATE/issues/198)

### 🔍 문제 분석

**현재 댓글 표시 흐름 (수정 전)**:
```
Job 1 (prepare) 완료  →  진행 중... 댓글 생성 (즉시)
Job 2 (IPA 빌드) 완료  →  "iOS TestFlight 빌드 진행 중..." (여전히 진행 중 표시)
Job 3 (TestFlight 배포) 완료  →  "✅ iOS TestFlight 빌드 완료" (완료 표시)
```

**원인**:
- `build-ios-test` Job의 성공 댓글 제목이 `"iOS TestFlight 빌드 진행 중..."`으로 설정되어 있어, IPA 빌드가 완료됐음에도 "완료" 상태가 표시되지 않음
- 최종 `✅` 완료 표시가 `deploy-testflight-test` Job(TestFlight 배포, 약 5~10분 추가 소요)에서만 달려 Android 대비 현저히 늦게 보임

**Android와의 비교**:
- Android: Job 2(APK 빌드) 완료 즉시 `✅ 완료` 댓글 → 약 13~18분
- iOS(수정 전): Job 3(TestFlight 배포) 완료 후에야 `✅ 완료` 댓글 → 약 25~35분

### ✅ 구현 내용

#### Job 2 성공 댓글 제목 변경
- **파일**: `.github/workflows/project-types/flutter/PROJECT-FLUTTER-IOS-TEST-TESTFLIGHT.yaml` (line 570)
- **변경 내용**: `'## 🍎 iOS TestFlight 빌드 진행 중...'` → `'## 🍎 ✅ iOS 빌드 완료'`
- **이유**: IPA 빌드 완료 즉시 "완료" 상태를 표시하여 Android와 동일한 피드백 타이밍 제공

#### Job 3 성공 댓글 제목 명확화
- **파일**: `.github/workflows/project-types/flutter/PROJECT-FLUTTER-IOS-TEST-TESTFLIGHT.yaml` (line 813)
- **변경 내용**: `'## 🍎 ✅ iOS TestFlight 빌드 완료'` → `'## 🍎 ✅ iOS TestFlight 배포 완료'`
- **이유**: Job 3은 빌드가 아닌 TestFlight 배포를 담당하므로 "배포 완료"가 더 정확한 표현

#### Job 3 실패 댓글 제목 명확화
- **파일**: `.github/workflows/project-types/flutter/PROJECT-FLUTTER-IOS-TEST-TESTFLIGHT.yaml` (line 881)
- **변경 내용**: `'## 🍎 ❌ iOS TestFlight 빌드 실패'` → `'## 🍎 ❌ iOS TestFlight 배포 실패'`
- **이유**: Job 3 단계에서의 실패는 IPA 빌드 성공 이후 배포 단계 실패이므로 "배포 실패"가 정확한 표현

### 🔧 주요 변경사항 상세

#### 수정 후 댓글 표시 흐름
```
Job 1 완료  →  "iOS TestFlight 빌드 진행 중..." 댓글 생성 (즉시)
Job 2 완료  →  "✅ iOS 빌드 완료" + "📤 TestFlight 배포 ⏳ 진행 중..." (IPA 완료 즉시!)
Job 3 완료  →  "✅ iOS TestFlight 배포 완료" (최종 배포 결과)
```

#### 전체 댓글 제목 타이틀 현황
| 라인 | 시점 | 제목 |
|------|------|------|
| 154, 364 | Job 1 (준비 중) | `iOS TestFlight 빌드 진행 중...` (변경 없음) |
| 570 | Job 2 성공 (IPA 완료) | `✅ iOS 빌드 완료` (변경) |
| 629 | Job 2 실패 (IPA 실패) | `❌ iOS TestFlight 빌드 실패` (변경 없음) |
| 813 | Job 3 성공 (배포 완료) | `✅ iOS TestFlight 배포 완료` (변경) |
| 881 | Job 3 실패 (배포 실패) | `❌ iOS TestFlight 배포 실패` (변경) |

**특이사항**:
- Job 2 성공 댓글 내부에 `| 📤 TestFlight 배포 | ⏳ 진행 중... |` 행이 유지되어, 배포가 아직 진행 중임을 명확히 표시
- Job 3 실패 시 댓글 제목이 "❌ 배포 실패"로 덮어써지므로 이전 "✅ iOS 빌드 완료" 상태와의 혼동 없음
- `*이 댓글은 자동으로 업데이트됩니다.*` 문구로 라이브 상태 업데이트 안내 유지

### 🧪 테스트 및 검증
- `grep -n "## 🍎"` 로 모든 제목 라인 확인 완료
- Job 2 완료 후 즉시 ✅ 제목 표시 → Android와 동일한 UX 달성
- Job 3 실패 케이스에서 올바른 실패 제목 표시 확인

### 📌 참고사항
- `build-ios-test` Job 내 중간 진행 댓글(`준비 완료 후 IPA 빌드 중`)은 기존과 동일하게 유지
- `deploy-testflight-test` Job에서 댓글 최종 업데이트가 이루어지므로 TestFlight 배포 결과도 정상 반영됨
