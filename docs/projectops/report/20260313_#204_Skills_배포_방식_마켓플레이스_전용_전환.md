# Skills 배포 방식 변경 - 마켓플레이스 전용 전환

## 개요

Skills(IDE 명령어)의 배포 방식을 `template_integrator` 복사 방식에서 Claude Code 플러그인 마켓플레이스 전용으로 전환했다. 기존 `.claude/skills/`, `.claude/scripts/`, `.cursor/commands/`, `.cursor/scripts/` 를 삭제하고, 루트 `skills/`, `scripts/`로 이동하여 마켓플레이스에서 직접 설치하는 구조로 변경했다. 버전은 `2.8.6` → `2.9.0`으로 minor 업데이트.

## 변경 사항

### 플러그인 매니페스트 신규 추가
- `.claude-plugin/plugin.json`: 플러그인 메타 정보 (이름 `cassiiopeia`, 버전, 키워드 등)
- `.claude-plugin/marketplace.json`: 마켓플레이스 등록 정보 (저장소, 스킬 경로, 스크립트 경로)

### 버전 동기화 워크플로우 추가
- `.github/workflows/PROJECT-TEMPLATE-PLUGIN-VERSION-SYNC.yaml`: `version.yml` 변경 시 `plugin.json`, `marketplace.json`의 버전을 자동 동기화하는 워크플로우 (main push + workflow_dispatch 트리거)

### Skills 경로 이동
- `.claude/skills/` (20개 스킬 + references 6개) → 루트 `skills/`로 이동 (마켓플레이스 배포용)
- `.claude/skills/` → `.cursor/skills/`로도 이동 (Cursor IDE용)
- `.claude/scripts/` (README.md, worktree_manager.py) → 루트 `scripts/`로 이동

### 기존 Cursor commands/scripts 삭제 (~11,300줄)
- `.cursor/commands/` 19개 커맨드 파일 전체 삭제 (`.cursor/skills/`로 대체)
- `.cursor/scripts/` 3개 유틸리티 파일 삭제 (README.md, common_util.py, powershell_common_util.py, worktree_manager.py)

### template_integrator.sh 수정 (~370줄 변경)
- `--mode commands` 모드 제거, `--target` 옵션 제거
- `copy_cursor_folder()`, `copy_claude_folder()`, `copy_custom_commands()`, `show_custom_command_menu()`, `install_custom_command()` 함수 삭제
- `offer_ide_tools_install()` 함수 신규 추가: Claude Code CLI 감지 시 마켓플레이스 자동 설치 제안, Cursor CLI 감지 시 `skills/` → `.cursor/skills/` 복사 제안, CLI 미감지 시 수동 설치 안내
- 다운로드 시 `.claude-plugin/`, `scripts/` 플러그인 전용 파일 자동 제외
- 대화형 메뉴 선택지 6개 → 5개로 축소

### template_integrator.ps1 수정 (~362줄 변경)
- `template_integrator.sh`와 동일한 변경사항을 PowerShell 버전에 적용

### template_initializer.sh 수정
- `cleanup_template_files()` 함수에 플러그인 전용 파일/폴더 삭제 로직 추가: `.claude-plugin/`, `skills/`, `scripts/`

### CLAUDE.md 문서 업데이트
- 폴더 구조도에 `.claude-plugin/`, `skills/`, `scripts/` 추가
- "IDE 명령어" 섹션 → "Skills (Claude Code 플러그인)" 섹션으로 변경
- 플러그인 설치 방법, 사용법 (`/cassiiopeia:명령어`) 문서화
- 초기화/통합 시 제외 파일 목록 업데이트
- Breaking Changes 관리 항목에 Skills/플러그인 배포 방식 변경 추가

### docs/TEMPLATE-INTEGRATOR.md 업데이트
- commands 모드 관련 내용 제거, IDE 도구 설치 안내로 교체

### breaking-changes.json 등록
- `2.9.0` 버전에 severity `critical` 등록: 기존 `.claude/skills/` 사용자 대상 마켓플레이스 재설치 안내

### 버전 업데이트
- `version.yml`: `2.8.6` → `2.9.0`

## 주요 구현 내용

`offer_ide_tools_install()` 함수가 핵심 변경이다. 통합 완료 후 사용자 환경의 CLI를 자동 감지하여:
1. **Claude Code CLI 존재** → `claude plugin marketplace add` + `claude plugin install --scope user` 실행 제안
2. **Cursor CLI 존재** → 템플릿의 `skills/` 폴더를 `.cursor/skills/`로 복사 제안
3. **둘 다 없음** → 수동 설치 명령어 안내 출력

`skills/` 폴더는 다운로드 시 삭제하지 않고 보존하여 Cursor 복사용으로 활용한 후, 최종 임시 폴더 정리 시 함께 삭제된다.

## 주의사항

- 기존 `.claude/skills/`, `.claude/scripts/` 경로를 직접 참조하던 프로젝트는 마켓플레이스로 재설치 필요
- `--mode commands` 또는 `--target` 옵션을 사용하던 CI/스크립트는 해당 옵션 제거 필요
- `version.yml`이 변경될 때마다 `PROJECT-TEMPLATE-PLUGIN-VERSION-SYNC` 워크플로우가 자동 실행되어 매니페스트 버전을 동기화함
