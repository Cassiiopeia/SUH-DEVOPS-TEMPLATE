# ❗[버그][Flutter] macos-26 러너에서 iOS 26.0 빌드 실패되는 현상

## 개요

`macos-26` GitHub Actions 러너에서 `xcode-select`로 Xcode 26.0을 지정해도 `DEVELOPER_DIR` 환경변수가 서브프로세스에 전파되지 않아 `iOS 26.0 is not installed` 오류가 발생하던 문제를 수정했다. `flutter build ios`, `xcodebuild` 스텝에 `DEVELOPER_DIR`을 명시적으로 선언하고, Xcode 26.0 iOS 플랫폼 설치 스텝을 추가하여 해결했다.

## 변경 사항

### Flutter iOS TestFlight 워크플로우

- `PROJECT-FLUTTER-IOS-TESTFLIGHT.yaml`: `Select Xcode version` 스텝 이후 `Install iOS device platform` 스텝 추가, `Flutter build (no codesign)` 및 `Create Archive` 스텝에 `DEVELOPER_DIR` 환경변수 명시
- `PROJECT-FLUTTER-IOS-TEST-TESTFLIGHT.yaml`: 동일 변경 적용 (테스트 빌드 워크플로우)

## 주요 구현 내용

### 원인

`sudo xcode-select -s`는 글로벌 심볼릭 링크만 변경하며, 현재 셸의 `DEVELOPER_DIR` 환경변수는 자동으로 설정되지 않는다. `xcodebuild`, `xcrun` 등 Xcode CLI 도구는 `DEVELOPER_DIR`을 우선 참조하기 때문에, 이 환경변수가 없으면 이전 Xcode를 참조하여 iOS 26.0 플랫폼을 찾지 못한다.

### 해결 방법

**1. `DEVELOPER_DIR` 명시적 선언**

빌드에 관여하는 모든 스텝(`Flutter build (no codesign)`, `Create Archive`)에 아래 env 블록 추가:

```yaml
env:
  DEVELOPER_DIR: /Applications/Xcode_${{ env.XCODE_VERSION }}.app/Contents/Developer
```

**2. `Install iOS device platform` 스텝 추가**

`Select Xcode version` 바로 다음 스텝으로 삽입. MobileDevice 패키지 설치 → Xcode 라이선스 동의 → iOS 플랫폼 다운로드 순으로 진행:

```yaml
- name: Install iOS device platform
  env:
    DEVELOPER_DIR: /Applications/Xcode_${{ env.XCODE_VERSION }}.app/Contents/Developer
  run: |
    sudo installer -pkg .../MobileDevice.pkg -target /
    sudo installer -pkg .../MobileDeviceDevelopment.pkg -target /
    sudo xcodebuild -license accept
    xcrun simctl list >/dev/null 2>&1 || true
    xcodebuild -downloadPlatform iOS
```

## 주의사항

- `xcodebuild -downloadPlatform iOS`는 네트워크 상태에 따라 수 분 소요될 수 있음
- 해당 수정은 `macos-26` 러너 기준이며, 이전 러너에서는 기존 방식(`xcode-select`만)으로도 동작할 수 있음
- 두 워크플로우(`TESTFLIGHT`, `TEST-TESTFLIGHT`)에 동일하게 적용해야 하며, 한쪽만 수정 시 불일치 발생 주의
