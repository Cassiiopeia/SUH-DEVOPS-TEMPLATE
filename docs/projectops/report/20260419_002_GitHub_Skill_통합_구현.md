# ⚙️[기능추가][Skills] GitHub Skill 통합 구현 및 전반적인 개선

## 개요

`gh` CLI 없이 Python 표준 라이브러리(urllib)만으로 GitHub REST API를 직접 호출하는 클라이언트를 구현하고, `/issue`, `/report`, `/github` 스킬이 GitHub와 완전히 연동되도록 개선했다. 브랜치명 계산도 GitHub Actions 의존에서 로컬 즉시 계산 방식으로 전환했다.

## 변경 사항

### Python 모듈 신규 추가

- `scripts/suh_template/gh_branch.py`: SUH-ISSUE-HELPER TypeScript 로직을 Python으로 포팅. `normalize_title`, `create_branch_name`(YYYYMMDD_#번호_제목 형식), `get_commit_template` 구현
- `scripts/suh_template/gh_client.py`: urllib 기반 GitHub REST API 클라이언트. `create_issue`, `add_comment`, `get_issue`, `list_issues`, `create_pull_request`, `list_pulls` 구현. `GitHubAPIError` 예외 처리 포함

### CLI 커맨드 확장

- `scripts/suh_template/cli.py`: `create-issue`, `add-comment`, `get-issue`, `create-branch-name`, `create-pr`, `list-prs` 6개 커맨드 추가. PAT는 `GITHUB_PAT` 환경변수로 전달

### 스킬 업데이트

- `skills/issue/SKILL.md`: GitHub API로 이슈 실제 생성 → 브랜치명 즉시 로컬 계산 → worktree/브랜치/현재 작업 선택지 제공
- `skills/report/SKILL.md`: 이슈 번호 자동 감지 후 GitHub 댓글 자동 포스팅
- `skills/github/SKILL.md`: 이슈 조회·댓글·PR 생성·PR 조회를 독립적으로 수행하는 신규 스킬
- `skills/references/common-rules.md`: GitHub 작업 원칙 섹션 추가 (gh CLI 사용 금지, python3 cli 커맨드 통일)

### 테스트

- `scripts/tests/test_gh_branch.py`, `test_gh_client.py`, `test_cli_github.py` 3개 파일, 71 tests passed

## 주요 구현 내용

### 왜 gh CLI를 쓰지 않는가

Windows/Mac 환경 호환성 문제, 별도 설치 필요, Claude Code 환경에서 PATH 미포함 케이스 발생. Python 표준 라이브러리 urllib만 사용하면 외부 의존성 없이 모든 플랫폼에서 동작한다.

### 브랜치명 로컬 계산

기존에는 이슈 생성 후 GitHub Actions `SUH-ISSUE-HELPER`가 댓글로 브랜치명을 달아주길 수 초 기다려야 했다. `gh_branch.py`가 동일 로직을 Python으로 구현하여 이슈 생성 즉시 로컬에서 계산한다.

### PAT 보안

GitHub PAT는 항상 환경변수 `GITHUB_PAT`로만 전달하며, 스킬 파일이나 설정 파일에 하드코딩하지 않는다. `.suh-template/config/issue.config.json`에 암호화 없이 저장되므로 `.gitignore` 필수 등록.

## 주의사항

- `gh` CLI는 이 프로젝트 스킬에서 완전히 사용 금지. 신규 스킬 작성 시 반드시 `python3 -m suh_template.cli` 커맨드 사용
- PAT에 `repo` 권한 필요 (이슈 생성, PR 생성 모두 포함)
- `list-prs` 커맨드는 `--state open|closed|all` 옵션 지원
