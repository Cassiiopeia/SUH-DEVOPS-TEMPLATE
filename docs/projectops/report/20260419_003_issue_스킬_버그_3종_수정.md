# ❗[버그][Skills] issue 스킬 산출물 경로, GitHub 즉시 등록, PYTHONPATH 누락 수정

## 개요

`/issue` 스킬의 버그 3종(산출물 경로 불일치, 사용자 확인 없는 즉시 GitHub 등록, PYTHONPATH 누락)을 수정했다. 아울러 `/commit` 스킬을 신규 추가하여 이슈 컨텍스트 기반 커밋 메시지 자동 완성 흐름을 구축했다.

## 변경 사항

### 산출물 경로 통일

- `skills/issue/SKILL.md`: `.issue/` → `docs/suh-template/issue/` (`get-output-path issue` 커맨드 사용)
- `skills/report/SKILL.md`: `.report/` → `docs/suh-template/report/`
- `.gitignore`: `/.report`, `/.issue` → `/docs/suh-template/`
- `.github/scripts/template_initializer.sh`: `ensure_gitignore()` 항목 동일하게 변경
- `template_integrator.sh`, `template_integrator.ps1`: `.gitignore` 항목 동일하게 변경

### 사용자 확인 단계 추가

- `skills/issue/SKILL.md`: 4단계(로컬 저장 + 사용자 확인) → 5단계(승인 후 GitHub 등록) 순서로 재구성. 승인 전 GitHub API 호출 절대 금지 명시

### PYTHONPATH 전체 수정

- `skills/references/common-rules.md`: `suh_template CLI 실행 규칙` 섹션 추가 — `PYTHONPATH="$PROJECT_ROOT/scripts"` 필수 명시
- 전체 스킬 13개 파일의 `python3 -m suh_template.cli` 호출 앞에 `PYTHONPATH` 일괄 추가

### 신규: `/commit` 스킬 및 커밋 컨벤션

- `skills/commit/SKILL.md`: 이슈 컨텍스트(`.suh-template/context/current-issue.json`) 기반 커밋 메시지 자동 완성. staged 파일 분석 → 타입 추천 → 사용자 확인 후 커밋
- `scripts/suh_template/cli.py`: `get-commit-template "<title>" "<url>"` 커맨드 추가
- `skills/references/common-rules.md`: 커밋 메시지 컨벤션 섹션 추가 (`이슈제목 : feat : 설명 이슈URL` 형식)

### 기타

- `.github/scripts/template_initializer.sh`: `.suh-template.example/` 폴더 삭제 처리 추가 (새 프로젝트 초기화 시 제거)

## 주요 구현 내용

### PYTHONPATH 누락 원인

`suh_template` 패키지가 `scripts/` 하위에 있어 Python 기본 경로에 포함되지 않는다. `python3 -m suh_template.cli` 단독 실행 시 `ModuleNotFoundError` 발생. 모든 호출 앞에 `PYTHONPATH="$PROJECT_ROOT/scripts"`를 붙이는 것으로 해결.

### 커밋 컨벤션 설계 원칙

강제하지 않고 옵션으로 제공. `/commit` 스킬을 명시적으로 호출할 때만 이슈 컨텍스트를 읽어 형식을 맞춰준다. 사용자가 직접 커밋하는 경우 어떤 형식도 강제하지 않는다.

## 주의사항

- 이전에 `.issue/`, `.report/` 경로로 저장된 파일은 `docs/suh-template/issue/`, `docs/suh-template/report/`로 수동 이동 필요
- `/commit` 스킬은 `git add` 대신 실행하지 않음 — staged 파일 없으면 안내 후 종료
- `push`는 `/commit` 스킬 범위 밖 — 별도로 처리
