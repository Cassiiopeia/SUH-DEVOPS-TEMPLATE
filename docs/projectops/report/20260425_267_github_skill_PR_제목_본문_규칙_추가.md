# github skill PR 생성 시 제목·본문 규칙 추가

## 개요

github skill로 PR을 생성할 때 일관된 형식이 없어 PR 제목에 이모지·태그가 그대로 포함되거나 이슈 링크가 누락되는 문제를 해결했다. PR 제목·본문 규칙을 skill에 명시하여 이후 PR 생성 시 자동으로 적용되도록 했다.

## 변경 사항

### Skills

- `skills/github/SKILL.md`: PR 생성 섹션에 제목·본문 필수 규칙 추가

## 주요 구현 내용

**PR 제목 규칙**
브랜치명(`YYYYMMDD_#번호_...`)에서 이슈 번호를 추출해 GitHub API로 이슈 제목을 조회한 뒤, 앞에 붙은 이모지와 `[태그]` 형식을 모두 제거한 순수 텍스트만 PR 제목으로 사용한다.

예) 이슈 제목 `❗[버그][개발자도구] SSE 서버 로그 스트리밍 연결 즉시 종료 및 구독자 누적 문제`
→ PR 제목: `SSE 서버 로그 스트리밍 연결 즉시 종료 및 구독자 누적 문제`

**PR 본문 규칙**
본문에 관련 이슈 링크를 한 줄 추가한다:
```
- https://github.com/{owner}/{repo}/issues/{이슈번호}
```

## 주의사항

- 이슈 번호는 브랜치명 패턴에서 자동 추출되므로, 브랜치명이 `YYYYMMDD_#번호_...` 형식을 따르지 않으면 수동으로 번호를 입력해야 한다.
