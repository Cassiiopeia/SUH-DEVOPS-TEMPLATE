## 문제 요약

`/testcase` 스킬 실행 시 생성 MD 파일이 프로젝트 루트에 저장됨.  
다른 스킬들(`analyze`, `plan`, `report` 등)은 `docs/suh-template/{skill_id}/`에 저장하는 반면, testcase만 `testcase-[번호]-[설명].md` 형태로 루트에 생성됨.

추가로 `analyze`, `plan` 등 7개 스킬이 경로 계산을 위해 `suh_template.cli get-output-path` CLI를 bash로 호출하고 있어, CLI 실패 시 경로를 잡지 못하는 리스크 존재.

---

## 변경 내용

### 수정 파일 (8개)

| 파일 | 변경 내용 |
|------|-----------|
| `skills/testcase/SKILL.md` | 저장 경로 `프로젝트 루트` → `docs/suh-template/testcase/YYYYMMDD_{이슈번호}_{제목}.md` |
| `skills/analyze/SKILL.md` | `get-output-path` CLI 호출 bash 블록 → 경로 직접 명시 |
| `skills/design-analyze/SKILL.md` | 동일 |
| `skills/plan/SKILL.md` | 동일 |
| `skills/ppt/SKILL.md` | 동일 |
| `skills/refactor-analyze/SKILL.md` | 동일 |
| `skills/review/SKILL.md` | 동일 |
| `skills/troubleshoot/SKILL.md` | 동일 |

### 저장 경로 통일 규칙

```
{PROJECT_ROOT}/docs/suh-template/{skill_id}/YYYYMMDD_{이슈번호}_{정규화된제목}.md
```

- 이슈번호 없으면 순번(`001`, `002`…) 자동 사용
- 제목 정규화: 특수문자 제거, 공백→`_`, 50자 이내

- CLI 의존성 제거로 안정성 향상
- 프로젝트 루트 오염 방지
