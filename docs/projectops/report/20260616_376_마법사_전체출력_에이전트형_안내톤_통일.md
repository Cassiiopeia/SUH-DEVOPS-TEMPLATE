# template_integrator 마법사 전체 출력을 에이전트형 안내 톤으로 전수조사하여 통일

## 개요
`template_integrator.sh` / `template_integrator.ps1` 마법사가 사용자에게 출력하던 메시지를 전수조사하여, 결과만 짧게 통보하던 기존 톤을 "지금 무엇이 감지됐고, 이제 무엇을 왜 할지"를 먼저 설명하는 맥락 우선·능동 화자(에이전트형) 톤으로 통일했다. 경로 설정 단계부터 시작해 타입/버전 감지, 정보 확인, 항목 수정, 템플릿 다운로드, 파일 생성, 워크플로우 복사, 옵션 안내 등 마법사의 모든 단계 메시지를 대상으로 했으며, sh와 ps1 양쪽 출력 문구를 1:1로 동일하게 맞추고 표기 규칙(가운뎃점 제거, "N개" 붙임 등)도 함께 정리했다.

## 변경 사항

### 경로 설정 단계 안내 강화 (5f4f2e9)
- `template_integrator.sh` · `template_integrator.ps1`: `resolve_project_paths` / `Resolve-ProjectPaths` 도입부에 "무엇이 감지됐고 이제 무엇을 할지"를 먼저 설명하는 요약 블록 추가. 멀티타입이면 "총 N개 타입", 단일이면 "총 1개 타입"으로 분기하고 타입별 마커 파일을 정렬해 나열.
- 후보 1건 확인 문구를 발견한 마커 파일과 `<레포루트>/...` 전체 경로가 함께 노출되도록 변경. "경로" 표현을 "프로젝트 루트"로 통일.
- 직접 입력 프롬프트에 해당 타입의 마커 파일 힌트(`pubspec.yaml`이 있는 폴더 등)를 명시.

### 마법사 출력 전체 톤 통일 (e235269)
- `template_integrator.sh` · `template_integrator.ps1`: 타입 감지(`detect_project_type` / `Detect-ProjectType`, `detect_project_types`)·버전 감지(`detect_version` / `Detect-Version`)·정보 확인(`detect_and_confirm_project` / `Ask-YesNoEdit`)·항목 수정(`handle_project_edit_menu`)·템플릿 다운로드(`download_template`)·README/version.yml 생성·옵션 읽기·breaking change 확인·워크플로우 복사 등 모든 단계의 사용자 출력 메시지를 능동 화자 톤으로 교체.
- sh/ps1 양쪽 출력 문구를 1:1로 동일하게 정렬(실행 로직 변경 없이 문자열만).
- 표기 정리: 항목 나열의 가운뎃점(·)을 쉼표로 교체, 파일 개수 표기를 "N개" 형태로 통일.

## 주요 구현 내용

핵심은 코드 동작이 아니라 **출력 문자열만** 손대는 작업이라, 메뉴/입력/감지 로직(`run:`성 분기, `choose_menu`/`Invoke-ArrowMenu` 호출, 파일 스캔)은 건드리지 않고 `print_*` / `Print-*`로 흘러가는 메시지만 일괄 교체했다. 톤 통일의 방향은 "결과 통보" → "맥락 설명 + 다음 행동 안내"다.

before / after 예시 (sh, ps1 동일하게 적용):

- 타입 감지
  - before: `감지됨: Flutter`
  - after: `✓ Flutter 감지됨`
- 기존 설정 재사용
  - before: `기존 version.yml 타입 사용: spring`
  - after: `✓ 기존 설정 적용: spring (version.yml에서 불러옴)`
- 정보 확인 헤더
  - before: `이 정보가 맞습니까?`
  - after: `위 분석 결과가 맞습니까?`
- 확인 완료
  - before: `프로젝트 정보 확인 완료`
  - after: `프로젝트 정보 검증 완료 — 이 설정으로 통합을 진행합니다`
- 취소 안내
  - before: `취소되었습니다`
  - after: `통합을 취소했습니다. (다시 실행하면 처음부터 설정할 수 있습니다)`
- 템플릿 다운로드
  - before: `템플릿 다운로드 중...` / `템플릿 다운로드 완료`
  - after: `템플릿을 GitHub 저장소에서 내려받고 있습니다...` / `템플릿 다운로드 완료 — 이제 프로젝트에 맞게 구성합니다`
- 경로 설정 도입부 (신규)
  - after: `🔍 멀티타입 프로젝트가 감지되었습니다 — 총 N개 타입` + 타입별 마커 파일 나열 + `💡 '프로젝트 루트' = 그 타입의 버전 파일이 있는 폴더 (레포 루트 기준 상대경로)`

옵션 안내·경고도 단순 통보에서 "왜 그런지 + 어떻게 대응할지"까지 덧붙이는 방향으로 통일했다. 예: `jq가 설치되지 않아 breaking change 확인을 건너뜁니다.` → `jq가 없어 호환성 변경(breaking change) 확인을 건너뜁니다 — jq 설치를 권장합니다.`, Synology 제외 시 `(나중에 --synology 옵션으로 추가 가능)` 안내 추가.

## 주의사항
- 이 작업은 사용자 출력 문자열만 변경하며 마법사의 실행 흐름·분기·파일 처리 로직은 변경하지 않는다(commit e235269 기준 sh/ps1 각각 +208/-208로 추가·삭제 라인 수가 동일한 순수 치환 성격).
- sh와 ps1의 동일 단계 출력 문구가 1:1로 일치하는 것이 유지 기준이므로, 이후 한쪽 메시지를 수정할 때는 반드시 다른 쪽도 같은 문구로 함께 맞춰야 한다.
- 표기 규칙(나열 시 쉼표 사용, 가운뎃점 미사용, 개수는 "N개")을 새 메시지 추가 시에도 동일하게 적용한다.
