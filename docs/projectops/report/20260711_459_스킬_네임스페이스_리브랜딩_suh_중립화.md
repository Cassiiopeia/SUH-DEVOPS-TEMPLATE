# 스킬 네임스페이스 리브랜딩: suh-* 중립화 (#459)

## 개요

`suh-*` 개인 브랜딩 스킬 네임스페이스를 `pro-*`로 중립화하는 작업의 **최종 검증 및 마무리**. 폴더 rename·문서 치환·매니페스트 갱신 등 본 작업은 선행 커밋들에서 완료돼 있었고, 이번에 전 영역 정합성을 검증해 잔여 문제 1건(테스트 기대값 불일치)을 수정하고 완료를 확정했다. v4.2.8로 릴리스됨.

## 검증 결과

| 검증 항목 | 결과 |
|---|---|
| `skills/` 아래 `suh-` 폴더 | 0개 (24종 전부 `pro-*`) |
| 활성 문서/코드의 `cassiiopeia:suh-` 표기 잔재 | 0건 (히스토리 문서·CHANGELOG 제외) |
| 루트 SUH 잔재 파일 (`SUH-DEVOPS-TEMPLATE-SETUP-GUIDE.md` 등) | 없음 (`PROJECTOPS-SETUP-GUIDE.md`로 대체됨) |

| 보존 대상 (과잉 치환 방지) | `me.suhsaechan:suh-logger` Maven 의존성·`Guide by SUH-LAB` 봇 서명 정상 보존 |
| 정합성 회귀 테스트 (`rename-consistency.test.js`) | 전 항목 통과 |
| 전체 테스트 (node + pytest + shell) | 전부 green |

## 변경 사항

### 테스트
- `test/rename-consistency.test.js`: 기대 스킬 폴더 목록을 25종 → **24종**으로 갱신 — `pro-issue`가 #467에서 `pro-github`로 통합·삭제된 것이 반영되지 않아 실패하던 것을 수정

## 주요 구현 내용

- 리브랜딩 자체는 선행 작업(폴더 25종 rename, SKILL.md `name:` 필드, CLI 헤더 경로, py 테스트 sys.path, 문서·CLAUDE.md 표기, breaking-changes 4.3.0)으로 완료된 상태였다
- 이번 작업의 본질은 **"완료 주장"을 회귀 테스트로 증명**하는 것 — `rename-consistency.test.js`가 suh- 잔재 금지 + 보존 대상 생존을 동시에 지키는 가드로 동작함을 확인
- 이슈에서 논의된 하위 호환(구 커맨드 alias)은 채택하지 않음 — 4.3.0 breaking change로 안내하고 플러그인 재설치 + 레거시 자동 마이그레이션(#464)으로 이주 경로를 제공

## 주의사항

- 히스토리 문서(`docs/projectops/issue|report`, `docs/superpowers/`, CHANGELOG)의 `suh-` 표기는 의도적으로 보존 — 과거 기록이므로 치환 대상이 아니다
- 외부 실체(`me.suhsaechan:suh-logger`, `Guide by SUH-LAB`, `*.suhsaechan.kr`)는 치환 금지 대상으로 테스트가 지키고 있다
