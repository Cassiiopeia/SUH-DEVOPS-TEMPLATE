# 마법사 출력 가독성 및 옵션 승계 UX 개선 3건

## 개요

suh-project-utility 실물 통합 QA에서 수집한 마법사 UX 3건을 수정했다. ① breaking-changes 배너가 CRITICAL/WARNING 전문을 통째로 덤프해 벽글이 되고 `║` 박스 경계가 래핑으로 붕괴되던 것을 버전·제목 요약 리스트로 축약, ② 완료 요약의 "새로 설치됨" 목록이 디렉토리 스캔 기반이라 실제 설치와 무관하게 존재 파일 전부(레거시 포함)를 나열하고 카운터와 불일치하던 것을 복사 엔진의 실복사 목록 기반으로 교체, ③ 구 `synology: true` 옵션이 신 축으로 승계되지 않아 비대화형 업데이트에서 Secret 백업 워크플로우를 조용히 잃던 것을 자동 승계로 해결했다.

## 변경 사항

### ① breaking-changes 배너 요약화
- `src/core/breaking-check.js`: 전문 통덤프 + 박스 테두리 출력을 제거하고 `[CRITICAL] 버전 — 제목` 한 줄 요약 리스트로 교체. 대화형은 "상세 내용을 볼까요?" 선택 시에만 전문 표시, 비대화형은 `breaking-changes.json` 참고 경로 안내. CRITICAL 진행 확인 게이트는 기존 그대로 유지

### ② 완료 요약 실목록화
- `src/core/copy/workflows.js`: 복사 엔진이 실제 복사·교체한 파일명을 `copiedFiles` 배열로 반환 (7개 복사 지점 전부 계측)
- `src/ui/summary.js`: 디렉토리 스캔 분류를 제거하고 `copiedFiles`를 유일한 소스로 출력 — 카운터=목록 길이 일치, 복사 0건이면 "모두 최신 상태" 안내
- `src/index.js` / `src/commands/interactive.js`: 요약 호출부에 `workflowFiles` 전달

### ③ 구 synology 옵션 승계
- `src/core/version-yml.js`: `parseTemplateOptions`에 `synology: true` → `secret_backup` 미기재 시 `true` 승계 규칙 추가 (기존 nexus/npm_publish 구 키 자동 마이그레이션과 동일 패턴, 신 키 명시 시 신 키 우선)

### 테스트
- `test/breaking.test.js` 3종(비대화형 요약·대화형 선택 열람·진행 게이트), `test/summary.test.js` 갱신(실목록·레거시 미표기·카운터 일치), `test/options-ask.test.js` 3종(승계·신 키 우선·false 미승계)

## 주요 구현 내용

- 요약 목록의 "새로 설치됨 (0개): 파일 9개 나열" 불일치는 카운터(실복사 수)와 목록(디렉토리 스캔)의 **소스가 달랐던 것**이 원인 — 소스를 복사 엔진 반환값 하나로 통일
- 실물 스모크: 구식 레포 픽스처에서 요약 배너 7건 한 줄씩 출력, 설치 목록 14개 정확 표기, `synology: true` 레포에 `PROJECT-COMMON-SECRET-FILE-UPLOAD.yaml` 자동 포함 확인
