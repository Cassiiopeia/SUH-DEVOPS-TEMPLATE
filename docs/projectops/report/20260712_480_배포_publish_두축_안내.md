# 배포/publish 두 축을 먼저 갈라 묻기

## 개요

배포 질문에서 deploy(서버 실행물)와 publish(라이브러리 레지스트리)가 서로를 모르는 채 순차로 나와, Spring 사용자가 "서버 앱인데 왜 라이브러리를 묻지?", "라이브러리는 여기서 고르나?" 하고 혼란스러워하던 문제를 완화했다. 두 질문이 하나의 그림(독립된 두 축)임을 먼저 안내하고, deploy 선택지에서 다른 축(라이브러리/CI) 용어를 제거했다.

## 변경 사항

### 두 축 큰 그림 안내
- `src/core/options-ask.js`: deploy·publish를 둘 다 물을 참이면, 질문 앞에 "배포는 두 가지가 따로 있습니다 — (1) 실행물 배포 (2) 라이브러리 배포" 안내를 한 번 출력. deploy 질문을 "(1) 실행물 배포", publish를 "(2) 라이브러리 배포"로 번호 매겨 관계를 명시.

### deploy 선택지 문구 정정
- deploy의 "배포하지 않음 (라이브러리/CI 전용)" → "서버에 올리지 않음 (빌드 검증만 · 라이브러리는 다음에서 선택)"로 교체. 이 질문 범위(실행물)에 없는 용어(라이브러리/CI) 제거.

### 수정 메뉴 격리와의 조합
- 수정 메뉴로 한 축(deploy 또는 publish)만 편집할 때는 맥락이 이미 명확하므로 큰 그림 안내를 생략(#483 scope와 연동).

### 테스트
- `test/options-ask.test.js`: 두 축 다 물을 때 큰 그림 안내 출력 + deploy 문구 갱신 확인, 한 축만 수정(scope)이면 안내 생략 2종.

## 주요 구현 내용

- 이번 변경은 두 축을 "먼저 갈라 안내"하는 문구 개선이다. 더 근본적으로는 "프로젝트 성격을 먼저 묻고 옵션을 유도"하는 진입 구조 재설계 여지가 있어(별도 이슈), 이 개선은 그 중간 단계다.
