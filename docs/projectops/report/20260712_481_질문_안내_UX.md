# 마법사 질문 안내 UX 개선 (publish 명시 스킵·CodeRabbit 후속 안내·changelog 폴백 표현)

## 개요

대화형 통합 흐름에서 질문 문구·안내가 사용자를 혼란시키던 지점을 개선했다. ① publish를 "그냥 Enter"로 스킵하게 해 무엇을 건너뛴 건지 불명확하던 것, ② CodeRabbit "사용"만 고르면 실제로는 앱 설치·접근 권한이 필요한데 안내가 없어 리뷰가 안 붙던 것, ③ changelog가 "하나만 골라야" 하는 것처럼 보였지만 실제로는 자동 폴백 사다리라는 점이 안 드러나던 것. 저장 로직·폴백 동작은 그대로 두고 문구·안내만 개선했다.

## 변경 사항

### publish 명시 스킵
- `src/core/options-ask.js`: "없으면 그냥 Enter" → "선택 없이 Enter = 배포 안 함"으로 명시. 결과 로그도 "라이브러리 배포: 안 함"으로 분명히 표기.

### CodeRabbit 후속 안내
- `src/core/options-ask.js`: "사용" 선택 시 coderabbit.ai 로그인 → GitHub 앱 설치 → 저장소 접근 권한(grant access) 부여 3단계 안내를 그 자리에서 출력. "이 단계를 안 하면 워크플로우가 켜져도 리뷰가 안 달린다" 경고 포함.
- `src/ui/summary.js`: 완료 요약의 "CodeRabbit 활성화" 안내도 grant access 문구로 보강.

### changelog 폴백 표현
- `src/core/options-ask.js`: "1순위로 뭘로 만들까요" + "고른 게 실패해도 자동으로 GitHub AI → 커밋 분석 순으로 폴백한다" 안내 추가. 선택이 "1순위 생성기 지정"임을 명확히.

### 테스트
- `test/options-ask.test.js`: CodeRabbit grant access 안내 출력/미출력, changelog 폴백 안내, publish "배포 안 함" 명시 로그 4종.

## 주요 구현 내용

- 세 개선 모두 `say()` 안내 문구 추가와 라벨 수정 수준이라 저장 스키마·폴백 사다리(ladder.py) 동작에는 영향이 없다.
