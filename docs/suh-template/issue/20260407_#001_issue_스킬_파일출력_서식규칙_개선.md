# ⚙️[기능개선][Skills] issue 스킬 파일 출력 및 서식 규칙 개선

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- `/issue` 스킬이 이슈 내용을 인라인(터미널)으로만 출력하고 `.issue/` 폴더에 `.md` 파일을 생성하지 않음
- `/report` 스킬은 `.report/` 폴더에 파일을 정상 생성하지만 `/issue`는 파일 생성 지시가 스킬 정의에 없음
- 이모지와 텍스트 사이 공백 규칙이 명시되어 있지 않아 `🗒️ 설명` 같이 공백이 포함되는 경우 발생
- 파일 네이밍 컨벤션이 정의되어 있지 않음
- 이슈 제목을 바로 복사할 수 있도록 파일 첫 줄에 헤딩으로 출력하는 규칙 부재

🛠️해결 방안 / 제안 기능
---

- `skills/issue/SKILL.md`에 **4단계: 파일 저장** 섹션 추가
  - 파일 위치: `.issue/[YYYY-MM-DD]-[제목-kebab-case].md`
  - `/report` 스킬의 `.report/` 파일 생성 패턴 참고
- **이모지 서식 절대 규칙** 섹션 추가
  - 이모지와 텍스트/대괄호 사이 빈칸 금지: `🗒️설명` (O), `🗒️ 설명` (X)
- 파일 첫 줄에 이슈 제목을 `# ` 헤딩으로 작성하여 바로 복사 가능하게 지시
- 기존 `4단계: 출력`을 `5단계: 사용자에게 안내`로 변경하여 파일 경로 안내

⚙️작업 내용
---

- `skills/issue/SKILL.md` 수정
  - `## 절대 규칙 > 이모지 서식` 섹션 추가
  - `### 4단계: 파일 저장` 섹션 추가 (파일 위치, 네이밍 컨벤션, 출력 포맷)
  - `### 5단계: 사용자에게 안내` 섹션 추가
  - 기능 세분류 이모지 공백 제거 (`➕ 기능추가` → `➕기능추가`)

**영향 받는 파일**:
- `skills/issue/SKILL.md`
