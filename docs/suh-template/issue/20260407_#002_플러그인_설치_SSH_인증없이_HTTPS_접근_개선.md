# ⚙️[기능개선][Plugin] 플러그인 설치 시 SSH 인증 없이 HTTPS로 접근 가능하도록 개선

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- `claude plugin install cassiiopeia@cassiiopeia-marketplace --scope user` 실행 시, 내부적으로 **SSH 프로토콜(`git@github.com:`)로 저장소를 clone** 시도
- Public 저장소임에도 불구하고 SSH 키가 설정되지 않은 사용자 환경에서 아래 에러 발생:
  ```
  ✘ Failed to install plugin "cassiiopeia@cassiiopeia-marketplace":
  Failed to clone repository:
  git@github.com: Permission denied (publickey).
  fatal: Could not read from remote repository.
  ```
- **영향 범위**:
  - SSH 키를 등록하지 않은 모든 사용자가 플러그인 설치 불가
  - `template_integrator.sh`, `template_integrator.ps1`의 자동 설치 기능도 동일하게 영향받음
- **관련 파일**:
  - `.claude-plugin/marketplace.json` — `source.repo` 필드
  - `template_integrator.sh` — `offer_ide_tools_install()` 함수
  - `template_integrator.ps1` — `Offer-IdeToolsInstall()` 함수

🛠️해결 방안 / 제안 기능
---

### 원인 분석

`marketplace.json`의 플러그인 소스 정의:
```json
"source": {
  "source": "github",
  "repo": "Cassiiopeia/SUH-DEVOPS-TEMPLATE"
}
```
Claude Code 플러그인 시스템이 `source: "github"` + `repo` 조합을 **SSH URL(`git@github.com:`)로 변환**하는 것이 근본 원인

**Claude Code 공식 레포 관련 이슈** (Anthropic 측 알려진 버그):
- [anthropics/claude-code#29722](https://github.com/anthropics/claude-code/issues/29722) — `plugin install`에 HTTPS 폴백 로직 없음 (`marketplace add`에는 있음)
- [anthropics/claude-code#26588](https://github.com/anthropics/claude-code/issues/26588) — Public 저장소에 SSH 인증 요구 불필요
- [anthropics/claude-code#31930](https://github.com/anthropics/claude-code/issues/31930) — 공개 저장소는 HTTPS가 기본값이어야 함

### 해결 방안: `source: "url"` 타입으로 변경

`marketplace.json`의 `source` 필드를 `"github"` 타입 대신 `"url"` 타입으로 변경하여 HTTPS URL을 직접 지정:

**변경 전**:
```json
"source": {
  "source": "github",
  "repo": "Cassiiopeia/SUH-DEVOPS-TEMPLATE"
}
```

**변경 후**:
```json
"source": {
  "source": "url",
  "url": "https://github.com/Cassiiopeia/SUH-DEVOPS-TEMPLATE.git"
}
```

### 추가 조치

1. **설치 가이드에 workaround 추가**: `source: "url"` 변경이 안 될 경우, SSH 키가 없는 사용자를 위해 `git config --global url."https://github.com/".insteadOf "git@github.com:"` 안내 추가
2. **template_integrator에 사전 체크 로직 추가**: 설치 전 SSH 연결 테스트 후 실패 시 HTTPS 전환 안내
