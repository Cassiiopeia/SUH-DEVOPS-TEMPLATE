# ⚙️[기능개선][Skills] commit 스킬 superpowers 원칙 준수 개선

**라벨**: `작업전`
**담당자**: @Cassiiopeia

---

📝현재 문제점
---

- `commit` 스킬이 이슈 컨텍스트 없을 때 선택지 제시 없이 바로 자유 형식 커밋으로 진행하거나, 이슈 번호 입력만 받는 단순한 분기만 존재
- superpowers `executing-plans` 원칙("블로커 만나면 멈추고 명확한 선택지 제시")을 따르지 않음
- 이슈 생성이 필요한 상황에서 `/issue` 플로우로 자연스럽게 연결되는 흐름 없음
- staged 파일 없을 때도 단순 안내만 하고 선택지 없이 종료

🛠️해결 방안 / 제안 기능
---

- 이슈 컨텍스트 없을 때 4개 선택지 명확히 제시: 이슈 새로 만들기 / 번호 입력 / 자유 형식 / 취소
- staged 파일 없을 때도 선택지 제시 후 사용자 응답 대기 (자동 `git add` 절대 금지)
- 커밋 메시지 제안 후 반드시 사용자 확인 대기 — 응답 전 커밋 실행 금지
- 1번(이슈 새로 만들기) 선택 시 `/issue` 스킬 안내 후 종료, 이슈 생성 후 다시 `/commit` 호출하도록 안내

⚙️작업 내용
---

- `skills/commit/SKILL.md`: 핵심 원칙 섹션 추가, 각 블로커 단계마다 선택지 구조로 재구성
