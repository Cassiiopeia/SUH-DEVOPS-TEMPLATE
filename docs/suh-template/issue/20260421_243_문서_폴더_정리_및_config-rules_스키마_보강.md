📝 현재 문제점
---

- `.issue/`, `.report/` 폴더가 프로젝트 루트에 흩어져 있어 문서 위치가 일관되지 않았음
- `.suh-template/context/current-issue.json`이 gitignore 없이 템플릿 저장소에 커밋되어 있었음
- `.suh-template.example/config/`에 예시 파일이 별도 폴더로 존재해 혼란을 줬음
- `config-rules.md`에 skill별 config 스키마가 없어 어떤 필드를 써야 하는지 알기 어려웠음

🛠️ 해결 방안 / 제안 기능
---

- `.issue/`, `.report/` 문서를 `docs/suh-template/issue/`, `docs/suh-template/report/`로 이동
- `.suh-template/`, `.suh-template.example/` 불필요한 폴더 삭제
- `skills/references/config-rules.md`에 §7 추가 — `issue.config.json`, `synology-expose.config.json` 스키마 및 필드 설명 인라인화

⚙️ 작업 내용
---

- `docs/suh-template/issue/`로 이슈 문서 이동 (3개)
- `docs/suh-template/report/`로 리포트 문서 이동 (51개)
- `.suh-template/`, `.suh-template.example/` 폴더 삭제
- `config-rules.md` §7 Skill별 Config 스키마 섹션 추가
