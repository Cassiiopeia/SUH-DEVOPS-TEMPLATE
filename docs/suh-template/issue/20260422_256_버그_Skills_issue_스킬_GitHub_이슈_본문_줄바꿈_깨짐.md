# ❗[버그][Skills] issue 스킬 GitHub 이슈 본문 줄바꿈 깨짐

- **라벨**: 작업전
- **담당자**: Cassiiopeia

---

🗒️ 설명
---

`/issue` 스킬로 GitHub 이슈를 등록할 때, 이슈 본문의 줄바꿈이 실제 개행 문자가 아닌 `\n` 이스케이프 문자 그대로 렌더링되는 버그다.

- `skills/issue/SKILL.md` 5단계에서 curl `-d "..."` 인라인 문자열 방식으로 JSON body를 전달
- 멀티라인 본문이 `\n` 텍스트로 그대로 GitHub API에 전송됨
- GitHub 웹 UI에서 본문이 단락 구분 없이 `\n` 문자가 그대로 표시됨

🔄 재현 방법
---

1. `/issue` 스킬로 이슈 내용 작성 후 GitHub 등록 승인
2. 생성된 GitHub 이슈 본문 확인
3. 줄바꿈 대신 `\n` 문자가 그대로 표시됨

📸 참고 자료
---

영향받는 파일:

| 파일 | 문제 코드 |
|------|-----------|
| `skills/issue/SKILL.md` (5단계) | `curl -d "{\"body\": \"{본문}\"}"` 인라인 방식 |

✅ 예상 동작
---

- GitHub 이슈 본문이 마크다운 형식으로 정상 렌더링되어야 함
- 각 섹션(🗒️ 설명, 🔄 재현 방법 등)이 줄바꿈으로 구분되어 표시되어야 함

⚙️ 환경 정보
---

- **영향 범위**: `/issue` 스킬을 사용하는 모든 프로젝트
- **발생 조건**: 이슈 본문에 줄바꿈이 포함될 때 (사실상 항상 발생)
