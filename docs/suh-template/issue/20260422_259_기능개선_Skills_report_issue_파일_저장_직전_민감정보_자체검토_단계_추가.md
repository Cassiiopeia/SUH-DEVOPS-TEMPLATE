# 🚀[기능개선][Skills] report·issue 스킬 파일 저장 직전 민감정보 자체검토 단계 추가

- **라벨**: 작업전
- **담당자**: Cassiiopeia

---

📝 현재 문제점
---

- `report`, `issue` 스킬이 파일을 저장할 때 민감 정보(실제 비밀번호, ID/PW, API Key 실제값 등) 포함 여부를 검토하는 명시적 단계가 없다
- `common-rules.md`에 마스킹 규칙이 정의되어 있지만, 스킬 본문에는 "참조하라"는 한 줄만 있어 실제로 AI가 체크를 건너뛰는 경우가 발생한다
- 결과적으로 관리자 계정 아이디/비밀번호 등 민감 정보가 보고서나 이슈 파일에 그대로 커밋될 위험이 있다

🛠️ 해결 방안 / 제안 기능
---

- `common-rules.md`에 **파일 저장 직전 자체검토 프로토콜** 섹션 추가
  - 위험/안전 예시 표 (비밀번호, ID/PW, API Key, PAT, DB 접속정보, 개인정보)
  - 판단 기준 명시 (`${{ secrets.XXX }}`는 안전, 실제값 평문 노출은 위험)
  - 마스킹 처리 및 사용자 고지 방식 정의
- `report/SKILL.md` 산출물 저장 섹션 직전에 자체검토 단계 명시적 삽입
- `issue/SKILL.md` 4단계 파일 저장 직전에 자체검토 단계 명시적 삽입

⚙️ 작업 내용
---

- `skills/references/common-rules.md`: 민감 정보 보호 섹션 하단에 자체검토 프로토콜 추가
- `skills/report/SKILL.md`: `## 파일 저장 직전 민감정보 자체검토` 단계 삽입
- `skills/issue/SKILL.md`: 4단계 저장 직전 체크 문구 삽입
