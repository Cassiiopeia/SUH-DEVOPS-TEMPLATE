📝 현재 문제점
---

- github skill로 PR을 생성할 때 제목과 본문에 대한 명시적인 규칙이 없어 일관성이 없음
- PR 제목에 이슈 제목의 이모지·`[태그]`가 그대로 포함되어 가독성이 저하됨
- PR 본문에 관련 이슈 링크가 누락되어 PR과 이슈 간 연결 추적이 어려움

🛠️ 해결 방안 / 제안 기능
---

- PR 제목 규칙: 이슈 제목에서 앞에 붙은 이모지와 `[태그]` 형식을 제거한 순수 텍스트 사용
- PR 본문 규칙: 본문에 `- https://github.com/{owner}/{repo}/issues/{번호}` 형식의 이슈 링크 한 줄 추가
- 브랜치명(`YYYYMMDD_#번호_...`)에서 이슈 번호를 자동 추출하여 링크 생성

⚙️ 작업 내용
---

- `skills/github/SKILL.md` PR 생성 섹션에 제목·본문 규칙 추가
