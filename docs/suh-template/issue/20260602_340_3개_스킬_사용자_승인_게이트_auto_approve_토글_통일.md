📝 현재 문제점
---

- `suh-commit` / `suh-issue` / `suh-changelog-deploy` 3개 스킬은 모두 작업 직전 사용자 확인을 받는 게이트가 있다. 같은 레포에서 반복 작업할 때 매번 동일한 확인이 떠 마찰이 크다.
- `suh-changelog-deploy`는 `changelog_deploy.auto_approve_release_notes` 키로 자동 모드를 지원하지만, `suh-commit`·`suh-issue`는 토글 자체가 없어 매번 수동 승인이 필요하다.
- 키 이름·구조·해석 우선순위가 스킬마다 다르거나 부재해 일관성이 없고 학습 비용이 든다.

🛠️ 해결 방안 / 제안 기능
---

- 3개 스킬의 사용자 승인 게이트 토글을 **단일 패턴**으로 통일한다.
  - 키 이름: `commit.auto_approve` / `issue.auto_approve` / `changelog_deploy.auto_approve`
  - 해석 우선순위: `repos[i].{skill}.auto_approve` → `github.{skill}.auto_approve` → `false`
- SKILL.md에 표준 A/B/C 분기 패턴을 적용한다.
  - A. 자동 모드: 요약 표시 후 즉시 진행 (응답 대기 없음)
  - B. 수동 모드: 기존 게이트 그대로
  - C. 첫 실행 자동화 제안: 키가 처음 적용될 때 1회만 "이 레포 / 모든 레포 / 매번 확인" 선택 → agent가 config 갱신
- `suh-issue` 자동 모드는 "최종 등록 승인"만 스킵한다. 중복 검사(2-1, 4-1)와 open 동일 이슈 발견 시 중단 정책은 항상 적용된다.
- 사용자에게 키 이름·파일 경로를 노출하지 않는다. 자연어 응답("자동으로 진행해줘" / "확인받게 해줘")만으로 토글 가능하며 agent가 Read/Write로 config를 직접 갱신한다.
- 구 키 `changelog_deploy.auto_approve_release_notes`는 **명시적 break**. 더 이상 인식하지 않으며 config에 남아 있어도 무시된다.
