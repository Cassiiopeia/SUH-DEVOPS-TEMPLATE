🗒️ 설명
---

`template_integrator.sh`의 대화형 메뉴(`interactive_menu`)에서 화살표 키(↑↓)로 프로젝트 타입을 선택하려 하면, **화살표 키가 ESC로 오인되어 메뉴가 멋대로 취소**된다.

- macOS 기본 셸 환경(`/bin/bash` 3.2.57)에서 재현된다.
- 사용자가 ↓(아래 화살표)를 눌렀을 뿐인데 `💥 프로젝트 타입 선택이 취소되었습니다. 기존 값을 유지합니다.` 메시지가 출력되며 선택 메뉴를 빠져나간다.

🔄 재현 방법
---

1. macOS 기본 bash(3.2.57) 환경에서 `template_integrator.sh`를 실행한다.
2. 프로젝트 분석 결과 화면에서 `E`(수정하기)를 선택해 "프로젝트 타입을 선택하세요" 메뉴로 진입한다.
3. ↑ 또는 ↓ 화살표 키를 눌러 항목을 이동하려 한다.
4. 이동되지 않고 곧바로 `💥 프로젝트 타입 선택이 취소되었습니다. 기존 값을 유지합니다.`가 출력되며 메뉴가 종료된다.

📸 참고 자료
---

원인 분석 (ESC 시퀀스 파싱):

- 화살표 키는 터미널에서 3바이트 시퀀스(`ESC [ A` / `ESC [ B`)로 전송된다.
- 기존 코드는 첫 바이트 `ESC`를 읽은 뒤, 나머지 `[A`/`[B`를 `read -rsn2 -t 0.01`(소수점 타임아웃)로 별도로 읽는다.
- **macOS bash 3.2는 `-t 0.01` 소수점 타임아웃을 0초로 절삭**한다. 즉시 타임아웃되어 나머지 시퀀스를 놓치고, `rest=""`가 되어 **ESC 키로 오인** → 메뉴가 취소된다.

검증 결과(bash 3.2):

| 코드 | `[B`(아래 화살표) 입력 시 | 결과 |
|------|------|------|
| 기존 `read -rsn2 -t 0.01` | `rest=''` (즉시 타임아웃) | ESC로 오인 → 메뉴 취소 |
| 수정 `read -rsn1 -t 1` 2회 | `rest='[B'` | DOWN 정상 인식 |

관련 파일: `template_integrator.sh` (interactive_menu 함수, ESC 시퀀스 파싱부)

> `template_integrator.ps1`(PowerShell)은 `[Console]::ReadKey()`가 OS 레벨에서 화살표 키를 단일 키 이벤트로 해석하므로 동일 버그가 없다(확인 완료).

✅ 예상 동작
---

- ↑/↓ 화살표 키로 메뉴 항목 사이를 정상적으로 이동할 수 있어야 한다.
- 진짜 ESC 키를 눌렀을 때만 메뉴가 취소되어야 한다.
