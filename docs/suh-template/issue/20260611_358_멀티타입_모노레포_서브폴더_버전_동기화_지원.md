📝 현재 문제점
---

- 멀티타입(`--type flutter,react,python`)으로 통합한 프로젝트라도, 각 타입의 버전 파일이 레포 **루트**에 있을 때만 동기화되었다.
- `passQL`(app/·client/·ai/·server/), `PickerPicker`(루트 프론트 + backend/ 백엔드)처럼 타입별 프로젝트가 **서브폴더에 흩어진 모노레포**에서는 `version_manager.sh`가 루트만 확인하여, flutter/react/python 동기화가 **에러 없이 조용히 no-op** 되었다.
- 결과적으로 `version.yml`만 버전이 올라가고 `app/pubspec.yaml`, `client/package.json`, `backend/pyproject.toml` 등 실제 서브프로젝트 버전 파일은 영원히 동기화되지 않는 문제가 있었다.
- 한쪽은 루트, 다른 쪽은 서브폴더인 **비대칭 구조**(PickerPicker)는 더더욱 지원 범위 밖이었다.

🛠️ 해결 방안 / 제안 기능
---

- `version.yml`에 **`project_paths` 맵**(타입 → 레포 루트 기준 상대경로)을 신설한다.
- `template_integrator.sh` / `.ps1`이 통합 시 타입별 **마커 파일을 자동 감지**(maxdepth 3, 잡음 폴더 제외, 오탐 방지)하여 "이 경로 맞아요?"를 확인하고, 후보가 여러 개면 선택, 없으면 상대경로를 직접 입력받아 `project_paths`에 기록한다.
- 루트에 마커가 있으면 질문 없이 `"."`로 자동 확정한다. 비대화형은 `--paths "flutter=app,react=client"`(`.ps1`은 `-Paths`)로 지정한다.
- `version_manager.sh`가 `project_paths`를 읽어 해당 서브폴더의 버전 파일을 동기화한다.
- **`project_paths` 키가 없으면 기존 루트 동작을 100% 유지**한다(완전한 하위 호환).
- 버전 관리 워크플로우(`PROJECT-COMMON-VERSION-CONTROL`)는 **수정하지 않는다** — 스크립트가 경로를 알고, 커밋이 `git add -A`라 서브폴더 변경이 자동 포함되기 때문이다.

⚙️ 작업 내용
