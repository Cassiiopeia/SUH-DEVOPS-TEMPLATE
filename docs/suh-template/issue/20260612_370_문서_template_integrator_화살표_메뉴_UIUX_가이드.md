📝 현재 문제점
---

대화형 마법사 `template_integrator`(`template_integrator.sh` / `template_integrator.ps1`)의 선택 UI 사용법이 문서로 정리되어 있지 않습니다.

- 사용자가 OS별(macOS/Linux의 zsh·bash, Windows의 PowerShell)로 마법사를 **어떻게 실행해야 화살표 키 메뉴를 쓸 수 있는지** 안내하는 가이드가 없습니다.
- 화살표 메뉴의 조작법(↑/↓ 이동, Enter 확정, Esc 취소, 멀티 셀렉트의 Space 토글·`a` 전체 토글)이 문서화되어 있지 않습니다.
- 어떤 실행 방식에서 화살표 메뉴가 동작하고, 어떤 환경에서 번호 입력으로 폴백되는지에 대한 설명이 없어, 사용자가 콤마 번호 입력 화면을 보고 "왜 화살표가 안 되지?"라고 혼란을 겪습니다.

> **이 이슈는 문서 작성만을 범위로 한다.** 마법사의 화살표 메뉴 동작/코드 변경은 이 이슈의 범위가 아니다. 오직 사용 방법과 UI/UX 가이드 문서를 작성하는 것이 목적이다.

🛠️ 해결 방안 / 제안 기능
---

대화형 마법사의 좋은 UI/UX 사용법을 **OS별로 정리한 가이드 문서**를 작성합니다.

**문서에 담을 내용**

1. **OS별 실행 방법**
   - **macOS / Linux (zsh · bash)** — `template_integrator.sh` 실행 방법. 원격 실행(`curl`)과 로컬 파일 실행 각각에서 화살표 메뉴가 동작함을 안내.
   - **Windows (PowerShell)** — `template_integrator.ps1` 실행 방법. 화살표 메뉴를 쓰기 위한 권장 실행 방식과, 화면에 콤마 번호 입력이 나올 때의 의미를 안내.

2. **화살표 메뉴 조작법**
   - 단일 선택: ↑/↓ 이동, Enter 확정, Esc 취소, 숫자 점프.
   - 멀티 셀렉트: ↑/↓ 이동, Space 토글(`[✓]`/`[ ]`), `a` 전체 토글, Enter 확정.

3. **번호 입력 폴백 안내**
   - 화살표 메뉴가 동작하지 않는 환경(예: 일부 비대화형 실행, PowerShell ISE 등)에서는 번호 입력 방식으로 자동 전환됨을 설명.
   - 번호 입력 시 단일은 숫자 하나, 멀티는 `1,3,5`처럼 콤마로 입력하는 방법 안내.

4. **OS별 동작 비교 표**
   - zsh/bash vs PowerShell에서 화살표 메뉴·멀티 셀렉트가 어떻게 보이고 동작하는지 한눈에 비교.

**가이드 작성의 근거 (검증으로 확인된 사실)**

문서의 정확성을 위해 아래 실측 결과를 근거로 삼습니다.

- macOS/Linux의 `template_integrator.sh`는 화살표 키(↑/↓) 이동 메뉴(`interactive_menu`)가 이미 동작하며, `curl | bash` 파이프 실행에서도 `/dev/tty` 우회로 화살표가 동작함.
- Windows의 `template_integrator.ps1`은 화면에서 콤마 번호 입력으로 보이는 경우가 있으나, 이는 실행 방식(원격 다운로드 스트림)에 따른 것이며 PowerShell 자체가 화살표를 못 쓰는 것은 아님을 실측으로 확인함.
- 화살표 메뉴의 좋은 UX 원칙: 화살표를 우선 제공하고, 동작 불가 환경에서만 번호 입력으로 안전하게 폴백.
