📝 현재 문제점
---

- `harness/PERSONA.md`·`harness/WORKFLOW.md`에는 5개 전문가 페르소나(Architect / Developer / Reviewer / SDET / Frontend)와 강력한 품질 강제 장치(Devil's Advocate, Stop-and-Think Gate, Anti-Confirmation Bias)가 정의되어 있습니다.
- 그러나 이 자산은 PI(pi-coding-agent) 에이전트 전용으로만 시스템 프롬프트에 주입되고, Claude Code에서 `suh-plan` / `suh-analyze` / `suh-implement` 3종 코어 스킬을 실행할 때는 전혀 활용되지 않습니다.
- 그 결과 3종 스킬은 superpowers 패턴(writing-plans·executing-plans·verification)은 흡수했으나, "의심 → 대안 비교 → 적대적 자기검증" 루프가 없어 매 실행마다 결과 품질이 들쭉날쭉합니다.

🛠️ 해결 방안 / 제안 기능
---

- harness의 페르소나 시스템과 품질 강제 메커니즘을 3종 코어 스킬에 추합 강화하여, 에이전트가 항상 일관적이고 요구사항에 정확히 맞는 결과를 내도록 구현력을 끌어올립니다.
- 페르소나는 장식이 아니라 **행동 강제 레이어**로 동작합니다 — 각 스킬이 시작 시 자기 페르소나 카드와 공통 마인드셋을 명시적으로 로드합니다.
- HARD-GATE(리뷰 로그 저장 전 다음 단계 진입 금지)와 Devil's Advocate(결함 1개 이상 의무 지적)로 같은 수준의 검증을 거친 결과만 통과하게 합니다.
- 단순 작업에는 Fast-Track 예외를 두어 의식(ritual)의 무게를 지우지 않습니다.
- PI harness 원본은 수정하지 않고 single source로 보존하며, 강화 로직은 `skills/references/` 공유 자산과 3종 SKILL.md에만 반영해 중복 없이 유지보수합니다.
