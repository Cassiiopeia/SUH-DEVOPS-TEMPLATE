📝 현재 문제점
---

`template_integrator.sh` / `template_integrator.ps1`의 "어떤 항목을 수정할까요?" 메뉴에서 항목 하나(예: 버전)를 수정하면, 변경 직후 곧바로 **같은 수정 메뉴가 다시 출력**된다.

- 사용자는 방금 바꾼 값이 전체 개요(`🛰️ 프로젝트 분석 결과`)에 어떻게 반영됐는지 확인할 수 없다.
- 여러 항목을 연달아 수정할 때, 현재 프로젝트 타입·버전·브랜치·옵션 워크플로우 상태를 한눈에 다시 보지 못한 채 다음 항목을 고르게 된다.
- 즉, 수정 루프가 "변경 → 개요 확인 → 다음 선택"이 아니라 "변경 → 곧바로 메뉴"로 흘러 현재 상태 파악이 끊긴다.

해당 흐름:
- `template_integrator.sh` → `handle_project_edit_menu()`
- `template_integrator.ps1` → `Edit-ProjectInfo`

🛠️ 해결 방안 / 제안 기능
---

수정 메뉴 루프에서 **항목을 수정할 때마다 `🛰️ 프로젝트 분석 결과` 개요를 다시 출력한 뒤** 수정 메뉴를 보여주도록 개선한다.

- 항목 변경(타입/버전/브랜치/옵션 워크플로우) 후 메뉴를 재출력하기 직전, 현재 확정된 전체 설정 개요를 다시 그린다.
- 사용자는 매 수정 직후 변경이 반영된 최신 상태를 확인하고 다음 항목을 선택할 수 있다.
- `.sh`와 `.ps1` 양쪽 동작을 동일하게 맞춘다.

⚙️ 작업 내용
---

- 분석 결과 개요 출력 로직을 재사용 가능한 단위로 분리(또는 수정 루프에서 호출 가능하게 정리).
- `handle_project_edit_menu()` / `Edit-ProjectInfo` 루프에서 매 반복 시작 시 개요를 출력하도록 수정.
- macOS/Windows 양쪽 검증: Docker PowerShell 파서로 `.ps1` 구문 확인, `expect`로 `.sh` 메뉴 동작 확인.
