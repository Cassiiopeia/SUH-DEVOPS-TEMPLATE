📝 현재 문제점
---

- `suh-issue` 스킬이 이슈를 생성할 때 담당자(assignee)를 자동으로 지정하지 않는다.
- config에 `default_assignee` 설정값이 있는데도 이슈에 담당자가 붙지 않았다. `issue_cli.py`의 이슈 생성 코드가 담당자를 빈 리스트로 하드코딩해 넘겼기 때문이다.
- 즉 설정값을 받아만 두고 실제로 쓰지 않는 "죽은 설정"이었고, 사용자에게 담당자를 제대로 물어보는 흐름도 없었다.
- 다른 옵션(자동 승인 등)은 "글로벌 + 레포별 + 첫 실행만 질문" 패턴으로 잘 동작하는데, 담당자만 이 패턴에서 빠져 있어 일관성이 깨졌다.

🛠️ 해결 방안 / 제안 기능
---

- 담당자 지정을 자동 승인과 **동일한 패턴**으로 통일한다: 글로벌 기본값 + 레포별 오버라이드 + 설정이 없을 때만 첫 이슈에서 1회 질문 후 저장.
- 설정이 있으면 사용자에게 묻지 않고 자동 적용하며, 승인 화면에 담당자를 표시만 한다.
- 설정이 전혀 없을 때만 첫 이슈에서 한 번 묻고 저장해, 이후로는 영구히 자동 적용한다.
- 사용자는 설정 파일을 직접 만지지 않고, 에이전트가 자연어 응답을 받아 대신 저장한다.

⚙️ 작업 내용
---

- (1) `skills/suh-issue/scripts/issue_cli.py`: `create-issue`에 `--assignees` 인자 추가, 담당자 하드코딩 제거. 요청한 담당자가 반영되지 않으면 경고를 출력하되 이슈 생성 자체는 성공으로 처리.
- (2) `scripts/common/gh_client.py`: 이슈 생성 응답에 실제 반영된 담당자 목록을 함께 반환해, 호출자가 요청 대비 결과를 비교·경고할 수 있게 함.
- (3) `skills/suh-issue/SKILL.md`: 담당자 결정(레포별 → 글로벌 우선순위), 승인 화면 담당자 표시, 설정이 없을 때만 1회 질문하는 첫 설정 분기 추가.
- (4) `skills/config.json.example` / `skills/references/config-rules.md`: 담당자 키(`default_assignee`, `repos[].issue.assignee`) 문서화.
