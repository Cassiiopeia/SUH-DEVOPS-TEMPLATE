## 📝 현재 문제점

`template_integrator`(`.sh`/`.ps1`)가 기존 프로젝트에 워크플로우를 통합할 때 **파일명만** 비교한다. 그래서 내용이 한 글자도 안 바뀐 워크플로우도 불필요하게 처리되어 사용자에게 혼란을 준다.

- **공통(common) 워크플로우**: 메뉴 없이 무조건 `cp`로 덮어쓰며 `✓ ... 업데이트`만 출력한다.
- **타입별(spring 등) 워크플로우**: "이미 존재함" 목록에 넣고 3지선 메뉴(`.template` 추가 / 건너뛰기 / `.bak` 백업 후 덮어쓰기)를 띄운다.

실제 사례: `PROJECT-SPRING-GITHUB-PACKAGES-PUBLISH.yml`이 기존 파일과 동일한데도 `.bak`을 만들고 교체하겠다고 물어, **변경점이 0인 파일에 대해서도 "덮어쓸까요?"를 묻는** 상황이 발생한다.

### 왜 단순 비교가 안 되나

워크플로우는 복사 직후 `configure_workflow_env`로 env 토큰이 치환된다.

- 템플릿 원본: `KEY: "__PROJECT_NAME__"  # @wizard ask`, `paths-anchor` 마커 등이 **남은** 상태
- 기존 설치본: 그 자리에 레포명·서버 등 **값이 채워진** 상태

→ 원본 vs 기존본을 그냥 비교하면 **항상 다르게** 나온다. 단순 `cmp`로는 "동일 여부"를 판단할 수 없다.

## 🛠️ 해결 방안

비교 직전에 **템플릿 원본을 임시 사본으로 떠서 실제 치환 로직(`configure_workflow_env`)을 그대로 한 번 적용**한 "설치 예상 최종형"을 만들고, 그 결과를 기존 파일과 비교한다.

> "이 템플릿을 지금 설정대로 깔면 결과가 똑같아지는가?"를 직접 본다. 정규식 마스킹 같은 휴리스틱보다 오판이 거의 없다.

- 가상 치환은 **항상 기본값 모드(`WF_USE_DEFAULTS=true`)** 를 강제하여 사용자에게 다시 질문하지 않게 하고, deploy 기록·CSV 누적 같은 부수효과는 비교 모드 플래그로 격리한다.
- 비교 결과:
  - **동일(unchanged)** → 공통은 `cp` 생략하고 `✓ ... (변경 없음)` 표시, 타입별은 **메뉴 자체를 안 띄움**.
  - **다름(changed)** → 기존 동작 유지(공통 덮어쓰기 / 타입별 메뉴). 메뉴 대상도 `changed`만.
- `changed`가 1개도 없으면(전부 동일) 타입별 메뉴를 통째로 건너뛰고 `ℹ N개 워크플로우가 현재 설정과 동일해 건너뜁니다` 한 줄만 안내한다.
- 비교가 실패하면(임시 폴더 생성 불가 등) **안전하게 `changed`로 fallback** → 기존 동작을 유지하고 업데이트를 절대 놓치지 않는다.
- `.sh`와 `.ps1` **양쪽 동일**하게 적용한다(`.ps1`은 CRLF/LF 정규화 후 문자열 비교).

## ✅ 기대 효과

- 변경점이 없는 워크플로우는 조용히 건너뛰어, "덮어쓸까요?" 류의 불필요한 질문과 의미 없는 `.bak` 생성이 사라진다.
- 실제로 바뀐 워크플로우만 메뉴/업데이트 대상이 되어, 사용자가 무엇이 갱신되는지 명확히 인지한다.
