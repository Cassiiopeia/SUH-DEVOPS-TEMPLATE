📝 현재 문제점
---

`template_integrator`(`.sh`/`.ps1`)로 기존 프로젝트에 배포 워크플로우를 통합할 때, 마법사가 env 값을 채우는 방식에 두 가지 불편이 있습니다.

- **사용처가 안 보임**: `SERVICE_DOMAIN`, `CONTAINER_PORT` 같은 값을 물을 때 그 값이 *어느 워크플로우/어느 타입*에 쓰이는지 화면에 나오지 않아, 사용자가 자기 프로젝트와 무관한 값까지 일일이 판단해야 합니다.
- **Y/N 텍스트 입력의 거친 UX**: "전부 기본값으로 채울까요? (Y/N)" 한 번 물은 뒤 N이면 모든 KEY를 순차로 직접 입력해야 합니다. 일부만 바꾸고 싶어도 전부를 거쳐야 하고, 무엇이 기본값인지 한눈에 미리 볼 수 없습니다.

🛠️ 해결 방안 / 제안 기능
---

타입 루프 진입 전에 전 워크플로우를 한 번 스캔해 ask KEY를 모으고(수집), **기본값 미리보기 표 + 메뉴**를 한 번 보여준 뒤 확정 값을 기존 캐시에 미리 채우는(prefill) 방식으로 바꿉니다.

- **사용처 표시**: 각 env 항목 옆에 `[사용처]`를 붙입니다. 단일 타입이면 용도(예: `무중단배포(Nginx)·무중단배포(Traefik)`), 멀티 타입이면 타입명으로 표기. 파일명→사람말 매핑은 `labels.yml`의 `_workflow_names:` 섹션에서 읽고, 매핑이 없으면 파일명 그대로 폴백합니다.
- **메뉴화**: Y/N 대신 `1=전부 기본값 / 2=하나씩 직접 입력 / 3=몇 개만 골라서 바꾸기(나머지 기본값)` 메뉴를 제공합니다. 3번은 멀티선택으로 바꿀 항목만 고릅니다.

⚙️ 작업 내용
---

- `labels.yml`에 워크플로우 파일명→사람말 매핑(`_workflow_names`) 섹션 추가
- 파일명→사람말 변환 헬퍼 (`wf_workflow_name` / `Get-WfWorkflowName`)
- 전 워크플로우 ask KEY 수집기 (`wf_collect_asks` / `Get-WfAskTable`)
- 사용처 문자열 조립 (`wf_scope_string` / `Get-WfScopeString`)
- 기본값 미리보기 표 + 1/2/3 메뉴 + 멀티선택 + 값 prefill (Y/N 입력 대체)
- "하나씩 입력" 분기 라벨에도 `[사용처]` 표시
- **회귀 가드**: "전부 기본값(1번)" 경로의 최종 토큰 치환 결과가 변경 전과 **byte-identical**임을 검증 (`_wf_is_unchanged` / `Test-WorkflowUnchanged` 재사용)

> `.sh`와 `.ps1`은 1:1 동일 동작 유지. env KEY 이름·`labels.yml`의 label/help/example·워크플로우 YAML의 실행 로직은 건드리지 않고 화면/흐름만 바꿉니다.
