🗒️ 설명
---

`template_integrator.sh`를 **macOS 기본 bash(3.2)**에서 실행하면, 배포 워크플로우의 `@wizard ask:` 환경설정 키가 **여러 개인데도 1개만 수집**된다.

예를 들어 Spring 워크플로우에는 `PROJECT_NAME`, `JAVA_VERSION`, `DEPLOY_PORT`, `SERVICE_DOMAIN`, `VOLUME_HOST_PATH`, `VOLUME_CONTAINER_PATH`, `SSH_AUTH_METHOD` 등 다수의 `@wizard ask:` 키가 있는데, 마법사의 "배포 워크플로우 환경설정" 단계에서 **`PROJECT_NAME`(서비스 식별자) 1개만** 나타나고 나머지는 조용히 누락된다. 통합 자체는 끝까지 완주하지만, env 설정이 1개만 채워진 반쪽짜리 결과가 된다.

**원인**: macOS 기본 `/bin/bash`는 라이선스 문제로 **3.2.57(2007년 버전)에 고정**돼 있고, bash 3.2는 **연관배열(`declare -A`)을 지원하지 않는다.** 스크립트는 env 키를 연관배열(`WF_ASK_DEFAULT`, `WF_ASK_SCOPE`, `WF_ASK_TYPE_DEFAULT`, `WF_ASK_FILES`, `WF_WFNAME_VAL`)에 담는데, bash 3.2에서는 이 배열의 모든 문자열 키가 **인덱스 0 하나로 뭉개진다.** 그래서 처음 만난 키(`PROJECT_NAME`)만 남고 나머지가 전부 덮어써진다.

**영향 범위**:
- macOS 기본 bash(3.2)를 쓰는 모든 사용자 — `bash <(curl ...)` 실행 시 발생
- 윈도우(`template_integrator.ps1`)는 PowerShell 해시테이블이라 정상
- 윈도우 Git Bash는 bash 4+라 정상
- 즉 **macOS 기본 bash 3.2에서만 발생하는 포팅 버그**

📸 참고 자료
---

bash 3.2 실측 (Spring 워크플로우 1개, `@wizard ask:` 마커 6개):

```
════════ 수정 전 ════════
키 개수=1
키 목록=PROJECT_NAME

════════ 수정 후 ════════
키 개수=6
키 목록=PROJECT_NAME JAVA_VERSION DEPLOY_PORT VOLUME_HOST_PATH VOLUME_CONTAINER_PATH SSH_AUTH_METHOD
```

🔄 재현 방법
---

1. macOS(기본 `/bin/bash` 3.2)에서 Spring 등 배포 워크플로우가 있는 멀티타입 프로젝트로 이동
2. 마법사 실행:
   ```
   bash <(curl -fsSL "https://raw.githubusercontent.com/Cassiiopeia/SUH-DEVOPS-TEMPLATE/main/template_integrator.sh")
   ```
3. 전체 설치 진행 → "배포 워크플로우 환경설정을 채웁니다" 단계 도달
4. 항목이 `(1/1) 서비스 식별자` **1개만** 표시됨 (실제로는 6개 이상이어야 함)

✅ 예상 동작
---

- macOS 기본 bash(3.2)에서도 워크플로우의 모든 `@wizard ask:` 키를 빠짐없이 수집해야 한다.
- Spring 단일 워크플로우면 `PROJECT_NAME`, `JAVA_VERSION`, `DEPLOY_PORT`, `VOLUME_HOST_PATH`, `VOLUME_CONTAINER_PATH`, `SSH_AUTH_METHOD` 6개가 모두 환경설정 단계에 나타나야 한다.
- 멀티타입(spring/flutter/react/python)이면 각 타입의 키가 모두 합쳐져 표시돼야 한다.
- 연관배열을 bash 3.2/4 양쪽에서 동작하는 방식으로 대체해야 한다.
