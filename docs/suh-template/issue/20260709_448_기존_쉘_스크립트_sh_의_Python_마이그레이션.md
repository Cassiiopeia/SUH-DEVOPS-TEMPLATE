# 🚀[기능개선][CI/CD] 기존 쉘 스크립트(.sh)의 Python 마이그레이션

📝 현재 문제점
---

- 기존 빌드, 릴리즈, 버전 동기화 등의 자동화 유틸이 Bash 쉘 스크립트(`.sh`)로 작성되어 있어 Windows 환경에서 독립적인 파워셸이나 CMD 등에서 바로 실행하기 어렵고, Git Bash나 WSL을 반드시 거쳐야 하는 인프라 제약이 존재합니다.
- 다양한 운영체제(Windows, macOS, Linux) 환경에서 개발 도구와 CI/CD 도구가 원활하고 일관되게 작동할 수 있도록, 시스템 호출 및 파일 경로 처리가 크로스 플랫폼에 최적화된 Python 스크립트(`.py`)로의 단일화가 절실히 요구됩니다.

🛠️ 해결 방안 / 제안 기능
---

- 프로젝트 내에 산재된 쉘 스크립트(`.sh`)를 면밀히 분석하고, 동일한 기능을 수행하는 고품질 Python 스크립트(`.py`)로 전면 포팅 및 리팩토링합니다.
- 주요 마이그레이션 대상:
  - `template_integrator.sh` -> `template_integrator.py`
  - `.github/scripts/template_initializer.sh` -> `.github/scripts/template_initializer.py`
  - `.github/scripts/truncate_release_notes.sh` -> `.github/scripts/truncate_release_notes.py`
  - `.github/scripts/version_manager.sh` -> `.github/scripts/version_manager.py`
  - `.github/util/...` 하위 버전 동기화 및 마법사 쉘 스크립트 일체
- 각 스크립트 마이그레이션 시, OS 독립적인 `pathlib`, `subprocess`, `sys` 모듈 등을 적극적으로 활용하여 안전한 시스템 오퍼레이션을 보장합니다.
- GitHub Actions 워크플로우(`.github/workflows/*.yml`)에서도 파이썬 실행을 주입하여 빌드/테스트/배포 파이프라인의 완성도를 높입니다.
