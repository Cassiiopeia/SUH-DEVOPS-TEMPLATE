## 개요

skill 내부 Python 실행 구조를 **3-layer 아키텍처**로 완전 재설계.

기존 단일 `scripts/suh_template/suh_command.py` (1,900줄, 19개 서브커맨드) 폐기 → **7개 skill이 각자 자체 `_cli.py` 보유** + **`scripts/common/` 공유 인프라** 패턴.

- Closes #322

## 변경 사항

### Layer 1 — `scripts/common/` (NEW)

공유 인프라 9개 모듈. skill 의존성 0인 순수 도메인 로직.

- `gh_client.py` — GitHub API HTTP
- `gh_branch.py` — 브랜치명·커밋 템플릿
- `paths.py` — 출력 경로 계산
- `title.py` — 제목 정규화
- `issue_number.py` — 이슈 번호 추출
- `config.py` — config.json 로드 + `get_github_pat`

- `bootstrap.py` — **NEW** sys.path 부트스트랩 헬퍼

### Layer 2 — `skills/<skill>/scripts/<scope>_cli.py` (NEW)

### Layer 3 — SKILL.md

7개 SKILL.md 호출 패턴 = **self-contained 5줄 표준**:

```bash
PROJECT_ROOT=$(git rev-parse --show-toplevel 2>/dev/null || pwd)
PYTHON=$(for _py in python3 python; do _path=$(command -v "$_py" 2>/dev/null) || continue; "$_path" -c "import sys; sys.exit(0)" 2>/dev/null && echo "$_path" && break; done)
[ -z "$PYTHON" ] && { echo "Python not found"; exit 1; }
cd "$PROJECT_ROOT/skills/<skill>/scripts" || exit 1
PYTHONIOENCODING=utf-8 "$PYTHON" <scope>_cli.py <subcommand> [args]
```

매 블록 self-contained — agent가 어느 코드블록부터 시작해도 동작 보장.

## 다양성·확장성

spec §13.5 7가지 확장성 고려:

- 신규 skill 추가 = py 1개 + SKILL.md 1개 (references·common 손대지 않음)
- 신규 서브커맨드 = 함수 1개 + subparser 1개
- 신규 공통 모듈 = `scripts/common/<new>.py` 추가
- 외부 시스템 통합(GitLab/Jira) = `scripts/common/<provider>_client.py` 추가
- multi-profile config 향후 지원 (`SUH_PROFILE` 환경변수)
- 출력 포맷 확장 (`--format json|text|yaml`) 가능
- 테스트 인프라 = skill 옆 `tests/` 폴더 표준
