### 🔧 주요 변경사항 상세

#### report.md 커맨드 구조
두 플랫폼 모두에서 동일한 기능을 제공하도록 동일한 내용으로 구현

**핵심 기능**:
- Git diff 분석 또는 이슈 내용 분석을 통한 구현 내용 파악
- 효율적인 분석 프로세스: `git status`로 변경 파일 확인 후 이슈 기반 파일 선별
- Git 명령어 최소화 원칙 준수로 토큰 절약
- 자연스럽고 읽기 쉬운 형식의 보고서 생성

**보고서 생성 프로세스**:
1. 변경사항 분석: `git status`로 변경 파일 확인, 이슈 기반 파일 선별
2. 구현 내용 파악: 버그 수정/기능 구현/리팩토링 타입별 분석
3. 보고서 작성: 간결하고 명확한 문체로 작성

**보고서 형식**:
- 작업 개요
- 문제 분석 또는 구현 목표
- 구현 내용 (주요 변경사항별 상세 설명)
- 주요 변경사항 상세
- 의존성 변경 (있는 경우)
- 테스트 및 검증
- 참고사항

**보고서 파일 저장**:
- 위치: `.report/` 디렉토리
- 파일명 형식: `[YYYYMMDD]_[ISSUE#]_[간단한설명].md`
- 예시: `20251212_#113_이슈에_대한_구현_내용_정리_커스텀_커맨드_추가_필요.md`

**특이사항**:
- 효율적 분석 원칙: `git status` 한 번만 실행 후 파일 직접 읽어 분석
- 이슈와 관련 없는 파일은 분석 생략하여 토큰 절약
- 능동태 사용, 키워드 기반 문장으로 가독성 향상
- 버그 수정 및 기능 구현 보고서 예시 포함하여 사용 가이드 제공
