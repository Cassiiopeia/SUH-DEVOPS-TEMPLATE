### 📌 작업 개요
자동 머지 기능에 폴백 전략 구현. PAT 토큰 없이도 Branch Protection이 없는 레포지토리에서 기본 GITHUB_TOKEN으로 정상 동작하도록 개선

**보고서 파일**: `.report/20251212_#117_자동_머지시_커스텀_PAT토큰_없이도_권한_있는_레포는_기본_토큰으로_정상_동작하도록_수정.md`

### 🎯 구현 목표
개인 레포지토리나 Branch Protection이 설정되지 않은 레포지토리에서 별도 PAT 토큰 없이도 자동 머지가 정상 동작하도록 폴백 전략 구현

### 🔍 문제 분석
기존 코드는 무조건 PAT 토큰(`_GITHUB_PAT_TOKEN`)을 사용하여 PR 머지를 시도함
- PAT 토큰이 없으면 워크플로우 실행 시점에 실패
- Branch Protection이 없는 개인 레포지토리에서도 불필요하게 PAT 토큰 등록 필요
- 개발 편의성 저하 및 초기 설정 복잡도 증가

### ✅ 구현 내용

#### 자동 머지 폴백 전략 구현
- **파일**: 
  - `.github/workflows/PROJECT-COMMON-AUTO-CHANGELOG-CONTROL.yaml`
  - `.github/workflows/project-types/common/PROJECT-COMMON-AUTO-CHANGELOG-CONTROL.yaml`
- **변경 내용**: 3단계 폴백 전략으로 자동 머지 로직 개선
- **이유**: PAT 토큰 없이도 가능한 환경에서는 기본 토큰으로 동작하도록 하여 사용자 편의성 향상

#### PAT 토큰 존재 여부 확인 로직 추가
- **파일**: 위 동일
- **변경 내용**: Secret 존재 여부를 확인하여 `PAT_AVAILABLE` 변수 설정
- **이유**: PAT 토큰 사용 가능 여부를 사전에 판단하여 불필요한 시도 방지

#### 1차 시도: GITHUB_TOKEN으로 머지 시도
- **파일**: 위 동일
- **변경 내용**: `--admin` 플래그 없이 기본 GITHUB_TOKEN으로 머지 시도
- **이유**: Branch Protection이 없거나 GitHub Actions bot이 예외로 설정된 경우 성공 가능

#### 2차 시도: PAT 토큰으로 재시도
- **파일**: 위 동일
- **변경 내용**: 1차 시도 실패 시 PAT 토큰으로 `--admin` 플래그 사용하여 재시도
- **이유**: 보호된 브랜치에서도 관리자 권한으로 머지 가능하도록 구현

#### 최종 에러 처리 및 사용자 안내
- **파일**: 위 동일
- **변경 내용**: 실패 시 PAT 토큰 설정 방법 또는 Branch Protection Rules 조정 방법 안내
- **이유**: 사용자가 문제를 해결할 수 있도록 명확한 가이드 제공

### 🔧 주요 변경사항 상세

#### 자동 PR Merge 단계 개선
기존에는 `env` 섹션에서 PAT 토큰을 직접 지정하고 `--admin` 플래그로 머지 시도

**변경 후 동작 흐름**:
1. PAT 토큰 존재 여부 확인 (`-z` 체크)
2. PR 상태 확인 (이미 머지되었는지 체크)
3. 1차 시도: GITHUB_TOKEN으로 머지 (`--admin` 없이)
4. 실패 시 2차 시도: PAT 토큰으로 머지 (`--admin` 사용)
5. 모두 실패 시 에러 메시지와 해결 방법 안내

**특이사항**:
- 각 시도 후 PR 상태를 재확인하여 다른 프로세스가 머지했는지 체크
- 머지 출력 메시지를 파싱하여 성공 여부 정확히 판단
- PAT 토큰 없음과 PAT 토큰으로도 실패한 경우를 구분하여 다른 에러 메시지 제공

#### 에러 처리 강화
- 종료 코드와 출력 메시지 모두 확인하여 정확한 성공/실패 판단
- PR 상태 재확인으로 중복 머지 방지
- 실패 시 PR에 자동으로 댓글 추가하여 사용자에게 알림

### 📦 의존성 변경
- 없음 (기존 GitHub CLI만 사용)

### 🧪 테스트 및 검증
다음 시나리오에서 동작 확인 필요:

1. **Branch Protection 없는 레포지토리 + PAT 없음**
   - GITHUB_TOKEN으로 머지 성공 예상
   
2. **Branch Protection 있는 레포지토리 + PAT 있음**
   - GITHUB_TOKEN 실패 → PAT로 성공 예상
   
3. **Branch Protection 있는 레포지토리 + PAT 없음**
   - GITHUB_TOKEN 실패 → 에러 메시지 및 해결 방법 안내
   
4. **PR이 이미 머지된 경우**
   - 조기 종료하여 중복 시도 방지

### 📌 참고사항
- Branch Protection Rules에서 "Allow specified actors to bypass required pull requests" 옵션으로 GitHub Actions bot을 예외로 추가하면 PAT 토큰 없이도 동작 가능
- PAT 토큰은 `repo` 권한 전체가 필요하며, Classic 토큰 사용 권장
- 워크플로우 실행 로그에서 각 단계별 시도 결과와 에러 메시지 확인 가능
