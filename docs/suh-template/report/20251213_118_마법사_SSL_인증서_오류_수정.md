### 📌 작업 개요
마법사 설치 명령어 실행 시 발생하는 `curl: (77) error setting certificate verify locations: CAfile: /tmp/cacert.pem CApath: none` 오류 수정

**이슈**: #118

### 🔍 문제 분석
사용자 환경에서 curl의 CA 인증서 경로가 잘못 설정된 경우 발생하는 문제.
원인: `~/.curlrc` 파일이나 환경 변수(`CURL_CA_BUNDLE`, `SSL_CERT_FILE` 등)에 존재하지 않는 CA 파일 경로(`/tmp/cacert.pem`)가 설정됨

**재현 명령어**:
```bash
bash <(curl -fsSL "https://raw.githubusercontent.com/Cassiiopeia/SUH-DEVOPS-TEMPLATE/main/template_integrator.sh")
```

**오류 메시지**:
```
curl: (77) error setting certificate verify locations: CAfile: /tmp/cacert.pem CApath: none
```

### ✅ 구현 내용

#### SSL 환경 변수 초기화 코드 추가 (Bash)
- **파일**: `template_integrator.sh`
- **위치**: 64-72줄 (`set -e` 다음)
- **변경 내용**: 스크립트 시작 시 SSL 관련 환경 변수를 초기화하여 사용자 설정 무시
- **이유**: 사용자 환경에 잘못된 CA 경로가 설정되어 있어도 스크립트가 정상 동작하도록 방어적 처리

#### SSL 환경 변수 초기화 코드 추가 (PowerShell)
- **파일**: `template_integrator.ps1`
- **위치**: 84-92줄 (`[Console]::OutputEncoding` 다음)
- **변경 내용**: PowerShell에서도 동일하게 SSL 환경 변수 초기화
- **이유**: Windows 환경에서도 동일한 문제 발생 가능성 대비

### 🔧 주요 변경사항 상세

#### template_integrator.sh
```bash
# ===================================================================
# SSL 인증서 관련 환경 변수 초기화
# 사용자 환경에서 잘못 설정된 CA 경로 문제 방지
# (예: curl: (77) error setting certificate verify locations: CAfile: /tmp/cacert.pem)
# ===================================================================
unset CURL_CA_BUNDLE
unset SSL_CERT_FILE
unset SSL_CERT_DIR
unset REQUESTS_CA_BUNDLE
```

#### template_integrator.ps1
```powershell
# ===================================================================
# SSL 인증서 관련 환경 변수 초기화
# 사용자 환경에서 잘못 설정된 CA 경로 문제 방지
# (예: curl: (77) error setting certificate verify locations: CAfile: /tmp/cacert.pem)
# ===================================================================
$env:CURL_CA_BUNDLE = $null
$env:SSL_CERT_FILE = $null
$env:SSL_CERT_DIR = $null
$env:REQUESTS_CA_BUNDLE = $null
```

**특이사항**:
- 환경 변수 초기화는 스크립트 내에서만 적용됨 (사용자 시스템 설정에 영향 없음)
- `REQUESTS_CA_BUNDLE`도 함께 초기화하여 Python requests 라이브러리 사용 시에도 동일 문제 방지

### 🧪 테스트 및 검증
- 잘못된 `CURL_CA_BUNDLE` 환경 변수가 설정된 환경에서 스크립트 실행 테스트
- macOS, Linux 환경에서 정상 동작 확인 필요

### 📌 참고사항
- 근본적인 원인은 사용자 시스템의 curl 설정 문제
- 이 수정은 스크립트 레벨의 방어적 조치로, 사용자가 `~/.curlrc` 파일을 직접 수정하거나 환경 변수를 확인하는 것이 근본적인 해결책
