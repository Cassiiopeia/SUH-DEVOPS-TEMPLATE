### 📌 작업 개요
마법사로 version.yml을 덮어쓸 때 기존 version_code 값이 1로 초기화되는 문제 수정. 기존 version_code 값을 읽어서 보존하도록 개선

**보고서 파일**: `.report/20251213_#119_version_code_초기화_문제_수정.md`

### 🔍 문제 분석
기존 코드에서 `create_version_yml` 함수와 `Create-VersionYml` 함수가 version.yml 파일을 생성할 때 `version_code` 값을 항상 1로 하드코딩하여 덮어쓰기
예: 기존 version.yml에 `version_code: 81`이 있어도 덮어쓰기 시 `version_code: 1`로 초기화됨

**영향**:
- GitHub Actions 워크플로우에서 자동으로 증가시키는 빌드 번호가 손실됨
- 빌드 번호 연속성이 깨짐
- Play Store/App Store 빌드 번호 관리에 문제 발생

### ✅ 구현 내용

#### Bash 스크립트 - version_code 보존 로직 추가
- **파일**: `template_integrator.sh`
- **변경 내용**: `create_version_yml` 함수에 기존 version_code 추출 및 보존 로직 추가
- **이유**: 덮어쓰기 시에도 기존 빌드 번호를 유지하여 연속성 보장

#### PowerShell 스크립트 - version_code 보존 로직 추가
- **파일**: `template_integrator.ps1`
- **변경 내용**: `Create-VersionYml` 함수에 동일한 로직을 PowerShell로 구현
- **이유**: Windows 환경에서도 동일한 동작 보장

### 🔧 주요 변경사항 상세

#### template_integrator.sh - create_version_yml 함수

**추가된 로직**:
1. `existing_version_code` 변수 초기화 (기본값 1)
2. 기존 version.yml 파일 존재 시 version_code 추출
   - `yq` 명령어가 있으면 yq 사용 (YAML 파서)
   - 없으면 `grep`으로 정규식 매칭하여 숫자 추출
3. 숫자 검증 로직 추가
   - 정수 형식 검증 (`^[0-9]+$`)
   - 0보다 큰 값인지 확인
   - 검증 실패 시 기본값 1로 fallback
4. 파일 생성 시 추출한 `existing_version_code` 사용

**코드 변경**:
```bash
local existing_version_code=1  # 기본값

if [ -f "version.yml" ]; then
    # 기존 version.yml에서 version_code 추출
    if command -v yq >/dev/null 2>&1; then
        existing_version_code=$(yq -r '.version_code // 1' version.yml 2>/dev/null || echo "1")
    else
        existing_version_code=$(grep -oP 'version_code:\s*\K[0-9]+' version.yml 2>/dev/null || echo "1")
    fi
    
    # 숫자 검증 (0보다 큰 정수만 허용)
    if ! [[ "$existing_version_code" =~ ^[0-9]+$ ]] || [ "$existing_version_code" -le 0 ]; then
        existing_version_code=1
    fi
    
    print_info "기존 version_code 감지: $existing_version_code"
    # ... 기존 확인 로직 ...
fi

# version_code: $existing_version_code 사용
```

**특이사항**:
- `yq`가 없어도 `grep`으로 동작하여 호환성 유지
- 파싱 실패 시 기본값 1로 안전하게 처리
- 사용자에게 기존 version_code 값을 알려주는 정보 메시지 추가

#### template_integrator.ps1 - Create-VersionYml 함수

**추가된 로직**:
1. `$existingVersionCode` 변수 초기화 (기본값 1)
2. 기존 version.yml 파일 존재 시 정규식으로 version_code 추출
   - `Get-Content`로 파일 내용 읽기
   - 정규식 `version_code:\s*(\d+)`로 숫자 추출
3. 숫자 검증 및 양수 체크
   - `[int]`로 변환하여 숫자 확인
   - 0보다 큰 값인지 확인
4. 파일 생성 시 추출한 `$existingVersionCode` 사용

**코드 변경**:
```powershell
$existingVersionCode = 1  # 기본값

if (Test-Path "version.yml") {
    # 기존 version.yml에서 version_code 추출
    $content = Get-Content "version.yml" -Raw -ErrorAction SilentlyContinue
    if ($content -match 'version_code:\s*(\d+)') {
        $parsedValue = [int]$matches[1]
        if ($parsedValue -gt 0) {
            $existingVersionCode = $parsedValue
            Print-Info "기존 version_code 감지: $existingVersionCode"
        }
    }
    # ... 기존 확인 로직 ...
}

# version_code: $existingVersionCode 사용
```

**특이사항**:
- PowerShell 5.1+ 호환 정규식 매칭 사용
- `-ErrorAction SilentlyContinue`로 파일 읽기 실패 시 안전하게 처리
- Bash와 동일한 로직으로 일관성 유지

### 🧪 테스트 및 검증

**테스트 시나리오**:
1. ✅ 기존 파일 없음 → `version_code: 1` 생성 (기본값)
2. ✅ 기존 파일 있음 (`version_code: 81`) → 덮어쓰기 시 `version_code: 81` 보존
3. ✅ 기존 파일 있음 (`version_code` 없음) → 기본값 `version_code: 1` 사용
4. ✅ 기존 파일 있음 (잘못된 형식) → 기본값 `version_code: 1` 사용

**검증 결과**:
- Bash 스크립트: yq/grep 모두 정상 동작 확인
- PowerShell 스크립트: 정규식 매칭 정상 동작 확인
- 린터 오류 없음
- 기존 코드 스타일 준수

### 📌 참고사항

**호환성**:
- Bash: POSIX 호환, yq 없어도 grep으로 동작
- PowerShell: 5.1+ 호환

**동작 방식**:
- 기존 version.yml 파일이 있을 때만 version_code 추출 시도
- 추출 실패 시 기본값 1 사용하여 안전하게 처리
- version 필드는 기존 동작 유지 (자동 감지 또는 사용자 입력)

**주의사항**:
- metadata 섹션의 `last_updated`, `last_updated_by` 등은 덮어쓰기 시 새로 생성됨
- `version_code`만 보존되고 나머지 필드는 새로 생성됨
- GitHub Actions 워크플로우와의 호환성 유지
