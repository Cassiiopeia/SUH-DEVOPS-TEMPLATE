# 버그 수정 보고서: version_code 초기화 문제

## 📌 작업 개요

기존 프로젝트에서 template_integrator 스크립트 재실행 시 `version.yml`의 `version_code`가 기존 값(예: 58)에서 1로 초기화되는 버그 수정

**이슈**: #126
**수정 파일**: `template_integrator.ps1`, `template_integrator.sh`

---

## 🔍 문제 분석

### 현상
- 기존 프로젝트에 `version_code: 58`이 있는 상태에서 스크립트 재실행
- 로그에 "기존 version_code 감지: 1"로 표시
- 덮어쓰기 선택 시 `version_code: 1`로 초기화됨

### 근본 원인
`version.yml` 파일 내 **주석에 있는 `version_code: 1`을 실제 값으로 오인**하는 정규식 매칭 문제

#### version.yml 구조
```yaml
# 2. version_code: 1 - Play Store/App Store 빌드 번호 (1부터 자동 증가)  ← 주석 (라인 10)
...
version_code: 58 # app build number  ← 실제 값 (라인 36)
```

#### 기존 코드 문제점 (PowerShell)
```powershell
$content = Get-Content "version.yml" -Raw
if ($content -match 'version_code:\s*(\d+)') {
    # 전체 파일에서 첫 번째 매칭 → 주석의 "version_code: 1"이 먼저 매칭됨!
```

#### 기존 코드 문제점 (Bash)
```bash
existing_version_code=$(grep -oP 'version_code:\s*\K[0-9]+' version.yml)
# 마찬가지로 주석 라인이 먼저 매칭됨
```

## ✅ 구현 내용

### 1. PowerShell 스크립트 수정

**파일**: `template_integrator.ps1` (라인 800-818)

**변경 전**:
```powershell
$content = Get-Content "version.yml" -Raw -ErrorAction SilentlyContinue
if ($content -match 'version_code:\s*(\d+)') {
    $parsedValue = [int]$matches[1]
```

**변경 후**:
```powershell
# 주석이 아닌 실제 데이터 라인에서만 추출 (주석 내 'version_code: 1' 오탐지 방지)
$lines = Get-Content "version.yml" -ErrorAction SilentlyContinue
foreach ($line in $lines) {
    # 주석 라인 건너뛰기 (# 으로 시작하는 라인)
    if ($line -match '^\s*#') {
        continue
    }
    # version_code 값 추출 (라인 시작부터 매칭)
    if ($line -match '^version_code:\s*(\d+)') {
        $parsedValue = [int]$matches[1]
        if ($parsedValue -gt 0) {
            $existingVersionCode = $parsedValue
            Print-Info "기존 version_code 감지: $existingVersionCode"
        }
        break
    }
}
```

**변경 이유**:
- 라인 단위로 처리하여 주석 라인(`#`으로 시작) 건너뛰기
- `^version_code:`로 라인 시작부터 매칭하여 정확한 값만 추출

---

### 2. Bash 스크립트 수정

**파일**: `template_integrator.sh` (라인 898-906)

**변경 전**:
```bash
existing_version_code=$(grep -oP 'version_code:\s*\K[0-9]+' version.yml 2>/dev/null || echo "1")
```

**변경 후**:
```bash
# grep: 주석(#)으로 시작하지 않는 라인에서만 version_code 추출
existing_version_code=$(grep -E '^version_code:\s*[0-9]+' version.yml 2>/dev/null | grep -oP '\d+' | head -1 || echo "1")
```

**변경 이유**:
- `^version_code:`로 라인 시작부터 매칭 (주석 라인 자동 제외)
- 파이프로 숫자만 추출 후 첫 번째 값 사용

## 🧪 테스트 및 검증

### 테스트 케이스

| 케이스 | 입력 | 기대값 | 결과 |
|--------|------|--------|------|
| 주석에 `version_code: 1`, 실제 `version_code: 58` | 기존 파일 | 58 | ✅ 58 반환 |
| 주석 없이 `version_code: 100` | 새 파일 | 100 | ✅ 100 반환 |
| version.yml 없음 | - | 1 (기본값) | ✅ 1 반환 |

### 검증 방법
1. `version_code: 58`이 있는 기존 프로젝트에서 스크립트 실행
2. "기존 version_code 감지: 58"로 표시되는지 확인
3. 덮어쓰기 후에도 `version_code: 58` 유지되는지 확인

## 📌 참고사항

- PowerShell과 Bash 모두 동일한 로직으로 수정하여 일관성 유지
- yq 명령어 사용 시(Bash)는 YAML 파서가 주석을 무시하므로 이 문제 발생하지 않음
- 향후 비슷한 YAML 파싱 이슈 발생 시 동일한 패턴 적용 권장
