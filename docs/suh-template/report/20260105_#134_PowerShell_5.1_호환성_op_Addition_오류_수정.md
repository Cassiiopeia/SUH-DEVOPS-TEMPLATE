### 📌 작업 개요
Windows PowerShell 5.1 환경에서 템플릿 설치 중 `[System.IO.FileInfo] op_Addition` 오류 발생 문제 수정

**이슈**: #134
**원인**: PowerShell 5.1과 7+ 간의 타입 변환 호환성 차이
**해결**: 배열 초기화 안전 처리 및 문자열 연결 시 명시적 타입 변환 적용

### 🔍 문제 분석

**오류 메시지**:
```
iex : [System.IO.FileInfo]에 이름이 'op_Addition'인 메서드가 없으므로 메서드를 호출하지 못했습니다.
```

**발생 시점**: "spring 전용 워크플로우 다운로드 중..." 단계

**근본 원인**:
1. **배열 연결 문제**: `Get-ChildItem` 결과가 `$null`일 때 `$workflows += ...` 연산 실패
2. **문자열 연결 문제**: `"$destPath.bak"` 형식에서 `$destPath`가 암시적 변환되지 않음
3. **PowerShell 버전 차이**: PowerShell 7+는 자동 변환 지원, 5.1은 `op_Addition` 메서드 호출 시도 후 실패

### ✅ 구현 내용

#### 1. 공통 워크플로우 배열 초기화 수정
- **파일**: `template_integrator.ps1` (라인 921-927)
- **변경 내용**: 배열을 먼저 `@()`로 초기화 후 조건부 추가
- **이유**: `$null += $item` 연산 시 PowerShell 5.1에서 오류 발생 방지

```powershell
# Before (문제 코드)
$workflows = Get-ChildItem -Path $commonDir -Filter "*.yaml"
$workflows += Get-ChildItem -Path $commonDir -Filter "*.yml"

# After (수정 코드)
$workflows = @()
$yamlFiles = Get-ChildItem -Path $commonDir -Filter "*.yaml" -ErrorAction SilentlyContinue
$ymlFiles = Get-ChildItem -Path $commonDir -Filter "*.yml" -ErrorAction SilentlyContinue
if ($yamlFiles) { $workflows += $yamlFiles }
if ($ymlFiles) { $workflows += $ymlFiles }
```

#### 2. 공통 워크플로우 백업 경로 수정
- **파일**: `template_integrator.ps1` (라인 933-937)
- **변경 내용**: 백업 경로 생성 시 명시적 `[string]` 타입 캐스팅 적용
- **이유**: `FileInfo` 객체와 문자열 연결 시 `op_Addition` 오류 방지

```powershell
# Before (문제 코드)
Move-Item -Path $destPath -Destination "$destPath.bak" -Force

# After (수정 코드)
$backupPath = [string]$destPath + ".bak"
Move-Item -Path $destPath -Destination $backupPath -Force
```

#### 3. 타입별 워크플로우 배열 초기화 수정
- **파일**: `template_integrator.ps1` (라인 953-958)
- **변경 내용**: 공통 워크플로우와 동일한 패턴 적용
- **이유**: Spring, Flutter 등 타입별 워크플로우 다운로드 시에도 동일 오류 발생 가능

#### 4. 타입별 워크플로우 백업 경로 수정
- **파일**: `template_integrator.ps1` (라인 964-968)
- **변경 내용**: 명시적 문자열 변환 적용
- **이유**: 기존 워크플로우 백업 시 오류 방지

#### 5. CodeRabbit 설정 백업 경로 수정
- **파일**: `template_integrator.ps1` (라인 1117-1125)
- **변경 내용**: 백업 경로를 변수로 분리하여 안전하게 처리
- **이유**: `.coderabbit.yaml.bak` 생성 시 동일한 호환성 문제 방지

```powershell
# Before (문제 코드)
Copy-Item -Path ".coderabbit.yaml" -Destination ".coderabbit.yaml.bak" -Force

# After (수정 코드)
$backupPath = ".coderabbit.yaml.bak"
Copy-Item -Path ".coderabbit.yaml" -Destination $backupPath -Force
```

### 🔧 주요 변경사항 상세

#### Copy-Workflows 함수
`Get-ChildItem` 결과를 바로 `+=` 연산하지 않고, 먼저 빈 배열 `@()`로 초기화 후 조건부 추가하도록 변경.
백업 파일 생성 시 `[string]$destPath + ".bak"` 형식으로 명시적 타입 변환 적용.

**특이사항**:
- PowerShell 5.1에서 `$null += $item`은 오류 발생
- PowerShell 7+에서는 정상 동작하지만, 5.1 호환성 위해 안전한 패턴 사용

#### Copy-CodeRabbitConfig 함수
백업 경로를 변수로 분리하여 문자열 연결 문제 방지.
`-Force` 모드와 일반 모드 모두에서 동일한 패턴 적용.
