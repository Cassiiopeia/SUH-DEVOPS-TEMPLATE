### 📌 작업 개요
GitHub `.github/util/` 하위의 여러 Util 모듈(playstore-wizard, testflight-wizard 등)의 버전 관리를 자동화하는 워크플로우 구현. 각 모듈의 `version.json` 수정 시 해당 HTML 파일의 버전 정보가 자동으로 동기화됨.

**보고서 파일**: `.report/20260108_#122_Util_모듈_버전_통합관리_워크플로우.md`

### 🎯 구현 목표
- Util 모듈 확장 대비 버전 관리 자동화 필요
- `version.json` 수정만으로 HTML 내 버전 정보 자동 동기화
- 프로젝트 루트 `version.yml` 수정 시와의 충돌 방지

### ✅ 구현 내용

#### GitHub Actions 워크플로우 추가
- **파일**: `.github/workflows/PROJECT-COMMON-TEMPLATE-UTIL-VERSION-SYNC.yml`
- **변경 내용**: Util 모듈 버전 동기화 전용 워크플로우 신규 생성
- **이유**: 여러 Util 모듈의 버전 관리를 통합적으로 자동화

### 🔧 주요 변경사항 상세

#### 워크플로우 트리거 조건
```yaml
on:
  push:
    branches: [ main ]
    paths:
      - '.github/util/**/version.json'
  workflow_dispatch:
```
- `.github/util/` 하위 모든 디렉토리의 `version.json` 변경 시 자동 실행
- 수동 실행(`workflow_dispatch`)도 지원

#### VERSION-CONTROL 워크플로우와 충돌 방지
```yaml
concurrency:
  group: version-sync
  cancel-in-progress: false
```
- `version-sync` 그룹으로 동시 실행 방지
- 루트 `version.yml` 변경 시 실행되는 VERSION-CONTROL과의 충돌 방지

#### 동적 스크립트 탐색 및 실행
```bash
SYNC_SCRIPTS=$(find .github/util -name "version-sync.sh" -type f 2>/dev/null || true)
```
- `.github/util/` 하위의 모든 `version-sync.sh` 스크립트 자동 탐색
- 새로운 Util 모듈 추가 시 별도 설정 없이 자동 적용
- 각 스크립트 실행 결과(성공/실패) 개별 출력

#### 변경사항 자동 커밋
```bash
git commit -m "chore: github workflow util module version sync [skip ci]"
```
- 변경사항 발생 시에만 커밋
- `[skip ci]` 태그로 무한 루프 방지
- 커밋 전 최신 main 브랜치 rebase로 충돌 방지

### 📦 현재 적용된 Util 모듈

| 모듈명 | 위치 | 버전 |
|--------|------|------|
| playstore-wizard | `.github/util/flutter/playstore-wizard/` | 1.0.0 |
| testflight-wizard | `.github/util/flutter/testflight-wizard/` | (별도 확인) |

**특이사항**:
- 각 모듈에는 `version.json`, `version-sync.sh`, HTML 파일이 필수로 포함
- `version-sync.sh`는 Python을 사용하여 JSON → HTML 동기화 수행
- 정규식으로 HTML 내 `<script id="versionJson">` 영역 교체

### 🔄 동작 흐름

```
version.json 수정
       │
       ▼
 main 브랜치 푸시
       │
       ▼
TEMPLATE-UTIL-VERSION-SYNC 워크플로우 트리거
       │
       ▼
.github/util/ 하위 모든 version-sync.sh 탐색
       │
       ▼
각 스크립트 실행 (version.json → HTML 동기화)
       │
       ▼
변경사항 있으면 자동 커밋 & 푸시
```

### 🧪 테스트 및 검증
- `version.json` 버전 수정 후 main 푸시 시 워크플로우 자동 실행 확인
- HTML 파일 내 버전 정보 자동 업데이트 확인
- VERSION-CONTROL 워크플로우와 동시 실행 시 충돌 없음 확인

### 📌 참고사항
- 새로운 Util 모듈 추가 시: 해당 디렉토리에 `version.json`, `version-sync.sh` 포함 필요
- `version-sync.sh`는 Python 3 환경 필요 (워크플로우에서 Python 3.11 설정)
- 커밋 시 `[skip ci]` 태그 자동 포함으로 재귀 실행 방지
