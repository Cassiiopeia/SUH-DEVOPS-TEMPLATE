### 📌 작업 개요

Template Integrator 업데이트 시 기존 프로젝트의 커스텀 CICD 워크플로우가 무조건 덮어씌워지는 문제 해결.
타입별 워크플로우(Flutter, Spring 등)에 대해 사용자 선택권 부여하여 기존 파일 보호.

**보고서 파일**: `.report/20260108_#139_템플릿_업데이트시_워크플로우_선택적_처리.md`

---

### 🔍 문제 분석

**기존 동작 (문제점)**:
- `template_integrator.sh/ps1` 실행 시 모든 워크플로우를 무조건 `.bak` 백업 후 덮어쓰기
- 프로젝트별로 커스텀 수정한 CICD 로직이 손실됨
- 다음 업데이트 시 이전 `.bak` 파일도 덮어씌워져 복구 불가

**영향 범위**:
- 모든 타입별 워크플로우 (Flutter 6개, Spring 3개, React 2개, Next 2개)
- 특히 CI/CD 관련 파일은 프로젝트마다 민감하게 수정되는 경우가 많음

---

### ✅ 구현 내용

#### 워크플로우 분류 체계 도입
- **파일**: `template_integrator.sh`, `template_integrator.ps1`
- **변경 내용**: COMMON 워크플로우와 타입별 워크플로우를 구분하여 처리
- **이유**: COMMON은 템플릿 핵심 기능이라 항상 최신 유지 필요, 타입별은 커스텀 가능성이 높아 보호 필요

| 분류 | 패턴 | 동작 |
|------|------|------|
| COMMON | `PROJECT-COMMON-*` | 항상 최신으로 업데이트 |
| 타입별 | `PROJECT-[TYPE]-*` | 사용자 선택에 따라 처리 |

#### 사용자 선택 UI 추가
- **파일**: `template_integrator.sh` (lines 1053-1121), `template_integrator.ps1` (lines 983-1057)
- **변경 내용**: 이미 존재하는 타입별 워크플로우에 대해 3가지 옵션 제공
- **이유**: 사용자가 상황에 맞게 선택할 수 있도록 유연성 확보

**선택 옵션**:
```
(T) .template.yaml로 추가
    → 기존 파일 유지 + 새 버전을 참고용으로 추가
    → 예: PROJECT-FLUTTER-ANDROID-PLAYSTORE-CICD.template.yaml

(S) 건너뛰기
    → 기존 파일만 유지, 아무것도 추가 안 함

(O) 덮어쓰기 (기존 방식)
    → 기존 파일을 .bak으로 백업 후 덮어쓰기
```

#### 결과 요약 표시 개선
- **변경 내용**: 처리 완료 후 복사/참고용 추가/건너뜀 개수를 명확히 표시
- **이유**: 사용자가 어떤 파일이 어떻게 처리되었는지 한눈에 파악 가능

### 🔧 주요 변경사항 상세

#### template_integrator.sh

**COMMON 워크플로우 처리** (lines 1005-1023):
- 기존: `.bak` 백업 후 덮어쓰기
- 변경: 백업 없이 바로 덮어쓰기 (핵심 기능이라 항상 최신 유지)
- `print_info "$filename 업데이트"` 메시지로 업데이트 사실만 알림

**타입별 워크플로우 처리** (lines 1025-1127):
1. 먼저 신규 파일과 기존 파일을 분류하여 배열에 저장
2. 신규 파일은 바로 복사
3. 기존 파일이 있으면 사용자에게 선택 UI 표시
4. 선택에 따라 `.template.yaml` 추가 / 건너뛰기 / 덮어쓰기 실행

**특이사항**:
- `.template.yaml` 파일은 GitHub Actions에서 워크플로우로 인식하지 않음 (확장자가 `.yaml`로 끝나지 않기 때문)
- 사용자는 참고용으로 새 버전을 확인하고 필요한 부분만 수동으로 반영 가능

#### template_integrator.ps1

Bash 버전과 동일한 로직으로 PowerShell 버전 구현:
- `$existingFiles`, `$newFiles` 배열로 파일 분류
- `Read-SingleKey` 함수로 단일 키 입력 받기
- `switch` 문으로 선택 처리
- 정규표현식 `-replace '\.yaml$', '.template.yaml'`로 파일명 변환

### 📌 참고사항

1. **`.template.yaml` 파일 관리**
   - GitHub Actions가 인식하지 않으므로 자동 실행되지 않음
   - 사용자가 참고 후 기존 파일에 수동 반영
   - 필요 없으면 삭제해도 무방

2. **COMMON 워크플로우 목록** (항상 업데이트됨)
   - `PROJECT-COMMON-VERSION-CONTROL.yaml`
   - `PROJECT-COMMON-AUTO-CHANGELOG-CONTROL.yaml`
   - `PROJECT-COMMON-README-VERSION-UPDATE.yaml`
   - `PROJECT-COMMON-SUH-ISSUE-HELPER-API.yaml`
   - `PROJECT-COMMON-QA-ISSUE-CREATION-BOT.yaml`
   - `PROJECT-COMMON-SYNC-ISSUE-LABELS.yaml`

3. **하위 호환성**
   - 기존 `--force` 옵션은 그대로 유지
   - 대화형 모드에서만 선택 UI 표시
