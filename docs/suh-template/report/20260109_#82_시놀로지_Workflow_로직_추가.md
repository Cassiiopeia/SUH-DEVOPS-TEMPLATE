### 📌 작업 개요
시놀로지 NAS 관련 워크플로우 템플릿 추가 구현. 시놀로지 특정 경로 파일 업로드, Docker 배포, 무중단 배포, PR Preview 등 다양한 배포 시나리오 지원

**이슈**: [#82](https://github.com/Cassiiopeia/SUH-DEVOPS-TEMPLATE/issues/82)

### 🎯 구현 목표
- 시놀로지 NAS에 파일 업로드 및 배포 워크플로우 제공
- 프로젝트 타입별 시놀로지 전용 워크플로우 템플릿 제공
- template_integrator에서 선택적 포함 옵션 지원

### ✅ 구현 내용

#### 1. Flutter - Synology APK 배포 워크플로우
- **파일**: `.github/workflows/project-types/flutter/synology/PROJECT-FLUTTER-ANDROID-SYNOLOGY-CICD.yaml`
- **주요 기능**:
  - Flutter APK 빌드 후 Synology NAS SMB 경로에 업로드
  - 빌드 히스토리 JSON 자동 관리
  - Fastlane 기반 빌드 자동화
- **트리거**: deploy 브랜치 푸시, CHANGELOG 워크플로우 완료 시

#### 2. Spring Boot - Synology 단순 배포 워크플로우
- **파일**: `.github/workflows/project-types/spring/synology/PROJECT-SPRING-SYNOLOGY-SIMPLE-CICD.yaml`
- **주요 기능**:
  - Spring Boot 애플리케이션 Gradle 빌드
  - Docker 이미지 생성 및 DockerHub 푸시
  - SSH를 통한 Synology NAS 원격 배포
  - 브랜치별 포트 및 컨테이너 관리 (deploy/test)
- **트리거**: deploy, test 브랜치 푸시

#### 3. Spring Boot - Blue/Green 무중단 배포 워크플로우
- **파일**: `.github/workflows/project-types/spring/synology/PROJECT-SPRING-SYNOLOGY-NONSTOP-CICD.yaml`
- **주요 기능**:
  - Blue/Green 무중단 배포 전략 (main 브랜치)
  - Nginx 리버스 프록시 자동 토글
  - 헬스체크 및 자동 롤백
  - 이전 컨테이너 자동 정리
- **트리거**: main, test 브랜치 푸시

#### 4. Spring Boot - PR Preview 시스템
- **파일**: `.github/workflows/project-types/spring/synology/PROJECT-SPRING-SYNOLOGY-PR-PREVIEW.yaml`
- **주요 기능**:
  - PR 코멘트 명령어로 Preview 환경 관리
    - `@suh-lab pr build`: PR 빌드 및 배포
    - `@suh-lab pr destroy`: Preview 환경 삭제
    - `@suh-lab pr status`: 상태 확인
  - Traefik 리버스 프록시 동적 라우팅
  - 실시간 진행 상황 댓글 업데이트
- **트리거**: 이슈 코멘트, PR 종료

#### 5. template_integrator 연동
- **파일**: `template_integrator.sh`, `template_integrator.ps1`
- **변경 내용**:
  - `--synology` / `--no-synology` CLI 옵션 추가
  - 대화형 모드에서 Synology 워크플로우 포함 여부 질문
  - `version.yml`의 `metadata.template.options.synology`에 설정 저장
  - 재통합 시 이전 Synology 설정 자동 감지

### 🔧 주요 변경사항 상세

#### 워크플로우 폴더 구조
```
.github/workflows/project-types/
├── flutter/
│   └── synology/
│       └── PROJECT-FLUTTER-ANDROID-SYNOLOGY-CICD.yaml
└── spring/
    └── synology/
        ├── PROJECT-SPRING-SYNOLOGY-SIMPLE-CICD.yaml
        ├── PROJECT-SPRING-SYNOLOGY-NONSTOP-CICD.yaml
        └── PROJECT-SPRING-SYNOLOGY-PR-PREVIEW.yaml
```

#### 필수 GitHub Secrets

**Flutter Synology 배포:**
```
SERVER_HOST          # Synology NAS 호스트
SERVER_USER          # SMB 접속 사용자명
SERVER_PASSWORD      # SMB 접속 비밀번호
DEBUG_KEYSTORE       # Android Debug Keystore (Base64)
ENV_FILE             # Flutter .env 파일 내용
GOOGLE_SERVICES_JSON # Firebase 설정
```

**Spring Boot Synology 배포:**
```
APPLICATION_PROD_YML  # Spring Boot 운영 설정 파일
DOCKERHUB_USERNAME    # DockerHub 사용자명
DOCKERHUB_TOKEN       # DockerHub 액세스 토큰
SERVER_HOST           # 배포 대상 서버 IP/도메인
SERVER_USER           # 서버 SSH 접속 사용자명
SERVER_PASSWORD       # 서버 SSH 접속 비밀번호
```

#### 포트 설정 (Spring Boot)

| 환경 | 기본 포트 | Secret 변수 |
|------|----------|-------------|
| deploy (운영) | 8080 | PROJECT_DEPLOY_PORT |
| test (테스트) | 8081 | PROJECT_TEST_PORT |
| Blue/Green Blue | 8080 | BLUE_PORT (env) |
| Blue/Green Green | 8081 | GREEN_PORT (env) |
