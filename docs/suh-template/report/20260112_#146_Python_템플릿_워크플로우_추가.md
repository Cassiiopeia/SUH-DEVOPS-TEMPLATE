### 작업 개요
Python/FastAPI 프로젝트를 위한 CI/CD 워크플로우 템플릿 추가. MapSee-AI 프로젝트의 워크플로우를 기반으로 SUH-DEVOPS-TEMPLATE에 템플릿화하여 추가

**이슈**: [#146](https://github.com/Cassiiopeia/SUH-DEVOPS-TEMPLATE/issues/146)

### 구현 목표
- Python/FastAPI 프로젝트를 위한 Docker 기반 CI/CD 워크플로우 제공
- Synology NAS 배포 및 PR Preview 시스템 지원
- 볼륨 마운트 설정을 환경변수로 분리하여 프로젝트별 유연한 구성 가능

### 구현 내용

#### 1. Python CI 워크플로우 (빌드 검증 전용)
- **파일**: `.github/workflows/project-types/python/PROJECT-PYTHON-CI.yaml`
- **주요 기능**:
  - Docker 이미지 빌드 검증 (push 없음)
  - main 브랜치 푸시 및 PR 시 자동 실행
  - GitHub Actions 캐시 활용으로 빌드 속도 향상
- **트리거**: main 브랜치 push, main으로의 PR, 수동 실행

#### 2. Python Synology CICD 워크플로우
- **파일**: `.github/workflows/project-types/python/synology/PROJECT-PYTHON-SYNOLOGY-CICD.yaml`
- **주요 기능**:
  - Docker 이미지 빌드 및 DockerHub 푸시
  - SSH를 통한 Synology NAS 원격 배포
  - 브랜치별 포트 및 컨테이너 관리 (deploy/test)
  - 조건부 볼륨 마운트 (환경변수로 제어)
- **트리거**: deploy, test 브랜치 push, 수동 실행

#### 3. Python Synology PR Preview 시스템
- **파일**: `.github/workflows/project-types/python/synology/PROJECT-PYTHON-SYNOLOGY-PR-PREVIEW.yaml`
- **주요 기능**:
  - PR/Issue 코멘트 명령어로 Preview 환경 관리
    - `@suh-lab server build`: 빌드 및 배포
    - `@suh-lab server destroy`: Preview 환경 삭제
    - `@suh-lab server status`: 상태 확인
  - Traefik 리버스 프록시 동적 라우팅
  - 실시간 진행 상황 댓글 업데이트
  - Issue Helper 댓글에서 브랜치명 자동 추출
  - 조건부 볼륨 마운트 (PR 빌드 + Issue 빌드 양쪽 적용)
- **트리거**: 이슈 코멘트, PR/Issue 종료

### 주요 변경사항 상세

#### 워크플로우 폴더 구조
```
.github/workflows/project-types/
└── python/
    ├── PROJECT-PYTHON-CI.yaml                    # CI Only (빌드 검증)
    └── synology/
        ├── PROJECT-PYTHON-SYNOLOGY-CICD.yaml     # Synology Docker 배포
        └── PROJECT-PYTHON-SYNOLOGY-PR-PREVIEW.yaml  # PR Preview 시스템
```

#### 볼륨 마운트 환경변수 추가
CICD 및 PR-Preview 워크플로우에 다음 환경변수 추가:

```yaml
# 볼륨 마운트 설정 (프로젝트별 수정 필요, 빈값이면 마운트 안함)
PROJECT_TARGET_DIR: "/volume1/projects/mapsee/ai"  # 서버의 데이터 디렉토리
PROJECT_MNT_DIR: "/mnt/mapsee"                     # 컨테이너 내부 마운트 경로
```

#### 조건부 볼륨 마운트 로직
환경변수가 설정된 경우에만 볼륨 마운트 실행:

```bash
# 볼륨 마운트 옵션 구성
VOLUME_OPTS="-v /etc/localtime:/etc/localtime:ro"
if [ -n "${{ env.PROJECT_TARGET_DIR }}" ] && [ -n "${{ env.PROJECT_MNT_DIR }}" ]; then
  VOLUME_OPTS="$VOLUME_OPTS -v ${{ env.PROJECT_TARGET_DIR }}:${{ env.PROJECT_MNT_DIR }}"
fi

echo $PW | sudo -S docker run -d \
  -p ${PORT}:8000 \
  --name $CONTAINER_NAME \
  -e TZ=Asia/Seoul \
  $VOLUME_OPTS \
  ${IMAGE}
```

#### 필수 GitHub Secrets

**공통:**
```
ENV_FILE             # .env 파일 전체 내용
DOCKERHUB_USERNAME   # DockerHub 사용자명
DOCKERHUB_TOKEN      # DockerHub 액세스 토큰
SERVER_HOST          # 배포 대상 서버 IP/도메인
SERVER_USER          # 서버 SSH 접속 사용자명
SERVER_PASSWORD      # 서버 SSH 접속 비밀번호
```

**테스트 환경 (선택적):**
```
PYTHON_TEST_PORT     # test 브랜치 배포 포트 (기본: 8001)
```

#### 포트 설정

| 환경 | 기본 포트 | 설정 방법 |
|------|----------|----------|
| deploy (운영) | 8092 | env.PROJECT_MAIN_PORT |
| test (테스트) | 8001 | secrets.PYTHON_TEST_PORT |
| 컨테이너 내부 | 8000 | 고정 |

#### Spring vs Python 워크플로우 차이점

| 항목 | Spring | Python |
|------|--------|--------|
| 빌드 도구 | Gradle | Docker (Dockerfile) |
| 설정 파일 | application-prod.yml | .env |
| Secret 키 | APPLICATION_PROD_YML | ENV_FILE |
| 기본 포트 | 8080 | 8000 |
| Health Check | /actuator/health | /docs (FastAPI) |
| 프레임워크 | Spring Boot | FastAPI/Uvicorn |

### 의존성 변경
- 없음 (기존 GitHub Actions 활용)

### 테스트 및 검증
- 워크플로우 YAML 문법 검증
- 조건부 볼륨 마운트 로직 검증 (빈값 처리)
- PR Preview의 2개 docker run 위치 모두 적용 확인

### 참고사항
- Synology 워크플로우는 `--synology` 옵션으로 통합 시에만 포함됨
- PR Preview 시스템은 Traefik 리버스 프록시 사전 설정 필요
- `PROJECT_TARGET_DIR`, `PROJECT_MNT_DIR` 환경변수가 모두 비어있으면 볼륨 마운트 로직 스킵
- Health Check는 HTTP (/docs) 우선, 실패 시 로그 패턴 매칭으로 폴백
