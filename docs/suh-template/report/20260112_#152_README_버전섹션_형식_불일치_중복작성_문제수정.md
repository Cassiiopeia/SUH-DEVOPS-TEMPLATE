# 구현 보고서: README 버전 섹션 형식 불일치로 인한 중복 작성 문제

**이슈**: [#152](https://github.com/Cassiiopeia/SUH-DEVOPS-TEMPLATE/issues/152)
**작성일**: 2026-01-12
**작성자**: Claude Code

---

### 📌 작업 개요

`template_initializer.sh`로 초기화한 프로젝트에서 `template_integrator` 실행 시 README 버전 섹션이 중복 추가되는 문제 수정. Windows PowerShell에서 한글 패턴 파싱 문제로 인해 기존 한글 마커를 영어로 통일.

**핵심 변경**: 주석 마커만 영어로 변경, 사용자에게 보이는 텍스트(버전 제목, 링크)는 한글 유지

---

### 🔍 문제 분석

#### 근본 원인
세 스크립트가 각각 다른 주석 마커 사용:
- `template_initializer.sh`: `<!-- 수정하지마세요 자동으로 동기화 됩니다 -->`
- `template_integrator`: `<!-- AUTO-VERSION-SECTION: DO NOT EDIT MANUALLY -->`
- 워크플로우: 혼합 사용

#### 발생 문제
1. 마커 형식 불일치로 중복 감지 실패
2. Windows PowerShell에서 한글 정규표현식 파싱 오류
3. 불필요한 종료 마커(`<!-- END-AUTO-VERSION-SECTION -->`)와 설명 주석 사용

#### 재현 상황
```markdown
# 프로젝트명

<!-- 수정하지마세요 자동으로 동기화 됩니다 -->
## 최신 버전 : v0.0.0
[전체 버전 기록 보기](CHANGELOG.md)

---

<!-- AUTO-VERSION-SECTION: DO NOT EDIT MANUALLY -->
## 최신 버전 : v0.0.1
[전체 버전 기록 보기](CHANGELOG.md)
<!-- END-AUTO-VERSION-SECTION -->
```

### ✅ 구현 내용

#### 1. template_initializer.sh 주석 마커 영어로 변경
- **파일**: `.github/scripts/template_initializer.sh`
- **위치**: 491줄 `initialize_readme()` 함수
- **변경 내용**: 한글 마커 → 영어 마커
- **이유**: 신규 프로젝트 초기화 시 표준 형식으로 생성

```bash
# 변경 전
<!-- 수정하지마세요 자동으로 동기화 됩니다 -->

# 변경 후
<!-- AUTO-VERSION-SECTION: DO NOT EDIT MANUALLY -->
```

#### 2. template_integrator.sh 감지 패턴 및 섹션 형식 변경
- **파일**: `template_integrator.sh`
- **위치**: 878-899줄 `add_version_section_to_readme()` 함수
- **변경 내용**:
  - 감지 패턴을 영어 마커 단일 체크로 간소화
  - 불필요한 설명 주석, 종료 마커 제거
- **이유**: Windows 호환성 확보 및 형식 단순화

```bash
# 감지 패턴 간소화
if grep -q "<!-- AUTO-VERSION-SECTION" README.md; then

# 버전 섹션 생성 (단순화)
<!-- AUTO-VERSION-SECTION: DO NOT EDIT MANUALLY -->
## 최신 버전 : v${version}

[전체 버전 기록 보기](CHANGELOG.md)
```

#### 3. template_integrator.ps1 동일 변경 적용
- **파일**: `template_integrator.ps1`
- **위치**: 762-785줄 `Add-VersionSectionToReadme` 함수
- **변경 내용**: bash 버전과 동일한 로직으로 수정
- **이유**: Windows PowerShell에서 한글 패턴 파싱 문제 해결

```powershell
# 감지 패턴 간소화 (한글 제거)
if ($readmeContent -match "<!-- AUTO-VERSION-SECTION") {
```

### 🧪 테스트 및 검증

**검증 시나리오**:
1. **신규 프로젝트**: `template_initializer.sh` 실행 후 `template_integrator` 재실행 → 중복 없어야 함
2. **기존 프로젝트**: 빈 README에 `template_integrator` 실행 → 정상 추가
3. **Windows 테스트**: PowerShell에서 한글 파싱 문제 없이 감지 동작

**확인 방법**:
- `grep -q "<!-- AUTO-VERSION-SECTION" README.md` 명령으로 마커 존재 여부 확인
- 중복 섹션 없이 단일 버전 정보만 표시되는지 확인

### 📌 참고사항

- 기존 프로젝트에서 한글 마커(`<!-- 수정하지마세요 자동으로 동기화 됩니다 -->`)가 있는 경우, 워크플로우가 자동으로 영어 마커로 교체
- 버전 업데이트 워크플로우는 버전 라인(`## 최신 버전 : vX.X.X`)만 수정하므로 종료 마커 불필요
- 향후 모든 신규 프로젝트는 표준 형식으로 생성됨
