### 📌 작업 개요

Flutter 프로젝트의 PR 생성 및 main 브랜치 푸시 시 **코드 정적 분석(flutter analyze)**과 **Android/iOS 빌드 검증**을 수행하는 CI 워크플로우 신규 생성. 빌드 결과를 PR 댓글로 실시간 업데이트하며, 플랫폼별 활성화/비활성화 옵션과 **Analyze Only 모드** 지원.

---

### 🎯 구현 목표

1. PR 생성/업데이트 시 `flutter analyze`로 코드 품질 검사
2. iOS, Android 빌드 테스트 수행
3. 빌드 결과를 마크다운 테이블 형태로 PR 댓글에 표시
4. 빌드/분석 실패 시 상세 에러 정보 제공
5. `.env` 파일 경로를 환경변수로 커스터마이징 가능 (기본값: 루트 `.env`)
6. ENABLE_ANDROID/ENABLE_IOS 옵션으로 개별 플랫폼 제어
7. **ANALYZE_ONLY 옵션으로 빌드 없이 코드 분석만 실행 가능**
8. Analyze와 빌드 병렬 실행으로 시간 단축

### 🔧 주요 구현 상세

#### 1. 워크플로우 구조 (5개 Job)

| Job | Runner | 역할 |
|-----|--------|------|
| `prepare` | ubuntu-latest | 환경 정보 수집, 진행 상황 댓글 생성 |
| `analyze` | ubuntu-latest | flutter analyze 코드 분석 (병렬) |
| `build-android` | ubuntu-latest | Android APK 빌드 (조건부, 병렬) |
| `build-ios` | macos-15 | iOS 빌드 (조건부, 병렬) |
| `report` | ubuntu-latest | 최종 결과 댓글 업데이트 |

**병렬 실행 구조**:
```
prepare → analyze (병렬)
        → build-android (병렬)
        → build-ios (병렬)
        → report
```

#### 2. 트리거 조건

```yaml
on:
  pull_request:
    branches: [main]
    types: [opened, synchronize, reopened]
  push:
    branches: [main]
  workflow_dispatch:
    inputs:
      analyze_only:
        description: 'Analyze Only 모드 (빌드 없이 코드 분석만)'
        type: boolean
        default: false
      enable_android:
        description: 'Android 빌드 활성화 (Analyze Only=false일 때)'
        type: boolean
        default: true
      enable_ios:
        description: 'iOS 빌드 활성화 (Analyze Only=false일 때)'
        type: boolean
        default: true
```

#### 3. CI 모드 및 플랫폼 활성화/비활성화 옵션

```yaml
env:
  # 🎯 CI 모드 설정
  ANALYZE_ONLY: "false"     # true: analyze만 실행 (빌드 스킵), false: analyze + 빌드

  # 🎯 빌드 대상 플랫폼 설정 (ANALYZE_ONLY=false일 때만 적용)
  ENABLE_ANDROID: "true"    # Android 빌드 활성화 (true/false)
  ENABLE_IOS: "true"        # iOS 빌드 활성화 (true/false)
```

- 워크플로우 상단 `env` 섹션에서 기본값 설정
- `workflow_dispatch` 수동 실행 시 입력값으로 오버라이드 가능
- `prepare` job에서 outputs으로 전달하여 각 빌드 job의 `if` 조건에서 사용
- **ANALYZE_ONLY=true**: 빌드 job이 스킵되고 코드 분석만 실행

#### 4. flutter analyze Job

```yaml
- name: Run Flutter Analyze
  id: analyze
  run: |
    echo "🔍 Flutter 코드 분석 시작..."
    flutter analyze --no-fatal-infos
    echo "✅ 코드 분석 완료"
```

**`--no-fatal-infos` 옵션**: info 수준 메시지는 실패로 처리하지 않음 (warning/error만 실패)

#### 5. 환경변수 파일 경로 커스터마이징

```yaml
env:
  ENV_FILE_PATH: ".env"     # 환경변수 파일 경로 (커스터마이징 가능)
```

`.env` 파일 생성 시 기존 패턴 준수:
```yaml
- name: Create .env file
  run: |
    cat << 'EOF' > ${{ env.ENV_FILE_PATH }}
    ${{ secrets.ENV_FILE || secrets.ENV }}
    EOF
```

### 🧪 테스트 및 검증

1. **PR 생성 테스트**: PR 생성 시 워크플로우 트리거 및 댓글 생성 확인
2. **Analyze 테스트**: `flutter analyze` 성공/실패 시 결과 반영 확인
3. **병렬 실행 확인**: Analyze/Android/iOS가 동시에 실행되는지 확인
4. **Enable/Disable 테스트**:
   - `ENABLE_ANDROID: "false"` 시 Android 빌드 스킵
   - `ENABLE_IOS: "false"` 시 iOS 빌드 스킵
5. **Analyze Only 모드 테스트**:
   - `ANALYZE_ONLY: "true"` 시 Android/iOS 빌드 모두 스킵
   - PR 댓글에 "Analyze Only 모드" 표시 확인
6. **workflow_dispatch 테스트**: 수동 실행 시 입력값으로 플랫폼 선택
7. **main 푸시 테스트**: main 푸시 시 댓글 없이 분석/빌드만 실행

### 📌 참고사항

1. **CI는 코드 품질 검사 + 빌드 검증 목적**: 서명/배포 관련 Secrets 불필요. 배포용 빌드는 별도 CD 워크플로우 사용
2. **flutter analyze**: `--no-fatal-infos` 옵션으로 info 수준은 무시, warning/error만 실패 처리
3. **iOS 빌드 시 `--no-codesign`**: CI 목적이므로 실제 서명 없이 빌드 성공 여부만 검증
4. **상세 헤더 주석 포함**: 기존 워크플로우 패턴을 따라 사용법, 설정 방법, 필요 Secrets 등 상세 주석 작성
5. **동시성 제어**: `cancel-in-progress: true`로 동일 브랜치의 이전 실행 자동 취소
6. **실패 시 항목별 가이드**: Analyze/Android/iOS 각각에 맞는 해결 방법 안내
7. **Analyze Only 모드**: 빌드 시간이 오래 걸리거나 코드 품질만 확인하려는 프로젝트를 위한 옵션
