### 📌 작업 개요
템플릿 업데이트 시 기존 프로젝트와 호환되지 않는 변경사항(Breaking Changes)을 감지하여 사용자에게 경고 알림 표시 기능 구현

**이슈**: #167

### 🎯 구현 목표
- `.github/config/breaking-changes.json` 파일로 호환성 문제 기록 관리
- `template_integrator.sh/ps1`에서 프로젝트 템플릿 버전과 비교하여 해당 버전 범위의 breaking changes 알림
- `critical` 레벨 변경사항은 Y/N 확인 필수, `warning`은 알림만 표시

---

### 🔧 주요 변경사항 상세

#### breaking-changes.json 구조
버전을 키로 사용하여 해당 버전의 breaking change 기록
```json
{
  "_comment": ["사용 방법 설명..."],
  "2.6.23": {
    "severity": "critical",
    "title": "Issue Labels 전면 변경",
    "message": "상세 설명..."
  }
}
```

**severity 기준**:
| 레벨 | 기준 | 동작 |
|------|------|------|
| `critical` | 수동 조치 필수 | Y/N 확인 필수 |
| `warning` | 알림만 | 알림 후 자동 진행 |

#### 알림 메시지 형식
```
╔══════════════════════════════════════════════════════════════════╗
║  ⚠️  BREAKING CHANGES (v2.5.0 → v2.7.0)
╠══════════════════════════════════════════════════════════════════╣
║
║  [CRITICAL] v2.6.23 - Issue Labels 전면 변경
║  → 이슈 라벨이 전면 변경되었습니다. 변경사항: ...
║
╚══════════════════════════════════════════════════════════════════╝
```

#### version.yml 템플릿 버전 정보
새 프로젝트 초기화 시 `metadata.template` 섹션 자동 생성:
```yaml
metadata:
  template:
    source: "SUH-DEVOPS-TEMPLATE"
    version: "2.6.22"
    initialized_date: "2026-01-22"
```

#### 버전 비교 로직
`current_version < ver <= new_version` 범위의 breaking changes만 수집
- jq 미설치 시 경고 메시지 출력 후 스킵
- IFS 전역 변수 오염 방지를 위해 `tr` 명령어 사용
- PowerShell 정규식은 template 섹션 내 version만 정확히 매칭
