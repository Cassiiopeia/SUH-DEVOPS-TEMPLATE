### 📌 작업 개요

템플릿 전체 워크플로우 파일에 대한 주석 컨벤션을 통일화. 기존에 제각각이던 주석 형식(테이블, 긴 설명, 다양한 구분선 길이)을 일관된 규칙으로 표준화.

### 🎯 구현 목표

- 모든 워크플로우에 67자 구분선 통일
- GitHub Secrets 작성 형식 1줄로 통일
- 테이블 형식 제거 (관리 불편)
- GITHUB_TOKEN 언급 금지 (자동 제공)
- 선택 항목에만 `(선택)` 표시
- 워크플로우 타입별 템플릿 정의 (A~E 타입)

### 🔧 주요 변경사항 상세

#### 워크플로우 타입 분류 (5가지)

| 타입 | 특징 | 예시 |
|------|------|------|
| Type A | Secrets 없음 | SYNC-ISSUE-LABELS, VERSION-CONTROL |
| Type B | Secrets 있음, 환경변수 단순 | CHANGELOG-CONTROL, PROJECTS-SYNC-MANAGER |
| Type C | CI, 빌드 검증 | FLUTTER-CI, REACT-CI |
| Type D | CD, 배포 | SPRING-CICD, PYTHON-CICD |
| Type E | 마법사 연동, 복잡 | IOS-TESTFLIGHT, PLAYSTORE-CICD |

### 📌 참고사항

1. **루트와 project-types 동기화**: 공통 워크플로우(COMMON)는 루트와 `project-types/common/` 두 곳에 동일하게 유지해야 함

2. **테이블 형식 사용 금지**: 향후 워크플로우 작성 시 환경변수 설명에 테이블 형식 사용하지 않음

3. **가이드라인 문서 참조**: 새 워크플로우 작성 시 `docs/WORKFLOW-COMMENT-GUIDELINES.md` 참조

### 🧪 검증 방법

1. 각 워크플로우 파일 상단 주석 형식 확인
2. 구분선 67자 여부 확인 (`# ===================================================================`)
3. Secrets 형식이 1줄인지 확인
4. 테이블 형식 잔존 여부 확인
