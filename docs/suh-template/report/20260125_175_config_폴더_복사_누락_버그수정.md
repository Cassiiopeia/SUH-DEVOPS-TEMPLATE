# Issue #175: .github/config 폴더 복사 누락 버그 수정

### 📌 작업 개요

template_integrator (sh/ps1) 실행 시 `.github/config/` 폴더가 복사되지 않는 버그 수정. 해당 폴더에는 `breaking-changes.json`, `issue-labels.yml` 등 중요 설정 파일이 포함되어 있어 템플릿 통합 시 필수적으로 복사되어야 함.

### 🔍 문제 분석

**원인**: `template_integrator.sh` 및 `template_integrator.ps1`에 `.github/config/` 폴더를 복사하는 로직이 누락됨

**영향 범위**:
- `breaking-changes.json`: 템플릿 버전 간 호환성 정보 누락
- `issue-labels.yml`: 이슈 라벨 동기화 워크플로우 동작 불가

**확인 결과**:
- `template_initializer.sh`: 변경 불필요 (초기화 시 config 폴더 삭제하지 않음)

### ✅ 구현 내용

#### 1. copy_config_folder() 함수 추가 (Bash)
- **파일**: `template_integrator.sh`
- **위치**: 1625번째 줄 (copy_scripts() 함수 다음)
- **변경 내용**: `.github/config` 폴더를 복사하는 새 함수 추가
- **이유**: 기존 스크립트에 config 폴더 복사 로직이 없었음

#### 2. Copy-ConfigFolder 함수 추가 (PowerShell)
- **파일**: `template_integrator.ps1`
- **위치**: 1542번째 줄 (Copy-Scripts 함수 다음)
- **변경 내용**: Bash 버전과 동일한 기능의 PowerShell 함수 추가
- **이유**: Windows 환경 지원을 위한 동기화

#### 3. 각 모드에서 함수 호출 추가
- **파일**: `template_integrator.sh`, `template_integrator.ps1`
- **변경 내용**: `execute_integration()` 함수 내 full/version/workflows 모드에서 호출 추가
- **이유**: 세 가지 주요 모드에서 config 폴더가 필요함

### 🔧 주요 변경사항 상세

#### template_integrator.sh - copy_config_folder()

```bash
copy_config_folder() {
    print_step ".github/config 폴더 복사 중..."

    local src_config_dir="$TEMP_DIR/.github/config"
    local dst_config_dir=".github/config"

    if [ ! -d "$src_config_dir" ]; then
        print_info ".github/config 폴더가 템플릿에 없습니다. 건너뜁니다."
        return
    fi

    # 기존 config 파일이 있으면 알림
    if [ -d "$dst_config_dir" ] && [ "$(ls -A "$dst_config_dir" 2>/dev/null)" ]; then
        print_info "기존 config 파일이 있습니다. 덮어씁니다."
    fi

    mkdir -p "$dst_config_dir"
    cp -r "$src_config_dir/"* "$dst_config_dir/" 2>/dev/null || true

    local copied=$(ls -1 "$dst_config_dir" 2>/dev/null | wc -l | tr -d ' ')
    print_success ".github/config 폴더 복사 완료 ($copied 개 파일)"
}
```

**특이사항**:
- 기존 `copy_issue_templates()` 함수 패턴과 동일한 스타일로 구현
- 기존 파일 존재 시 덮어쓰기 알림 제공
- 복사된 파일 개수 출력으로 사용자 피드백 개선

#### template_integrator.ps1 - Copy-ConfigFolder

```powershell
function Copy-ConfigFolder {
    Print-Step ".github/config 폴더 복사 중..."

    $srcConfigDir = Join-Path $TEMP_DIR ".github\config"
    $dstConfigDir = ".github\config"

    if (-not (Test-Path $srcConfigDir)) {
        Print-Info ".github/config 폴더가 템플릿에 없습니다. 건너뜁니다."
        return
    }

    if ((Test-Path $dstConfigDir) -and (Get-ChildItem $dstConfigDir -ErrorAction SilentlyContinue)) {
        Print-Info "기존 config 파일이 있습니다. 덮어씁니다."
    }

    if (-not (Test-Path $dstConfigDir)) {
        New-Item -Path $dstConfigDir -ItemType Directory -Force | Out-Null
    }

    Copy-Item -Path "$srcConfigDir\*" -Destination $dstConfigDir -Recurse -Force -ErrorAction SilentlyContinue

    $copied = (Get-ChildItem $dstConfigDir -File -ErrorAction SilentlyContinue | Measure-Object).Count
    Print-Success ".github/config 폴더 복사 완료 ($copied 개 파일)"
}
```

### 📌 참고사항

- `template_initializer.sh`는 변경 불필요 확인됨 (config 폴더 삭제하지 않음)
- config 폴더는 항상 최신 템플릿 버전으로 덮어쓰기됨
- issues 모드와 commands 모드에서는 config 폴더가 필요하지 않아 호출하지 않음
