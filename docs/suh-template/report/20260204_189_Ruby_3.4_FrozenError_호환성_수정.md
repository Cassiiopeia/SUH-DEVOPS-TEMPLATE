### 📌 작업 개요

macOS 26 환경에서 Ruby 3.4 버전 사용 시 Flutter iOS 빌드 및 changelog 업데이트 과정에서 Fastlane의 `FrozenError: can't modify frozen String` 에러 발생 문제 수정

**관련 이슈**: [#189](https://github.com/Cassiiopeia/SUH-DEVOPS-TEMPLATE/issues/189)

---

### 🔍 문제 분석

**원인**: Ruby 3.4+에서 `ENV["KEY"]` 반환값이 frozen string(불변 문자열)으로 처리됨. Fastlane 내부의 `strip_emoji` 등의 함수가 `gsub!`로 문자열을 직접 수정하려 할 때 `FrozenError` 발생

**발생 위치**:
- Fastfile의 `release_notes` 변수 (TestFlight 업로드 시)
- Fastfile의 `changelog` 변수 (로컬 빌드 시)

**에러 메시지**: `FrozenError: can't modify frozen String`

---

### ✅ 구현 내용

#### 1. Fastfile 템플릿 - release_notes 변수 수정
- **파일**: `.github/util/flutter/testflight-wizard/templates/Fastfile`
- **변경 내용**: `release_notes` 변수에 `.dup` 메서드 추가하여 mutable 복사본 생성
- **이유**: Ruby 3.4+에서 frozen string을 직접 수정하지 않고 복사본을 사용하도록 변경

```ruby
# Before
release_notes = ENV["RELEASE_NOTES"] || "새로운 빌드가 업로드되었습니다."

# After
release_notes = (ENV["RELEASE_NOTES"] || "새로운 빌드가 업로드되었습니다.").dup
```

#### 2. Fastfile 템플릿 - changelog 변수 수정 (로컬 빌드용)
- **파일**: `.github/util/flutter/testflight-wizard/templates/Fastfile`
- **변경 내용**: `build_and_deploy` lane의 `changelog` 변수에도 `.dup` 적용
- **이유**: 로컬 개발 환경에서도 동일한 Ruby 3.4+ 호환성 보장

```ruby
# Before
changelog: options[:changelog] || "새로운 빌드가 업로드되었습니다.",

# After
changelog = (options[:changelog] || "새로운 빌드가 업로드되었습니다.").dup
```

#### 3. Gemfile 템플릿 - Fastlane 버전 업그레이드
- **파일**: `.github/util/flutter/testflight-wizard/templates/Gemfile`
- **변경 내용**: Fastlane 버전을 `~> 2.225`에서 `~> 2.228`로 업그레이드
- **이유**: Fastlane 2.228 이상이 Ruby 3.4를 공식 지원

```ruby
# Before
gem "fastlane", "~> 2.225"

# After
gem "fastlane", "~> 2.228"
```

#### 4. 초기화 스크립트 동기화
- **파일**: `.github/util/flutter/testflight-wizard/testflight-wizard-setup.sh`
- **변경 내용**: Gemfile 생성 로직에서도 Fastlane 버전을 `~> 2.228`로 동기화
- **이유**: 템플릿 파일과 스크립트 내 Gemfile 생성 로직의 일관성 유지

### 🔧 주요 변경사항 상세

#### Ruby 3.4+ Frozen String Literal 배경

Ruby 3.4부터 문자열 리터럴의 기본 동작이 변경됨:
- `ENV["KEY"]` 반환값이 "chilled string" 상태로 반환
- 이 문자열을 직접 수정하려 하면 `FrozenError` 발생
- `.dup` 메서드로 mutable 복사본을 생성하면 문제 해결

**특이사항**:
- `.dup` 추가는 Ruby 3.1 이하 버전에서도 정상 동작 (하위 호환성 유지)
- 마법사 우선 아키텍처이므로 기존 프로젝트는 testflight-wizard를 다시 실행해야 적용됨
