# ⚙️[기능개선][Skills] issue 스킬 파일 출력 및 서식 규칙 개선

## 개요

`/issue` 스킬이 이슈 내용을 터미널에만 출력하고 파일로 저장하지 않던 문제를 해결. `/report` 스킬의 `.report/` 패턴을 참고하여 `.issue/` 폴더에 파일 자동 생성하도록 스킬 정의를 개선했다.

## 변경 사항

### 스킬 정의 (`skills/issue/SKILL.md`)
- `skills/issue/SKILL.md`: 스킬 전체 프로세스 재구성

## 주요 구현 내용

### 1. 파일 저장 프로세스 추가 (4단계 신설)
- **파일 위치**: `.issue/[YYYYMMDD]_#[번호]_[제목].md`
- 날짜(YYYYMMDD) + 자동 채번(#001~) + 한국어 제목 조합
- `/report` 스킬의 `.report/[YYYYMMDD]_[#이슈번호]_설명.md` 패턴과 통일

### 2. 이모지 서식 절대 규칙 추가
- 이모지와 텍스트/대괄호 사이 빈칸 금지 규칙 명시
- `🗒️설명` (O), `🗒️ 설명` (X)
- `❗[버그]` (O), `❗ [버그]` (X)
- 기능 세분류도 공백 제거: `➕기능추가`, `⚙️기능개선`, `🔧기능수정`

### 3. 출력 포맷 개선
- 파일 첫 줄에 이슈 제목을 `# ` 헤딩으로 작성 → 바로 복사 가능
- 5단계(사용자에게 안내)를 분리하여 파일 경로 + 제목 한 줄 출력

### 4. 코드 탐색 절차 명시 (3단계 강화)
- `.github/ISSUE_TEMPLATE/` 해당 템플릿을 Read로 읽어 형식 파악 후 본문 작성하도록 지시

## 주의사항

- `.issue/` 폴더는 `.gitignore`에 추가 여부를 프로젝트별로 결정 필요
- 번호 채번은 `.issue/` 폴더 내 기존 파일 수 기반이므로, 파일 삭제 시 번호 불연속 가능
