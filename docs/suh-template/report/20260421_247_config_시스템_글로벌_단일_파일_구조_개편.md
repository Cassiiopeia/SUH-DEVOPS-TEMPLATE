# [기능개선][Config] config 시스템 글로벌 단일 파일 구조로 개편

## 개요

config 파일 관리 구조를 프로젝트별 로컬 파일(`{PROJECT_ROOT}/.suh-template/config/{skill_id}.config.json`) 방식에서 **글로벌 단일 파일(`~/.suh-template/config/config.json`)** 방식으로 전면 개편했다. 기존에는 스킬별로 별도 파일을 두는 분산 구조였으나, 이를 하나의 파일에 `global_pat` + `repos[].pat` 계층 구조로 통합했다. 관련 스킬 5종(issue, commit, github, changelog-deploy, report)이 새 스키마를 반영하도록 함께 수정됐다.

## 변경 사항

### Config 구조 개편 (`skills/references/config-rules.md`)
- **로컬 config 폐지**: `{PROJECT_ROOT}/.suh-template/config/{skill_id}.config.json` 경로 제거
- **글로벌 단일 파일**: `{HOME}/.suh-template/config/config.json` 하나로 통합
- **스키마 변경**: `github_pat` → `global_pat`, `github_repos` → `repos`, 각 repo에 `pat` 필드 추가
- **PAT 우선순위 도입**: repo별 `pat`이 non-null이면 우선 사용, 없으면 `global_pat` fallback
- **읽기 순서 단순화**: 로컬/글로벌 2단계 탐색 → 단일 경로 직접 읽기로 변경
- **저장 위치 선택 제거**: 기존 "로컬/글로벌 중 선택" UI 제거, 항상 글로벌에만 저장
- **대화형 수집 순서 명시**: `global_pat` → `default_assignee` → 첫 번째 repo 순서로 고정

### issue 스킬 반영 (`skills/issue/SKILL.md`)
- Config 읽기 시 `github_pat` → `global_pat`, `github_repos` → `repos` 키로 변경
- 선택된 repo의 `pat` non-null 여부 확인 후 PAT 결정 로직 추가
- 초기 저장 시 새 스키마(`global_pat`, `repos[].pat`) 형식으로 저장

### commit 스킬 반영 (`skills/commit/SKILL.md`)
- config 참조 키를 새 스키마에 맞게 업데이트

### github 스킬 반영 (`skills/github/SKILL.md`)
- config 읽기 키 및 PAT 추출 로직을 새 스키마에 맞게 수정

### changelog-deploy 스킬 반영 (`skills/changelog-deploy/SKILL.md`)
- Config 파일 위치 안내 문구를 글로벌 단일 파일로 변경
- PAT 추출 로직을 `repos[].pat` 우선, `global_pat` fallback 구조로 교체
- 로컬 config 탐색 코드 제거

### report 스킬 반영 (`skills/report/SKILL.md`)
- config 참조 경로 및 키를 새 스키마에 맞게 업데이트

### 구현 보고서 저장 (`docs/suh-template/report/`)
- 이슈 #247 구현 보고서 최초 작성

## 주요 구현 내용

**Before — 분산 구조:**
```
{PROJECT_ROOT}/.suh-template/config/issue.config.json  ← 로컬 (1순위)
{HOME}/.suh-template/config/issue.config.json           ← 글로벌 (2순위)
```
각 스킬이 독립 파일을 관리하고, 로컬/글로벌 2단계 탐색이 필요했다.

**After — 글로벌 단일 파일 구조:**
```json
// ~/.suh-template/config/config.json
{
  "default_assignee": "Cassiiopeia",
  "global_pat": "ghp_xxxx",
  "repos": [
    { "name": "SUH-DEVOPS-TEMPLATE", "owner": "Cassiiopeia", "repo": "SUH-DEVOPS-TEMPLATE", "pat": null, "default": true },
    { "name": "다른프로젝트", "owner": "Cassiiopeia", "repo": "OtherRepo", "pat": "ghp_별도PAT", "default": false }
  ]
}
```
- 파일 하나로 모든 스킬·모든 프로젝트의 설정을 관리
- `repos[].pat`이 있으면 해당 PAT 우선 사용 (다중 조직 대응)
- `global_pat`은 repo별 PAT가 없을 때의 공통 fallback

## 주의사항

- 기존 `{skill_id}.config.json` 포맷으로 설정이 저장된 환경은 재등록이 필요하다 (`/cassiiopeia:issue` 스킬 실행 → 새 스키마로 저장)
- `breaking-changes.json`에 이 변경을 등록하면 template_integrator 업데이트 시 사용자에게 안내할 수 있다
