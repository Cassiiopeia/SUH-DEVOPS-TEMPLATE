# 구현 완료 보고 — #278 github 스킬 한글 브랜치명 PR 생성 422 오류

## 개요

한글이 포함된 브랜치명(`YYYYMMDD_#번호_한글제목`)으로 PR 생성 시 발생하던
`422 Validation Failed: head invalid` 오류를 두 가지 방어 로직으로 해결했다.

## 변경 사항

### Skills
- `skills/github/SKILL.md` — PR 생성 전 `git ls-remote` 사전 확인 단계 추가, `head` 필드 `owner:branch` 형식 강제

### 문서
- `docs/suh-template/issue/20260425_#278_...md` — 이슈 파일 등록

## 주요 구현 내용

**PR 생성 전 remote 브랜치 존재 확인**

PR API 호출 전 아래 명령으로 remote에 브랜치가 실제로 존재하는지 사전 확인한다:

```bash
git ls-remote --heads origin "$HEAD_BRANCH" | grep -q "$HEAD_BRANCH" \
  || echo "브랜치가 remote에 없습니다. git push 먼저 실행하세요."
```

push 없이 PR 생성을 시도하는 케이스를 사전 차단해 불필요한 422 오류를 방지한다.

**`head` 필드 `owner:branch` 형식 강제**

GitHub API의 `head` 파라미터에 `owner:branch` 형식을 사용하도록 변경했다:

```python
payload = {
    "head": "{owner}:{head_branch}",  # owner: 명시로 한글 브랜치 422 방지
    ...
}
```

단순 브랜치명 대신 `owner:branch` 형식을 사용하면 GitHub API가 레포 컨텍스트를
명확히 인식하여 한글 포함 브랜치명의 422 오류가 해소된다.
