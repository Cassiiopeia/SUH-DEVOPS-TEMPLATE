# suh-commit 자동 add·AI 서명 금지·커밋 메시지 정적 추출

## 개요

`suh-commit` 스킬의 세 가지 사용성·정확성 문제를 한 번에 개선했다. staged 파일이 없을 때 멈춰서 사용자에게 수동 `git add`를 요구하던 동작을 자동 스테이징으로 바꾸고, 커밋에 AI 서명/푸터가 남지 않도록 규칙을 명시했으며, 커밋 메시지의 이슈 제목·URL을 agent의 자유 해석 대신 정규식 기반 정적 추출로 결정화했다.

## 변경 사항

### 핵심 원칙
- `skills/suh-commit/SKILL.md`: staged 파일이 없으면 변경 파일을 자동 `git add` 한다는 규칙으로 교체 (기존 "git add 대신 안 함" → 자동 스테이징)
- `skills/suh-commit/SKILL.md`: 커밋에 `Co-Authored-By`, `Generated with Claude`, `🤖` 등 AI 서명/푸터를 절대 남기지 않는다는 규칙 추가

### 스테이징 (2단계)
- `skills/suh-commit/SKILL.md`: staged 없으면 멈추고 선택지 제시하던 로직을 `git add -A` 자동 실행으로 변경. 이슈별 분리 커밋이 명시된 경우에만 해당 파일만 선택적으로 add

### 이슈 정보 조회 (3단계)
- `skills/suh-commit/SKILL.md`: 이슈 제목·URL을 정규식 기반 정적 추출 스크립트로 결정적으로 뽑도록 변경. `SUH-ISSUE-HELPER`의 `extractIssueTitle` 로직(`[태그]`·이모지·제어문자 정규식 제거)을 Python으로 이식

### 메시지 제안 (5단계)
- `skills/suh-commit/SKILL.md`: 제목·URL은 정적 추출값을 손대지 않고 사용, agent의 자유 판단은 타입(feat/fix/…)과 변경설명으로만 제한

### 커밋 실행 (6단계)
- `skills/suh-commit/SKILL.md`: AI 서명/푸터 및 `--author` 옵션 금지를 명시. 사용자 git 설정 그대로 커밋

## 주요 구현 내용

기존 문제의 핵심은 커밋 메시지의 이슈 제목 부분을 agent가 매번 자유롭게 해석/요약해, 가끔 이슈를 제대로 살피지 않고 엉뚱한 내용을 넣는 것이었다.

해결책으로 `SUH-ISSUE-HELPER`(GitHub Action)의 정적 추출 로직을 그대로 가져왔다:

```
1) [태그] 제거       → re.sub(r"\[.*?\]", "", title)
2) 이모지/제어문자 제거 → So / C / VS16 / ZWJ 범위 제거
3) trim, 빈 문자열이면 원본 fallback
```

커밋 메시지 형식을 `{clean_title} : {타입} : {변경설명} {html_url}`로 고정하고, `clean_title`과 `html_url`은 위 스크립트 출력값을 **그대로** 사용한다. agent가 자유 판단하는 부분은 diff 근거가 있는 타입·변경설명으로만 한정해, 결정적이어야 할 부분과 판단이 필요한 부분을 분리했다.

## 주의사항

- `git push`는 기존대로 스킬에서 실행하지 않는다 (CLAUDE.md 규칙 유지).
- 정적 추출 스크립트는 GitHub API 401(PAT 만료) 시 사용자에게 직접 입력받도록 fallback 처리한다.
- 이모지 제거 정규식은 Python `re` 기준 유니코드 범위를 사용한다. helper의 TS `\p{So}` 대비 일부 기호가 다르게 처리될 수 있어, 실제 이슈 제목으로 추가 검증 권장.
