## ✅ 수정 확인 리포트

macOS bash 3.2 화살표 키 ESC 오인 버그가 해결됐음을 코드로 확인했습니다.

### 확인 내용

**1. ESC 시퀀스 파싱 수정 (`template_integrator.sh`)**
- 화살표 시퀀스를 읽던 `read -rsn2 -t 0.01`(소수점 타임아웃)이 bash 3.2에서 0초로 절삭되어 ESC로 오인되던 문제가, 정수 타임아웃(`-t 1`)으로 교체되어 해결됨.
- 화살표(`ESC [`) + 애플리케이션 커서 모드(`ESC O`) 양쪽 시퀀스를 정상 인식하도록 보강됨.

**2. 번호 입력 방식으로 안정화**
- `template_integrator.ps1`은 화살표+커서 좌표 제어 메뉴를 **완전히 제거**하고 번호 입력 방식(`Invoke-LegacyNumericMenu`)으로 통일됨 — `1,3,5` / `a=전체` / Enter=현재값 유지.
- `template_integrator.sh`도 비-TTY 환경에서 `legacy_numeric_menu`(번호 입력)로 폴백하며, TTY에서는 ESC 버그가 수정된 화살표 메뉴를 사용.

### 결과

화살표 키가 ESC로 오인되어 메뉴가 멋대로 취소되던 현상이 더는 발생하지 않습니다. 번호 입력 방식(`1,2,8` 식)으로도 멀티 선택이 안정적으로 동작합니다.
