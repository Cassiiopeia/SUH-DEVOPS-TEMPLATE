## 🛠️ 구현 완료 리포트

멀티타입 모노레포 서브폴더 버전 동기화(`project_paths`)를 `main`에 구현 완료했습니다. (이슈보다 구현이 먼저 진행되어 히스토리 관리용으로 본 이슈를 등록)

### 핵심 변경

`version.yml`에 타입별 경로 맵을 추가하고, integrator가 이를 자동 감지·기록, `version_manager.sh`가 따라가도록 했습니다. 워크플로우는 무수정입니다.

```yaml
project_types: ["flutter", "react", "python"]
project_paths:                # 신규 — 타입별 프로젝트 폴더 (레포 루트 기준 상대경로)
  flutter: "app"              # app/pubspec.yaml
  react: "client"             # client/package.json
  python: "ai"                # ai/pyproject.toml
```

### 동작 흐름

```
main push
  → PROJECT-COMMON-VERSION-CONTROL (무수정)
    → version_manager.sh increment
       → get_type_path 로 project_paths 읽어 서브폴더 버전 파일까지 동기화
    → git add -A → 커밋 → 태그
```

- `project_paths` **키가 없으면 기존 루트 동작 100% 유지** (완전 하위호환)
- 루트 마커는 질문 없이 `"."` 자동 확정 / 서브폴더는 감지 후 확인 / 비대화형은 `--paths`(`.ps1`은 `-Paths`)
- 한쪽 루트·한쪽 서브폴더인 **비대칭 구조**(예: PickerPicker = 루트 React + `backend/` Python)도 타입별 독립 경로라 그대로 지원
