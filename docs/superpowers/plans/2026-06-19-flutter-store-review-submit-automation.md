**Goal:** 이미 1회 정식 출시된 Flutter 앱의 업데이트에 대해, Android Production 심사 제출 / iOS App Store 심사 제출을 마법사 옵션으로 켜고 끌 수 있게 한다 (기본값은 기존 동작 유지).

**Architecture:** Fastfile은 마법사가 생성한다. 따라서 옵션을 마법사 질문으로 받아 (1) 생성되는 Fastfile lane 구성과 (2) 워크플로우 env 값을 함께 결정한다. 워크플로우는 env가 true일 때만 심사 제출 lane을 추가 호출한다. 검증 기준은 운영 중인 RomRom-FE다.

**Tech Stack:** GitHub Actions YAML, fastlane (Ruby), Bash/PowerShell 마법사 스크립트, HTML/JS 마법사 UI.

## Global Constraints

- **하위호환 절대 원칙:** 옵션 미설정 시 기존 동작과 100% 동일. Android는 internal 트랙까지, iOS는 TestFlight까지만.
- **기본값:** `PLAY_SUBMIT_FOR_REVIEW=false`, `IOS_SUBMIT_FOR_REVIEW=false`.
- **iOS 메타데이터 보호:** 심사 제출 시 `skip_metadata: true`, `skip_screenshots: true`. What's New(release notes)만 갱신.
- **Android 출시 방식:** `internal(completed) → promote production`, `rollout: '1.0'` (100% 전면).
- **첫 출시는 비목표:** 최초 1회 정식 출시는 콘솔 수동. Playwright/브라우저 자동화 부활 금지.
- **워크플로우 실행 로직 무손상:** env 추가·조건부 호출 step 외 기존 `run:`/`uses:`/`with:`/빌드 step은 한 줄도 변경하지 않는다. 변경 후 `git diff`로 자가검증.
- **공통 워크플로우 미러 규칙:** 이 두 워크플로우는 타입별(`project-types/flutter/`)이라 루트 `.github/workflows/` 미러 불필요. 단 작업 시 루트에 동일 파일이 있는지 실제 확인.
- **마법사 Fastfile single source:** Fastfile.playstore를 찍는 모든 경로(setup.sh / apply.sh / apply.ps1 / templates/*.template)가 동일 내용을 생성해야 한다.
- **검증 기준 레포:** RomRom-FE (`d:/0-suh/project/RomRom-FE`). passQL은 신뢰 기준 아님.
- **커밋 컨벤션:** `{이슈제목 순수내용} : {type} : {설명} {이슈URL}`. 이모지·태그 prefix 금지. 이슈 #399 URL: https://github.com/Cassiiopeia/projectops/issues/399

`.github/util/flutter/playstore-wizard/templates/Fastfile.playstore.template` 전체를 다음으로 작성:

```ruby
# Fastfile for Play Store Internal Testing Deployment
# Path: android/fastlane/Fastfile.playstore
# Generated by Flutter Play Store CI/CD Helper
#
# Copy this file to android/fastlane/Fastfile.playstore
# Replace {{APPLICATION_ID}} with your actual application ID.
#
# ============================================================
# ⚠️ 최초 사용 시 반드시 읽어주세요
# ============================================================
# Google Play Store에 한 번도 프로덕션 출시를 하지 않은 앱(Draft App)은
# release_status: 'completed' 및 프로덕션 자동 승급이 불가합니다.
# 최초 1회는 Play Console에서 수동으로 프로덕션 출시를 완료해야 합니다.
# 그 이후부터 internal(completed) → promote production 자동화가 동작합니다.
# ============================================================

default_platform(:android)

platform :android do
  desc "Deploy to Play Store Internal Testing"
  lane :deploy_internal do
    # Environment variables
    aab_path = ENV["AAB_PATH"] || "../build/app/outputs/bundle/release/app-release.aab"
    json_key = ENV["GOOGLE_PLAY_JSON_KEY"] || "~/.config/gcloud/service-account.json"

    puts "========================================="
    puts "Deploying to Play Store Internal Testing"
    puts "========================================="
    puts "AAB Path: #{aab_path}"
    puts "Service Account: #{json_key}"
    puts ""

    unless File.exist?(aab_path)
      UI.user_error!("AAB file not found: #{aab_path}")
    end

    unless File.exist?(json_key)
      UI.user_error!("Service Account JSON not found: #{json_key}")
    end

    # release_status 가이드:
    #   - "draft":     앱이 아직 한 번도 정식 출시되지 않은 경우 (신규 앱)
    #   - "completed": 앱이 이미 정식 출시(Published)된 경우
    upload_to_play_store(
      package_name: "{{APPLICATION_ID}}",
      track: "internal",
      aab: aab_path,
      json_key: json_key,
      skip_upload_apk: true,
      skip_upload_metadata: true,
      skip_upload_images: true,
      skip_upload_screenshots: true,
      skip_upload_changelogs: false,
      release_status: "{{RELEASE_STATUS}}"
    )

    puts ""
    puts "========================================="
    puts "Successfully deployed to Internal Testing!"
    puts "========================================="
  end

  desc "Promote internal track directly to production (Google 심사 요청)"
  lane :promote_internal_to_production do
    json_key = ENV["GOOGLE_PLAY_JSON_KEY"] || "~/.config/gcloud/service-account.json"

    # ⚠️ Draft App(미출시)일 때는 이 lane이 동작하지 않습니다.
    # Play Console에서 최초 1회 수동 프로덕션 출시 후 사용 가능합니다.
    upload_to_play_store(
      package_name: "{{APPLICATION_ID}}",
      track: "internal",
      track_promote_to: "production",
      json_key: json_key,
      skip_upload_apk: true,
      skip_upload_aab: true,
      skip_upload_metadata: true,
      skip_upload_images: true,
      skip_upload_screenshots: true,
      skip_upload_changelogs: true,
      rollout: "1.0"
    )

    puts "✅ 프로덕션 승급 완료 (Google 심사 요청됨)"
  end

  desc "Validate Service Account JSON"
  lane :validate do
    json_key = ENV["GOOGLE_PLAY_JSON_KEY"] || "~/.config/gcloud/service-account.json"

    validate_play_store_json_key(
      json_key: json_key
    )

    puts "Service Account validation successful!"
  end

  desc "Promote internal to beta"
  lane :promote_to_beta do
    json_key = ENV["GOOGLE_PLAY_JSON_KEY"] || "~/.config/gcloud/service-account.json"

    upload_to_play_store(
      package_name: "{{APPLICATION_ID}}",
      track: "internal",
      track_promote_to: "beta",
      json_key: json_key,
      skip_upload_apk: true,
      skip_upload_aab: true,
      skip_upload_metadata: true,
      skip_upload_images: true,
      skip_upload_screenshots: true
    )

    puts "Promoted from internal to beta!"
  end

  desc "Promote beta to production"
  lane :promote_to_production do
    json_key = ENV["GOOGLE_PLAY_JSON_KEY"] || "~/.config/gcloud/service-account.json"

    upload_to_play_store(
      package_name: "{{APPLICATION_ID}}",
      track: "beta",
      track_promote_to: "production",
      json_key: json_key,
      skip_upload_apk: true,
      skip_upload_aab: true,
      skip_upload_metadata: true,
      skip_upload_images: true,
      skip_upload_screenshots: true,
      rollout: "0.1"
    )

    puts "Promoted from beta to production (10% rollout)!"
  end
end
```

```ruby
  desc "App Store 정식 심사 제출 (이미 출시된 앱의 업데이트용)"
  # ⚠️ 최초 1회 App Store 정식 출시가 완료된 앱에서만 동작합니다.
  #    메타데이터·스크린샷은 건드리지 않고 빌드 + What's New만 갱신해 심사 제출합니다.

    UI.message("🚀 App Store 심사 제출 시작")

    api_key = app_store_connect_api_key(
      key_id: ENV["APP_STORE_CONNECT_API_KEY_ID"],
      issuer_id: ENV["APP_STORE_CONNECT_ISSUER_ID"],
      key_filepath: ENV["API_KEY_PATH"],
      duration: 1200,
      in_house: false
    )

    # Ruby 3.4+ frozen string 호환성: .dup으로 mutable 복사본 생성
    release_notes = (ENV["RELEASE_NOTES"] || "버그 수정 및 개선 사항이 반영되었습니다.").dup
    UI.message("📝 릴리즈 노트: #{release_notes}")

    upload_to_app_store(
      api_key: api_key,
      release_notes: {
        "ko" => release_notes.dup,
        "en-US" => release_notes.dup
      },
      skip_binary_upload: true,
      skip_metadata: true,
      skip_screenshots: true,
      submit_for_review: true,
      automatic_release: false,
      force: true,
      precheck_include_in_app_purchases: false,
      submission_information: {
        add_id_info_uses_idfa: false
      }
    )

    UI.success("✅ App Store 심사 제출 완료!")
  end
```
