# 메뉴 UX 개선 + IDE Skills 통합 설치 화면 설계

작성일: 2026-06-11
대상: `template_integrator.sh` (sh 전용 — ps1/Windows는 숫자 입력 방식 유지)

---

## 배경

passQL에 템플릿을 통합하면서 메뉴 UX 관련 문제·요구가 나왔다.

1. Y/N 프롬프트가 단일 키 입력이라, 화살표 메뉴(`choose_menu`)와 조작 방식이 일관되지 않았다.
2. 화살표 메뉴가 화면 하단/좁은 폭에서 redraw 시 첫 줄이 중복되는 잔상 버그가 있었다.
3. IDE Skills 설치가 Claude/Cursor/Gemini/Codex를 **순차로 4번 따로 Y/N**으로 물어, 전체를 한눈에 보고 한 번에 처리할 수 없었다.

> Windows(`template_integrator.ps1`)는 원격 iex 실행 환경에서 커서 좌표 제어가 불안정해 의도적으로 숫자 입력 방식으로 통일돼 있다(주석 명시). 따라서 이 설계의 화살표/멀티셀렉트 변경은 sh에만 적용하고 ps1은 현행 유지한다.

---

## 변경 ① ask_yes_no 화살표 메뉴화 (완료)

### 내용
`ask_yes_no` 내부만 `choose_menu` 기반 2지선(예/아니오)으로 재작성. 시그니처(`ask_yes_no "프롬프트" "기본값(Y|N)"` → 0/1)는 그대로라 호출부 13곳 무수정으로 자동 적용.

- 기본값이 커서 초기 위치 (Y → "예"가 첫 항목)
- value는 yes/no, 라벨은 예/아니오
- 프롬프트 끝의 `(Y/N, 기본: Y):` 같은 꼬리표는 sed로 제거(choose_menu가 자체 안내를 붙이므로 중복 방지)
- TTY 없음 / FORCE 모드 → 기존 텍스트 입력 폴백

### 검증
Python pty로 3케이스 통과: Enter→YES(0), DOWN+Enter→NO(1), 예/아니오 라벨 렌더.

## 변경 ② choose_menu redraw 보강 (부분 — 환경 한계 존재)

### 문제
`_interactive_menu_clear`가 `ESC[1A`(한 줄 위로)를 n번 실행 → "1항목=1줄" 가정. 라벨 wrap이나 화면 하단 스크롤이 생기면 좌표가 어긋나 잔상이 누적("점점 늘어남")됐다.

### 보강
메뉴가 차지할 공간(`n*2+1`줄)을 미리 출력해 스크롤을 선제적으로 일으킨 뒤 그만큼 커서를 되올려 앵커 저장(ESC7). redraw마다 앵커 복원(ESC8) + 화면 끝까지 삭제(ESC[J).

### 한계 (알려진 이슈)
VSCode 내장 터미널 + iex/원격 실행에서는 ESC7/8/J 커서 제어가 여전히 불안정해, 첫 줄이 가끔 중복될 수 있다. 사용자 판단으로 "큰 문제 아님 → 현행 유지"로 결정. 근본 해결(숫자 방식 전환)은 보류.

## 변경 ③ IDE Skills 통합 관리 — 2단계 화면 (신규)

### 현재 (4번 순차 질문)
상태는 한눈에 보여주지만, 관리는 Claude → Cursor → Gemini → Codex를 각각 Y/N으로 물어본다.

### 개선 (2단계)
**1단계 — 상태 표시 + 동작 선택 (단일 선택 메뉴)**
```
☀ IDE Skills 현재 상태
  Claude Code  user  v3.0.110  ✓ 최신
  Cursor      : 미설치
  Gemini CLI   : 감지
  Codex CLI    : 감지

무엇을 하시겠습니까?
  1) 설치/업데이트   (최신 상태로 맞춤 — 미설치는 설치, 설치됨은 업데이트)
  2) 제거
  3) 건너뛰기
```

**2단계 — 동작 적용 대상 IDE 멀티셀렉트**
```
설치/업데이트할 IDE 선택 (Space 토글, a 전체, Enter 확정)
  [✓] Claude Code
  [ ] Cursor
  [✓] Gemini CLI
  [✓] Codex CLI
```
- Enter 시 체크된 IDE를 일괄 처리
- "제거" 선택 시 2단계는 **설치된 IDE만** 후보로
- "건너뛰기" → 아무 동작 없음
- 기본 체크: 설치/업데이트는 감지된 IDE 모두 체크, 제거는 전부 해제 시작

### 동작 의미
- 설치/업데이트 = 최신화 (미설치→설치, 설치됨→업데이트/재적용)
- scope 변경 등 세부 동작은 이번 범위에서 제외 (필요 시 후속)

### 영향
- 기존 `offer_ide_tools_install`의 4개 순차 섹션을 2단계 흐름으로 재구성
- 각 IDE의 실제 설치/업데이트/제거 실행 로직(claude plugin update, Cursor 복사, Gemini extension, Codex marketplace)은 재사용 — 호출 위치만 2단계 뒤로 이동
